# 双亲委派机制原理

双亲委派机制是Java类加载器的核心工作原理，我来详细解释一下：

## 基本原理

双亲委派机制的工作流程是：当一个类加载器收到类加载请求时，它不会立即加载该类，而是先将请求委派给父类加载器。只有当父类加载器无法完成加载时，子类加载器才会尝试自己加载。

## 类加载器层次结构

```
Bootstrap ClassLoader (启动类加载器)
    ↓
Extension ClassLoader (扩展类加载器) 
    ↓
Application ClassLoader (应用程序类加载器)
    ↓
Custom ClassLoader (自定义类加载器)
```

## 具体执行流程

1. **接收请求**：应用程序类加载器收到加载`java.lang.String`的请求
2. **向上委派**：委派给扩展类加载器
3. **继续委派**：扩展类加载器委派给启动类加载器
4. **尝试加载**：启动类加载器发现可以加载该类，直接加载并返回
5. **返回结果**：类加载完成，无需下级加载器参与

## 核心实现代码

```java
protected Class<?> loadClass(String name, boolean resolve) throws ClassNotFoundException {
    // 首先检查类是否已经被加载
    Class<?> c = findLoadedClass(name);
    if (c == null) {
        try {
            if (parent != null) {
                // 委派给父类加载器
                c = parent.loadClass(name, false);
            } else {
                // 如果没有父类加载器，委派给启动类加载器
                c = findBootstrapClassOrNull(name);
            }
        } catch (ClassNotFoundException e) {
            // 父类加载器无法加载时，才由当前加载器尝试加载
            c = findClass(name);
        }
    }
    return c;
}
```

## 设计目的

1. **安全性**：防止核心API被恶意替换。例如，即使你自定义了一个`java.lang.String`类，也会被启动类加载器拦截，加载JDK自带的String类
    
2. **避免重复加载**：确保同一个类在JVM中只有一个Class对象实例
    
3. **稳定性**：保证Java核心库的优先级和完整性
