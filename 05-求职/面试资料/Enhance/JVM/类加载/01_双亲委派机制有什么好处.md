# 双亲委派机制有什么好处

双亲委派机制（Parent Delegation Model）是Java类加载器的核心工作机制，它带来了以下几个重要好处：

## 1. 避免类的重复加载

当父类加载器已经加载了某个类时，子类加载器就不会再次加载，确保JVM中每个类只被加载一次。这避免了内存浪费和类定义冲突。

## 2. 保证核心类库的安全性

Java核心类库（如java.lang.Object、java.lang.String等）只能由Bootstrap ClassLoader加载，防止应用程序恶意替换或篡改核心类。即使你在应用中创建一个java.lang.String类，也不会被加载，因为Bootstrap ClassLoader已经加载了真正的String类。

## 3. 维护类的唯一性和一致性

确保相同全限定名的类在整个JVM中保持唯一，避免了类型转换异常和不一致问题。比如不同类加载器加载的同名类，在JVM中被认为是不同的类型。

## 4. 提供良好的层次结构

形成了清晰的类加载层次：

- Bootstrap ClassLoader（启动类加载器）
- Extension ClassLoader（扩展类加载器）
- Application ClassLoader（应用程序类加载器）
- 自定义类加载器

## 相关扩展知识

**打破双亲委派的场景：**

- SPI（Service Provider Interface）机制，如JDBC驱动加载
- OSGI框架的模块化类加载
- 应用服务器的热部署功能
- Tomcat的Web应用隔离

**类加载器的工作流程：**

1. 检查类是否已被加载
2. 如未加载，委派给父加载器
3. 父加载器无法加载时，由当前加载器尝试加载
4. 抛出ClassNotFoundException（如果都无法加载）

**判断类是否相同的条件：**

- 全限定名相同
- 加载该类的类加载器相同

这种机制是Java安全模型的重要组成部分，确保了Java应用的稳定性和安全性。
