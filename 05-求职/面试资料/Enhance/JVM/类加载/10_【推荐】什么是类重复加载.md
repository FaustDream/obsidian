# 什么是类重复加载

类重复加载是指在同一个Java虚拟机中，同一个类被不同的类加载器加载多次，导致产生多个不同的Class对象的现象。

## 什么情况下会发生类重复加载

1. **不同类加载器加载同一个类**
    
    - 即使类名相同、字节码相同，但由不同的ClassLoader加载，JVM会认为它们是不同的类
    - 例如：应用类加载器和自定义类加载器都加载了同一个类
2. **Web应用中的热部署**
    
    - 当应用重新部署时，容器会创建新的类加载器
    - 旧的类加载器及其加载的类不会立即被垃圾回收，可能导致重复加载
3. **动态加载场景**
    
    - 插件系统中多次加载同一个插件
    - OSGi等模块化框架中的类加载

## 类重复加载的影响

**内存泄漏风险**

```java
// 示例：自定义类加载器重复加载同一个类
public class CustomClassLoader extends ClassLoader {
    @Override
    protected Class<?> findClass(String name) throws ClassNotFoundException {
        // 每次都重新加载类，不使用缓存
        byte[] classData = loadClassData(name);
        return defineClass(name, classData, 0, classData.length);
    }
}

// 多次创建类加载器加载同一个类
CustomClassLoader loader1 = new CustomClassLoader();
CustomClassLoader loader2 = new CustomClassLoader();
Class<?> clazz1 = loader1.loadClass("com.example.MyClass");
Class<?> clazz2 = loader2.loadClass("com.example.MyClass");
// clazz1 != clazz2，这是两个不同的Class对象
```

**类型转换异常**

```java
// 不同类加载器加载的同名类无法相互转换
Object obj1 = clazz1.newInstance();
Object obj2 = clazz2.newInstance();
// 以下代码会抛出ClassCastException
MyClass typedObj = (MyClass) obj1; // 如果MyClass是由不同加载器加载的
```

## 如何避免类重复加载

1. **遵循双亲委派模型**
    
    - 优先使用父类加载器加载类
    - 避免破坏双亲委派机制
2. **合理使用类加载器缓存**
    

```java
public class CachedClassLoader extends ClassLoader {
    private Map<String, Class<?>> cache = new ConcurrentHashMap<>();
    
    @Override
    protected Class<?> findClass(String name) throws ClassNotFoundException {
        Class<?> clazz = cache.get(name);
        if (clazz == null) {
            byte[] classData = loadClassData(name);
            clazz = defineClass(name, classData, 0, classData.length);
            cache.put(name, clazz);
        }
        return clazz;
    }
}
```

3. **正确管理类加载器生命周期**
    - 及时释放不再需要的类加载器引用
    - 避免静态变量持有类加载器引用

## 检测类重复加载的方法

1. **使用JVM参数**

```bash
-verbose:class  # 输出类加载信息
-XX:+TraceClassLoading  # 跟踪类加载过程
```

2. **程序化检测**

```java
// 检查同一个类是否被不同加载器加载
public void detectDuplicateClasses() {
    Class<?> clazz1 = Class.forName("com.example.MyClass");
    Class<?> clazz2 = anotherClassLoader.loadClass("com.example.MyClass");
    
    if (clazz1 != clazz2) {
        System.out.println("检测到类重复加载");
        System.out.println("类加载器1: " + clazz1.getClassLoader());
        System.out.println("类加载器2: " + clazz2.getClassLoader());
    }
}
```

## 相关知识扩展

**类的唯一性识别** 在JVM中，类的唯一性由类的全限定名和加载它的类加载器共同决定。即使是同样的字节码，被不同的类加载器加载后也会产生不同的Class对象。

**MetaSpace内存管理** Java 8之后，类的元数据存储在MetaSpace中。类重复加载会导致MetaSpace内存占用增加，严重时可能导致MetaSpace OOM。

类重复加载是一个需要重点关注的问题，特别是在企业级应用和容器化环境中，合理的类加载器设计和管理对于应用的稳定性和性能都至关重要。
