# JVM堆的基本结构

JVM堆是Java虚拟机中最重要的内存区域，用于存储对象实例和数组。让我详细解释堆的基本结构：

## 堆的整体划分

**按代划分（分代收集理论）：**

- **新生代（Young Generation）**：存放新创建的对象
- **老年代（Old Generation）**：存放长期存活的对象
- **永久代（PermGen）**：JDK 8之前存放类元数据（已被元空间替代）

## 新生代结构

新生代进一步细分为：

**Eden区**

- 新对象首先分配到Eden区
- 占新生代空间的大部分（默认约80%）

**Survivor区**

- 分为S0和S1两个区域
- 每个Survivor区占新生代约10%
- 任何时刻只有一个Survivor区被使用

**工作机制：**

```
Eden区满 → 触发Minor GC → 存活对象移到Survivor区 → 
多次GC后仍存活 → 晋升到老年代
```

## 老年代结构

- 存放经历多次GC仍存活的对象
- 对象晋升条件：
    - 在Survivor区中经历足够多的GC周期（默认15次）
    - 大对象直接进入老年代
    - Survivor区空间不足时

## 内存分配策略

**对象分配流程：**

1. 新对象优先在Eden区分配
2. 大对象直接进入老年代
3. 长期存活对象进入老年代
4. 动态年龄判断

## 相关JVM参数

```bash
# 堆大小设置
-Xms: 初始堆大小
-Xmx: 最大堆大小
-Xmn: 新生代大小

# 新生代比例
-XX:NewRatio=3  # 老年代:新生代 = 3:1
-XX:SurvivorRatio=8  # Eden:Survivor = 8:1:1

# 晋升阈值
-XX:MaxTenuringThreshold=15
```

## 垃圾收集器选择

不同的垃圾收集器适用于不同的堆区域：

- **Serial/ParNew**：适用于新生代
- **CMS/G1**：可用于整个堆
- **ZGC/Shenandoah**：低延迟收集器

## 性能优化考虑

**堆大小调优：**

- 根据应用特点调整新生代和老年代比例
- 避免堆过大导致GC时间过长
- 避免堆过小导致频繁GC

**监控指标：**

- 堆使用率
- GC频率和时间
- 对象晋升率

这种分代设计基于"大多数对象朝生夕死"的假设，通过不同的GC策略优化不同生命周期对象的管理，是JVM内存管理的核心设计思想。
