# jvm垃圾回收算法

## JVM垃圾回收算法详解

### 1. 标记-清除算法（Mark-Sweep）

**原理：**

- 标记阶段：从GC Root开始，标记所有可达的对象
- 清除阶段：遍历整个堆，回收未被标记的对象

**优点：**

- 实现简单，不需要移动对象
- 对于存活对象较多的场景效率较高

**缺点：**

- 产生内存碎片，可能导致大对象无法分配
- 清除过程需要遍历整个堆，效率较低

**适用场景：** 老年代收集器（如CMS）

### 2. 标记-复制算法（Mark-Copy）

**原理：**

- 将内存分为两个相等的区域
- 只使用其中一个区域，垃圾回收时将存活对象复制到另一个区域
- 清空原区域

**优点：**

- 没有内存碎片
- 分配内存时只需移动指针，效率高
- 适合存活对象少的场景

**缺点：**

- 浪费50%的内存空间
- 复制成本与存活对象数量成正比

**适用场景：** 新生代收集器（如Serial、ParNew）

### 3. 标记-整理算法（Mark-Compact）

**原理：**

- 标记阶段：标记所有存活对象
- 整理阶段：将所有存活对象移动到内存的一端
- 清理阶段：清除边界外的所有内存

**优点：**

- 没有内存碎片
- 不浪费内存空间
- 适合存活对象较多的场景

**缺点：**

- 移动对象成本较高
- 需要更新对象引用

**适用场景：** 老年代收集器（如Serial Old、Parallel Old）

### 4. 分代收集算法

**核心思想：** 根据对象生命周期的不同，将堆内存分为不同的代，使用不同的收集算法。

**分代假设：**

- 大部分对象朝生夕死
- 熬过多次GC的对象难以死掉

**具体实现：**

- **新生代：** 使用复制算法，分为Eden区和两个Survivor区（8:1:1）
- **老年代：** 使用标记-清除或标记-整理算法

### 5. 增量收集算法

**原理：** 将GC过程分解为多个小步骤，与应用程序交替执行，减少停顿时间。

**优点：**

- 减少GC停顿时间
- 提高系统响应性

**缺点：**

- 总体GC时间可能增加
- 实现复杂度高

### 6. 并发收集算法

**原理：** GC线程与应用线程并发执行，通过各种技术保证正确性。

**关键技术：**

- **三色标记法：** 白色（未访问）、灰色（已访问未处理）、黑色（已处理）
- **写屏障：** 监控对象引用的变化
- **读屏障：** 在读取对象时进行额外处理

### 7. 现代垃圾收集器中的算法应用

**G1收集器：**

- 将堆划分为多个Region
- 使用复制算法清理Region
- 采用增量和并发技术

**ZGC/Shenandoah：**

- 基于Region的内存管理
- 使用并发标记和并发整理
- 通过染色指针/Brooks指针实现并发移动

**Epsilon GC：**

- 无操作收集器，不进行任何回收
- 用于性能测试和短生命周期应用

### 实际应用中的选择策略

**选择依据：**

1. **应用特性：** 吞吐量优先 vs 延迟优先
2. **内存模式：** 对象生命周期分布
3. **硬件资源：** CPU核心数、内存大小
4. **业务需求：** 停顿时间要求

**推荐配置：**

- **高吞吐量场景：** Parallel GC
- **低延迟场景：** G1、ZGC或Shenandoah
- **内存受限场景：** Serial GC
- **大堆场景：** ZGC或Shenandoah

理解这些算法的特点和适用场景，能帮助我们在实际项目中做出正确的GC调优决策。
