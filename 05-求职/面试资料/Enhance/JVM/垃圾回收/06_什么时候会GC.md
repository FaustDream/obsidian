# 什么时候会GC

## 什么时候会触发GC（垃圾回收）？

Java的垃圾回收器会在以下几种情况下触发：

### 1. 堆内存不足时（主要触发条件）

- **年轻代空间不足**：Eden区满了，触发Minor GC
- **老年代空间不足**：老年代内存不够，触发Major GC或Full GC
- **元空间不足**：方法区/元空间满了，触发Full GC

### 2. 手动调用

```java
System.gc(); // 建议JVM进行垃圾回收，但不保证立即执行
Runtime.getRuntime().gc(); // 同上
```

### 3. JVM内部策略触发

- **分配担保失败**：年轻代对象晋升到老年代时，老年代空间不足
- **CMS收集器的并发模式失败**：并发收集过程中老年代空间不足
- **G1收集器的混合收集**：根据停顿时间目标定期触发

### 4. 特定算法的触发时机

#### Minor GC（年轻代GC）

- Eden区内存分配失败
- 新对象无法在Eden区分配空间

#### Major GC（老年代GC）

- 老年代空间不足
- 持久代/元空间不足（JDK8之前/之后）

#### Full GC（全堆GC）

- 老年代空间不足
- 持久代/元空间不足
- 手动调用System.gc()
- CMS收集器的Concurrent Mode Failure

### 5. 实际开发中的触发场景

```java
// 1. 大量临时对象创建
for(int i = 0; i < 1000000; i++) {
    String temp = new String("临时对象" + i); // 频繁创建对象
}

// 2. 内存泄漏导致
List<Object> list = new ArrayList<>();
while(true) {
    list.add(new Object()); // 对象无法被回收
}

// 3. 大对象分配
byte[] bigArray = new byte[100 * 1024 * 1024]; // 100MB大对象
```

### 6. 监控和调试GC

```bash
# JVM参数查看GC日志
-XX:+PrintGC
-XX:+PrintGCDetails
-XX:+PrintGCTimeStamps

# 示例输出
[GC (Allocation Failure) [PSYoungGen: 2048K->512K(2560K)] 2048K->520K(9728K), 0.0018270 secs]
```

### 相关知识扩展

**GC算法选择影响触发时机：**

- **串行GC**：单线程，适合小应用
- **并行GC**：多线程，适合吞吐量优先
- **CMS GC**：低延迟，适合响应时间敏感应用
- **G1 GC**：可预测停顿时间，适合大堆应用

**优化建议：**

- 合理设置堆大小参数（-Xms, -Xmx）
- 选择合适的垃圾收集器
- 避免频繁创建大对象
- 及时释放不再使用的对象引用

理解GC触发时机有助于优化应用性能，避免因为频繁GC导致的应用卡顿。
