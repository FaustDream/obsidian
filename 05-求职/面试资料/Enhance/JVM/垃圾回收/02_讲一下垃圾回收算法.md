# 讲一下垃圾回收算法

## 垃圾回收算法详解

垃圾回收（Garbage Collection, GC）是Java虚拟机自动管理内存的核心机制。我来详细讲解主要的垃圾回收算法：

### 1. 标记-清除算法（Mark-Sweep）

**工作原理：**

- **标记阶段**：从GC Root开始，标记所有可达的对象
- **清除阶段**：清除所有未被标记的对象，释放内存空间

**优缺点：**

- 优点：实现简单，不需要移动对象
- 缺点：会产生大量内存碎片，影响后续大对象分配

### 2. 复制算法（Copying）

**工作原理：**

- 将内存分为两个相等的区域（From区和To区）
- 只使用其中一个区域，当发生GC时，将存活对象复制到另一个区域
- 清空原区域，交换两个区域的角色

**优缺点：**

- 优点：没有内存碎片，分配内存简单高效
- 缺点：内存利用率只有50%，复制开销较大

**适用场景：** 新生代回收（对象存活率低）

### 3. 标记-整理算法（Mark-Compact）

**工作原理：**

- **标记阶段**：标记所有存活对象
- **整理阶段**：将存活对象向内存一端移动，清除边界外的内存

**优缺点：**

- 优点：没有内存碎片，内存利用率高
- 缺点：整理过程需要移动对象，成本较高

**适用场景：** 老年代回收（对象存活率高）

### 4. 分代收集算法

**核心思想：** 根据对象生存周期将内存分为不同代，对不同代采用不同的回收算法

**分代结构：**

- **新生代**：Eden区 + 2个Survivor区，使用复制算法
- **老年代**：使用标记-清除或标记-整理算法

**工作流程：**

1. 新对象在Eden区分配
2. Eden区满时触发Minor GC，存活对象复制到Survivor区
3. 经过多次GC仍存活的对象晋升到老年代
4. 老年代空间不足时触发Major GC

### 5. 增量收集算法

**目标：** 减少GC停顿时间

**实现方式：**

- 将GC工作分解为多个小步骤
- 与应用程序交替执行
- 每次只处理一小部分对象

### 6. 三色标记算法

**用于并发GC的核心算法：**

- **白色**：未被访问的对象
- **灰色**：已被访问但其引用还未全部扫描的对象
- **黑色**：已被访问且其引用已全部扫描的对象

**并发问题解决：**

- **写屏障**：记录并发期间的引用变化
- **读屏障**：在读取引用时进行额外检查

### 现代GC收集器应用

**G1收集器**：

- 结合了多种算法思想
- 将堆分为多个Region
- 优先回收垃圾最多的Region

**ZGC/Shenandoah**：

- 基于Region的并发收集器
- 使用着色指针技术
- 实现超低延迟回收

### 算法选择考虑因素

1. **应用类型**：吞吐量优先 vs 延迟优先
2. **堆内存大小**：大堆适合并发收集器
3. **对象生存特征**：年轻代对象死亡率高
4. **硬件配置**：多核CPU适合并行收集

### 性能调优要点

**新生代调优：**

- 合理设置Eden和Survivor比例
- 控制对象晋升年龄阈值

**老年代调优：**

- 选择合适的收集算法
- 避免频繁的Full GC

**并发调优：**

- 调整并发线程数
- 优化并发标记触发时机

这些算法的演进体现了Java在内存管理上的不断优化，从简单的串行回收到现代的低延迟并发回收，每种算法都有其特定的适用场景和权衡考虑。
