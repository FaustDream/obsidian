# 静态代码块存在哪里

静态代码块存在于**方法区（Method Area）**中，更具体地说是存储在**类的元数据信息**里。

## 详细解释

### 存储位置

静态代码块的**字节码指令**存储在方法区中，作为类定义的一部分。当类被加载到JVM时，静态代码块会被编译器转换为一个特殊的静态方法`<clinit>()`，这个方法的字节码就保存在方法区中。

### 执行时机

静态代码块在**类初始化阶段**执行，具体时机是：

- 类第一次被加载时
- 创建类的第一个实例之前
- 访问类的静态变量或静态方法之前
- 使用反射调用类之前

### 内存模型演示

```java
public class StaticBlockExample {
    static {
        System.out.println("静态代码块执行");
    }
    
    static int value = 10;
    
    public static void main(String[] args) {
        // 静态代码块在main方法执行前就已经执行了
    }
}
```

## 相关知识扩展

### 1. JVM内存结构中的位置

- **JDK 8之前**：存储在永久代（PermGen）
- **JDK 8及以后**：存储在元空间（Metaspace）

### 2. 静态代码块的特点

- **执行顺序**：按照在代码中出现的顺序执行
- **执行次数**：只执行一次
- **异常处理**：如果抛出异常，类初始化失败

### 3. 与其他初始化的关系

```java
public class InitOrder {
    static int a = 1;           // 1. 静态变量初始化
    
    static {                    // 2. 静态代码块
        System.out.println("static block");
        a = 2;
    }
    
    static int b = 3;           // 3. 后续静态变量初始化
}
```

### 4. 实际应用场景

- 初始化静态变量
- 加载配置文件
- 注册驱动程序
- 执行一次性的初始化工作

这样的设计确保了静态代码块在类的生命周期中只执行一次，且在任何静态成员被访问之前完成初始化工作。
