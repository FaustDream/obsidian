# 堆内存分配策略

## 堆内存分配策略

堆内存分配是JVM内存管理的核心部分，涉及对象的创建、存储和回收。让我详细解释一下主要的分配策略：

### 1. 对象优先在Eden区分配

**基本原理：**

- 新创建的对象首先尝试在年轻代的Eden区分配内存
- Eden区是一个连续的内存空间，分配速度很快
- 当Eden区空间不足时，触发Minor GC

**示例代码：**

```java
public class HeapAllocationExample {
    public static void main(String[] args) {
        // 这些对象会优先在Eden区分配
        String str1 = new String("Hello");
        List<Integer> list = new ArrayList<>();
        Person person = new Person("张三", 25);
    }
}
```

### 2. 大对象直接进入老年代

**什么是大对象：**

- 需要大量连续内存空间的对象（如长字符串、大数组）
- 通过`-XX:PretenureSizeThreshold`参数设置阈值

**为什么这样设计：**

- 避免在Eden区和Survivor区之间复制大对象
- 减少内存碎片
- 提高GC效率

```java
public class LargeObjectExample {
    public static void main(String[] args) {
        // 大数组直接进入老年代
        byte[] largeArray = new byte[1024 * 1024 * 10]; // 10MB
        
        // 长字符串也可能直接进入老年代
        StringBuilder sb = new StringBuilder();
        for (int i = 0; i < 100000; i++) {
            sb.append("这是一个很长的字符串");
        }
        String largeString = sb.toString();
    }
}
```

### 3. 长期存活的对象将进入老年代

**年龄判断机制：**

- 对象每经历一次Minor GC，年龄增加1
- 默认年龄阈值为15（`-XX:MaxTenuringThreshold`）
- 达到阈值后晋升到老年代

**对象年龄计算：**

```java
public class ObjectAgeExample {
    private static List<Object> cache = new ArrayList<>();
    
    public static void main(String[] args) {
        // 这些对象会在多次GC后进入老年代
        for (int i = 0; i < 1000; i++) {
            cache.add(new Person("用户" + i, 20 + i % 50));
            
            // 偶尔触发GC
            if (i % 100 == 0) {
                System.gc(); // 仅用于演示，实际开发中避免使用
            }
        }
    }
}
```

### 4. 动态对象年龄判定

**Survivor区空间分析：**

- 如果Survivor区中相同年龄的所有对象大小总和大于Survivor空间的一半
- 年龄大于或等于该年龄的对象可以直接进入老年代

**空间担保机制：**

- 在Minor GC之前，JVM会检查老年代最大可用连续空间
- 如果大于新生代所有对象总空间，Minor GC是安全的
- 否则查看`HandlePromotionFailure`设置

### 5. 内存分配与回收策略的优化

**TLAB（Thread Local Allocation Buffers）：**

```java
public class TLABExample {
    public static void main(String[] args) {
        // 每个线程都有自己的TLAB
        Thread t1 = new Thread(() -> {
            for (int i = 0; i < 10000; i++) {
                new Object(); // 在该线程的TLAB中分配
            }
        });
        
        Thread t2 = new Thread(() -> {
            for (int i = 0; i < 10000; i++) {
                new String("线程2-" + i); // 在该线程的TLAB中分配
            }
        });
        
        t1.start();
        t2.start();
    }
}
```

### 6. 相关JVM参数配置

**重要参数：**

```bash
# 堆内存大小
-Xms2g -Xmx4g

# 新生代大小
-Xmn1g
-XX:NewRatio=3  # 老年代:新生代 = 3:1

# Eden区和Survivor区比例
-XX:SurvivorRatio=8  # Eden:Survivor1:Survivor2 = 8:1:1

# 大对象阈值
-XX:PretenureSizeThreshold=1m

# 对象年龄阈值
-XX:MaxTenuringThreshold=15

# TLAB相关
-XX:+UseTLAB
-XX:TLABSize=256k
```

### 7. 实际应用中的考虑

**性能优化建议：**

1. **避免创建过多临时对象**

```java
// 不好的做法
public String badConcatenation() {
    String result = "";
    for (int i = 0; i < 1000; i++) {
        result += "item" + i; // 每次都创建新的String对象
    }
    return result;
}

// 好的做法
public String goodConcatenation() {
    StringBuilder sb = new StringBuilder();
    for (int i = 0; i < 1000; i++) {
        sb.append("item").append(i);
    }
    return sb.toString();
}
```

2. **合理使用对象池**

```java
public class ObjectPoolExample {
    private static final Queue<ExpensiveObject> pool = 
        new ConcurrentLinkedQueue<>();
    
    public static ExpensiveObject borrowObject() {
        ExpensiveObject obj = pool.poll();
        return obj != null ? obj : new ExpensiveObject();
    }
    
    public static void returnObject(ExpensiveObject obj) {
        obj.reset(); // 重置对象状态
        pool.offer(obj);
    }
}
```

### 8. 扩展知识点

**垃圾回收器对分配策略的影响：**

- **Serial GC**: 适合单线程应用，分配简单直接
- **Parallel GC**: 多线程并行回收，分配策略相似
- **G1 GC**: 区域化管理，分配策略更复杂
- **ZGC/Shenandoah**: 低延迟收集器，分配策略特殊

**监控和调优：**

```java
// 监控堆内存使用情况
MemoryMXBean memoryMXBean = ManagementFactory.getMemoryMXBean();
MemoryUsage heapMemoryUsage = memoryMXBean.getHeapMemoryUsage();

System.out.println("堆内存使用情况：");
System.out.println("初始大小: " + heapMemoryUsage.getInit() / 1024 / 1024 + "MB");
System.out.println("已使用: " + heapMemoryUsage.getUsed() / 1024 / 1024 + "MB");
System.out.println("最大可用: " + heapMemoryUsage.getMax() / 1024 / 1024 + "MB");
```

这些分配策略的目标是最大化内存利用率，减少GC频率和暂停时间，从而提高应用程序的整体性能。在实际开发中，需要根据应用特点选择合适的JVM参数和编程模式。
