# 常量池在哪？

在Java中，常量池主要分为以下几种类型，它们位于不同的内存区域：

## 1. 字符串常量池（String Pool）

- **位置**：位于**堆内存**中
- **JDK版本变化**：
    - JDK 1.7之前：位于方法区（永久代）
    - JDK 1.7及之后：移到了堆内存中
- **作用**：存储字符串字面量，避免重复创建相同的字符串对象

```java
String s1 = "hello";  // 存储在字符串常量池
String s2 = "hello";  // 直接从常量池获取
System.out.println(s1 == s2); // true
```

## 2. 运行时常量池（Runtime Constant Pool）

- **位置**：位于**方法区**中
- **JDK版本变化**：
    - JDK 1.8之前：在永久代（PermGen）
    - JDK 1.8及之后：在元空间（Metaspace）
- **作用**：存放编译期生成的各种字面量和符号引用

## 3. 类文件常量池（Class File Constant Pool）

- **位置**：在**.class文件**中
- **作用**：编译时生成，存储类的元信息、字面量、符号引用等

## 内存区域总结

```
JDK 1.8+ 内存布局：
┌─────────────────┐
│      堆内存      │ ← 字符串常量池
├─────────────────┤
│     方法区      │ ← 运行时常量池
│   (元空间)      │
├─────────────────┤
│     虚拟机栈     │
├─────────────────┤
│     本地方法栈   │
├─────────────────┤
│     程序计数器   │
└─────────────────┘
```

## 相关知识扩展

**为什么字符串常量池要从永久代移到堆？**

1. 永久代空间有限，容易出现OOM
2. 堆空间更大，GC机制更完善
3. 字符串是应用数据，放在堆中更合理

**intern()方法的作用：**

```java
String s1 = new String("hello");
String s2 = s1.intern(); // 将字符串放入常量池
```

这样的设计既保证了内存的有效利用，又提高了程序的执行效率。
