# 括号平衡算法用什么数据结构实现

括号平衡算法最适合用**栈（Stack）**数据结构来实现。

## 核心思路

栈的**后进先出（LIFO）**特性完美契合括号匹配的逻辑：

- 遇到左括号时入栈
- 遇到右括号时出栈匹配
- 最终栈为空则括号平衡

## 具体实现

```java
public boolean isBalanced(String s) {
    Stack<Character> stack = new Stack<>();
    
    for (char c : s.toCharArray()) {
        // 左括号入栈
        if (c == '(' || c == '[' || c == '{') {
            stack.push(c);
        }
        // 右括号匹配
        else if (c == ')' || c == ']' || c == '}') {
            if (stack.isEmpty()) return false;
            
            char top = stack.pop();
            if (!isMatch(top, c)) return false;
        }
    }
    
    return stack.isEmpty();
}

private boolean isMatch(char open, char close) {
    return (open == '(' && close == ')') ||
           (open == '[' && close == ']') ||
           (open == '{' && close == '}');
}
```

## 时间和空间复杂度

- **时间复杂度**：O(n)，需要遍历整个字符串
- **空间复杂度**：O(n)，最坏情况下所有字符都是左括号

## 扩展知识

**其他数据结构的局限性：**

- **数组**：需要额外维护索引，不如栈直观
- **队列**：先进先出特性不适合括号匹配
- **链表**：可以模拟栈，但操作复杂度更高

**实际应用场景：**

- 编译器的语法分析
- 数学表达式计算
- HTML/XML标签匹配
- 代码编辑器的括号高亮

**优化方案：** 对于只包含一种括号的情况，可以用简单的计数器代替栈，空间复杂度降为O(1)。
