# 实际开发中用到过哪些数据结构和算法

在实际Java开发中，我经常使用以下数据结构和算法：

## 常用数据结构

**集合框架相关：**

- **ArrayList/LinkedList**: 业务数据列表处理，如用户列表、订单列表等
- **HashMap/ConcurrentHashMap**: 缓存实现、配置管理、ID映射关系
- **HashSet**: 去重操作，如用户权限集合、标签去重
- **TreeMap/TreeSet**: 需要排序的场景，如时间范围查询、优先级处理
- **Queue/PriorityQueue**: 任务队列、消息队列的本地实现

**实际应用场景：**

```java
// 缓存用户信息
private Map<Long, User> userCache = new ConcurrentHashMap<>();

// 任务优先级队列
private PriorityQueue<Task> taskQueue = new PriorityQueue<>(
    Comparator.comparing(Task::getPriority)
);
```

## 常用算法

**排序算法：**
- **快速排序/归并排序**: 大数据量排序，通过Collections.sort()间接使用
- **自定义排序**: 业务对象按多字段排序
**查找算法：**
- **二分查找**: 有序数据的快速检索
- **哈希查找**: HashMap底层，O(1)查找效率
**字符串算法：**
- **KMP算法**: 文本搜索功能
- **正则表达式**: 数据验证、日志分析

## 具体开发场景

**1. 电商系统中的应用：**

```java
// 购物车 - 使用HashMap存储商品和数量
Map<Long, Integer> cart = new HashMap<>();

// 商品推荐 - 使用优先队列按评分排序
PriorityQueue<Product> recommendations = new PriorityQueue<>(
    (a, b) -> Double.compare(b.getScore(), a.getScore())
);
```

**2. 缓存系统：**

- **LRU算法**: 使用LinkedHashMap实现最近最少使用缓存
- **布隆过滤器**: 防止缓存穿透

**3. 搜索功能：**

- **Trie树**: 实现自动补全、敏感词过滤
- **倒排索引**: 全文搜索的底层实现

**4. 并发场景：**

- **ConcurrentHashMap**: 线程安全的缓存
- **BlockingQueue**: 生产者-消费者模式
- **AtomicInteger**: 计数器、ID生成

## 性能优化相关

**时间复杂度考虑：**

- 大数据量查询优先考虑HashMap（O(1)）而非ArrayList遍历（O(n)）
- 有序数据使用TreeMap而非每次排序
- 频繁插入删除选择LinkedList而非ArrayList

**空间复杂度权衡：**

- 使用对象池减少GC压力
- 合理选择初始容量避免频繁扩容

**实际项目中的算法应用：**

- **分布式一致性算法**: 如Raft算法在微服务注册中心
- **负载均衡算法**: 轮询、加权轮询、一致性哈希
- **限流算法**: 令牌桶、滑动窗口

这些数据结构和算法的选择通常基于具体的业务需求、数据规模和性能要求，在实际开发中需要根据场景特点做出最优选择。
