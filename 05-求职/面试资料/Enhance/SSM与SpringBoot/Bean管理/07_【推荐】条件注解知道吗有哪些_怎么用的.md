# 条件注解知道吗？有哪些，怎么用的

条件注解是Spring Framework中的一个重要特性，用于根据特定条件来决定是否创建Bean或启用某个配置。让我详细介绍一下：

## 什么是条件注解

条件注解允许我们在满足特定条件时才创建Bean或启用配置类，这为应用程序提供了更灵活的配置能力。

## 常用的条件注解

### 1. @ConditionalOnProperty

根据配置属性的值来决定是否创建Bean：

```java
@Component
@ConditionalOnProperty(name = "feature.enabled", havingValue = "true")
public class FeatureService {
    // 只有当 feature.enabled=true 时才创建此Bean
}

@Configuration
@ConditionalOnProperty(
    prefix = "cache",
    name = "type",
    havingValue = "redis",
    matchIfMissing = false
)
public class RedisConfig {
    // 只有当 cache.type=redis 时才生效
}
```

### 2. @ConditionalOnClass / @ConditionalOnMissingClass

根据类路径中是否存在某个类来决定：

```java
@Configuration
@ConditionalOnClass(RedisTemplate.class)
public class RedisAutoConfiguration {
    // 只有当Redis相关类存在时才生效
}

@Component
@ConditionalOnMissingClass("com.example.OtherService")
public class DefaultService {
    // 只有当指定类不存在时才创建
}
```

### 3. @ConditionalOnBean / @ConditionalOnMissingBean

根据容器中是否存在某个Bean来决定：

```java
@Bean
@ConditionalOnBean(DataSource.class)
public class DatabaseService {
    // 只有当DataSource Bean存在时才创建
}

@Bean
@ConditionalOnMissingBean(name = "customService")
public class DefaultCustomService {
    // 只有当名为customService的Bean不存在时才创建
}
```

### 4. @ConditionalOnProfile

根据激活的Profile来决定：

```java
@Component
@ConditionalOnProfile("dev")
public class DevService {
    // 只在dev环境下创建
}

@Configuration
@ConditionalOnProfile({"prod", "staging"})
public class ProductionConfig {
    // 在prod或staging环境下生效
}
```

### 5. @ConditionalOnResource

根据资源文件是否存在来决定：

```java
@Configuration
@ConditionalOnResource(resources = "classpath:config/custom.properties")
public class CustomConfig {
    // 只有当指定资源文件存在时才生效
}
```

### 6. @ConditionalOnExpression

根据SpEL表达式的结果来决定：

```java
@Bean
@ConditionalOnExpression("${feature.advanced:false} and ${feature.experimental:false}")
public class AdvancedFeatureService {
    // 根据复杂表达式条件创建
}
```

## 自定义条件注解

你也可以创建自定义的条件注解：

```java
// 1. 创建条件实现类
public class LinuxCondition implements Condition {
    @Override
    public boolean matches(ConditionContext context, AnnotatedTypeMetadata metadata) {
        return System.getProperty("os.name").toLowerCase().contains("linux");
    }
}

// 2. 使用@Conditional注解
@Component
@Conditional(LinuxCondition.class)
public class LinuxSpecificService {
    // 只在Linux系统下创建
}

// 3. 创建自定义条件注解
@Target({ElementType.TYPE, ElementType.METHOD})
@Retention(RetentionPolicy.RUNTIME)
@Conditional(LinuxCondition.class)
public @interface ConditionalOnLinux {
}

// 4. 使用自定义注解
@Component
@ConditionalOnLinux
public class AnotherLinuxService {
    // 使用自定义注解
}
```

## 实际应用场景

### 1. 数据库配置选择

```java
@Configuration
@ConditionalOnProperty(name = "database.type", havingValue = "mysql")
public class MySQLConfig {
    @Bean
    public DataSource mysqlDataSource() {
        // MySQL数据源配置
    }
}

@Configuration
@ConditionalOnProperty(name = "database.type", havingValue = "oracle")
public class OracleConfig {
    @Bean
    public DataSource oracleDataSource() {
        // Oracle数据源配置
    }
}
```

### 2. 缓存策略选择

```java
@Service
@ConditionalOnProperty(name = "cache.provider", havingValue = "redis")
public class RedisCacheService implements CacheService {
    // Redis缓存实现
}

@Service
@ConditionalOnProperty(name = "cache.provider", havingValue = "memory")
public class MemoryCacheService implements CacheService {
    // 内存缓存实现
}
```

## 注意事项

1. **条件注解的执行时机**：在Bean实例化之前进行条件判断
2. **性能考虑**：条件判断在应用启动时执行，不会影响运行时性能
3. **组合使用**：可以在同一个类或方法上使用多个条件注解，需要同时满足所有条件
4. **默认值处理**：合理使用`matchIfMissing`参数处理属性不存在的情况

条件注解是Spring Boot自动配置的核心机制，掌握它们能够帮助你构建更灵活、更易维护的应用程序。
