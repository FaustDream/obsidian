# 项目中使用到了哪些注解，哪些自定义注解

在Java项目开发中，注解的使用非常广泛。我来详细介绍一下常用的注解和自定义注解的实现。

## 常用的框架注解

### Spring框架注解

- **@Component, @Service, @Repository, @Controller** - 声明Spring管理的Bean
- **@Autowired** - 自动依赖注入
- **@Value** - 注入配置文件中的值
- **@Configuration** - 标识配置类
- **@Bean** - 声明方法返回的对象为Spring Bean

### Spring Boot注解

- **@SpringBootApplication** - 启动类注解，包含@Configuration、@EnableAutoConfiguration、@ComponentScan
- **@ConfigurationProperties** - 绑定配置文件属性到Java对象
- **@ConditionalOnProperty** - 基于配置属性的条件装配

### Spring MVC注解

- **@RestController** - RESTful控制器，相当于@Controller + @ResponseBody
- **@RequestMapping, @GetMapping, @PostMapping** - 映射HTTP请求
- **@RequestParam, @PathVariable, @RequestBody** - 处理请求参数
- **@Valid** - 参数校验

### JPA/Hibernate注解

- **@Entity, @Table** - 实体类和表映射
- **@Id, @GeneratedValue** - 主键配置
- **@Column** - 字段映射
- **@OneToMany, @ManyToOne** - 关联关系

### 数据校验注解

- **@NotNull, @NotEmpty, @NotBlank** - 非空校验
- **@Size, @Min, @Max** - 长度和数值范围校验
- **@Pattern** - 正则表达式校验
- **@Email** - 邮箱格式校验

## 自定义注解的实现

### 1. 参数校验注解

```java
// 自定义手机号校验注解
@Target(ElementType.FIELD)
@Retention(RetentionPolicy.RUNTIME)
@Constraint(validatedBy = PhoneValidator.class)
public @interface Phone {
    String message() default "手机号格式不正确";
    Class<?>[] groups() default {};
    Class<? extends Payload>[] payload() default {};
}

// 校验器实现
public class PhoneValidator implements ConstraintValidator<Phone, String> {
    @Override
    public boolean isValid(String value, ConstraintValidatorContext context) {
        if (value == null || value.isEmpty()) {
            return true;
        }
        return value.matches("^1[3-9]\\d{9}$");
    }
}
```

### 2. 缓存注解

```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Cache {
    String key() default "";
    int expireTime() default 300; // 秒
    String prefix() default "cache:";
}

// AOP切面处理
@Aspect
@Component
public class CacheAspect {
    @Around("@annotation(cache)")
    public Object around(ProceedingJoinPoint joinPoint, Cache cache) throws Throwable {
        String key = generateKey(joinPoint, cache);
        
        // 先查缓存
        Object result = redisTemplate.opsForValue().get(key);
        if (result != null) {
            return result;
        }
        
        // 执行方法
        result = joinPoint.proceed();
        
        // 存入缓存
        redisTemplate.opsForValue().set(key, result, cache.expireTime(), TimeUnit.SECONDS);
        
        return result;
    }
}
```

### 3. 日志注解

```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface OperationLog {
    String operation() default "";
    String module() default "";
    boolean logParams() default true;
    boolean logResult() default false;
}

@Aspect
@Component
public class OperationLogAspect {
    @Around("@annotation(operationLog)")
    public Object around(ProceedingJoinPoint joinPoint, OperationLog operationLog) throws Throwable {
        long startTime = System.currentTimeMillis();
        
        try {
            Object result = joinPoint.proceed();
            
            // 记录成功日志
            logOperation(joinPoint, operationLog, startTime, result, null);
            
            return result;
        } catch (Exception e) {
            // 记录异常日志
            logOperation(joinPoint, operationLog, startTime, null, e);
            throw e;
        }
    }
}
```

### 4. 权限控制注解

```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface RequirePermission {
    String[] value() default {};
    LogicalOperator logical() default LogicalOperator.AND;
    
    enum LogicalOperator {
        AND, OR
    }
}

@Aspect
@Component
public class PermissionAspect {
    @Before("@annotation(requirePermission)")
    public void checkPermission(JoinPoint joinPoint, RequirePermission requirePermission) {
        String[] permissions = requirePermission.value();
        
        // 获取当前用户
        UserDetails currentUser = getCurrentUser();
        
        // 检查权限
        boolean hasPermission = checkUserPermissions(currentUser, permissions, requirePermission.logical());
        
        if (!hasPermission) {
            throw new AccessDeniedException("权限不足");
        }
    }
}
```

### 5. 限流注解

```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface RateLimit {
    int count() default 10;
    int time() default 60;
    String key() default "";
}

@Aspect
@Component
public class RateLimitAspect {
    @Before("@annotation(rateLimit)")
    public void checkRateLimit(JoinPoint joinPoint, RateLimit rateLimit) {
        String key = generateRateLimitKey(joinPoint, rateLimit);
        
        // 使用Redis实现滑动窗口限流
        long current = System.currentTimeMillis();
        long window = rateLimit.time() * 1000;
        
        // 清除过期的记录
        redisTemplate.opsForZSet().removeRangeByScore(key, 0, current - window);
        
        // 检查当前请求数
        long count = redisTemplate.opsForZSet().count(key, current - window, current);
        
        if (count >= rateLimit.count()) {
            throw new RateLimitException("请求过于频繁，请稍后再试");
        }
        
        // 记录当前请求
        redisTemplate.opsForZSet().add(key, UUID.randomUUID().toString(), current);
        redisTemplate.expire(key, rateLimit.time(), TimeUnit.SECONDS);
    }
}
```

## 自定义注解的使用场景

### 1. 业务逻辑增强

- 数据校验
- 权限控制
- 操作日志
- 缓存管理

### 2. 横切关注点

- 性能监控
- 异常处理
- 事务管理
- 限流熔断

### 3. 代码简化

- 减少重复代码
- 提高代码可读性
- 统一处理逻辑

## 注解的最佳实践

### 1. 设计原则

- 职责单一，功能明确
- 参数设计合理，有默认值
- 文档注释清晰完整

### 2. 性能考虑

- 避免在高频方法上使用复杂的注解处理
- 合理使用缓存减少重复计算
- 注意AOP切面的执行顺序

### 3. 错误处理

- 提供清晰的错误信息
- 合理的异常处理机制
- 支持降级策略

这些注解的使用能够大大提升开发效率，使代码更加简洁和可维护。在实际项目中，根据具体需求选择合适的注解，并结合AOP技术实现横切关注点的统一处理。
