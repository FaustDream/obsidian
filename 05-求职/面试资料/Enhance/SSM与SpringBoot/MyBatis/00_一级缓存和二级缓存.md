## 一级缓存（SqlSession级别缓存）

**特点：**

- 默认开启，无法关闭
- 作用域是SqlSession，同一个SqlSession内有效
- 基于HashMap实现，存储在内存中

**工作原理：**

```java
// 同一个SqlSession内，相同查询只会执行一次SQL
SqlSession session = sqlSessionFactory.openSession();
UserMapper mapper = session.getMapper(UserMapper.class);

User user1 = mapper.selectById(1); // 执行SQL查询
User user2 = mapper.selectById(1); // 直接从一级缓存获取，不执行SQL
```

**缓存失效场景：**

- SqlSession关闭
- 执行增删改操作
- 手动调用`clearCache()`
- 查询参数不同

## 二级缓存（Mapper级别缓存）

**特点：**

- 需要手动开启配置
- 作用域是Mapper(Namespace)，多个SqlSession之间共享
- 支持多种缓存实现（如Redis、Ehcache等）

**开启方式：**

```xml
<!-- 1. 在mybatis-config.xml中开启 -->
<settings>
    <setting name="cacheEnabled" value="true"/>
</settings>

<!-- 2. 在Mapper.xml中配置 -->
<mapper namespace="com.example.UserMapper">
    <cache eviction="LRU" 
           flushInterval="60000" 
           size="1024" 
           readOnly="true"/>
</mapper>
```

**配置参数说明：**

- `eviction`: 缓存淘汰策略（LRU、FIFO、SOFT、WEAK）
- `flushInterval`: 缓存刷新间隔（毫秒）
- `size`: 缓存大小
- `readOnly`: 是否只读

## 缓存执行顺序

1. 先查询二级缓存（范围更大）
2. 二级缓存没有，查询一级缓存
3. 一级缓存没有，查询数据库
4. 查询结果依次填充一级缓存和二级缓存

## 实际应用注意事项

**优势：**

- 减少数据库访问次数
- 提升查询性能
- 降低数据库压力

**潜在问题：**

```java
// 脏读问题示例
SqlSession session1 = factory.openSession();
SqlSession session2 = factory.openSession();

// session1查询用户（缓存到二级缓存）
User user1 = session1.getMapper(UserMapper.class).selectById(1);
session1.close();

// session2直接修改数据库，但没有清理二级缓存
session2.getMapper(UserMapper.class).updateUser(new User(1, "新名称"));
session2.commit();
session2.close();

// session3从二级缓存读取到过期数据
SqlSession session3 = factory.openSession();
User user3 = session3.getMapper(UserMapper.class).selectById(1); // 可能读到旧数据
```

**最佳实践建议：**

1. 一级缓存一般保持默认开启
2. 二级缓存需谨慎使用，特别是在分布式环境
3. 对于频繁更新的数据，建议关闭二级缓存
4. 可以考虑使用Redis等外部缓存替代MyBatis二级缓存
5. 在微服务架构中，更推荐使用分布式缓存解决方案

这样的缓存机制设计体现了MyBatis在性能优化方面的考虑，但在实际项目中需要根据具体业务场景来权衡使用。
