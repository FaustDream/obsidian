# 线程的生命周期

线程的生命周期是Java多线程编程中的核心概念，理解它对于编写高效、稳定的并发程序至关重要。

## 线程的六个状态

Java中线程的生命周期可以分为以下六个状态：

### 1. NEW（新建状态）

- 线程对象已创建但尚未调用start()方法
- 此时线程还没有开始执行，只是一个普通的Java对象

```java
Thread thread = new Thread(() -> System.out.println("Hello"));
// 此时线程处于NEW状态
```

### 2. RUNNABLE（就绪/运行状态）

- 调用start()方法后，线程进入RUNNABLE状态
- 包含两个子状态：
    - **Ready（就绪）**：线程已准备好运行，等待CPU调度
    - **Running（运行）**：线程正在CPU上执行
- 在Java中，这两个状态统称为RUNNABLE

### 3. BLOCKED（阻塞状态）

- 线程因为等待监视器锁（synchronized）而被阻塞
- 当线程尝试获取一个已被其他线程持有的对象锁时进入此状态

```java
synchronized(obj) {
    // 如果obj的锁被其他线程持有，当前线程会进入BLOCKED状态
}
```

### 4. WAITING（等待状态）

- 线程无限期等待另一个线程执行特定操作
- 通过以下方法进入WAITING状态：
    - `Object.wait()`（不带超时）
    - `Thread.join()`（不带超时）
    - `LockSupport.park()`

### 5. TIMED_WAITING（超时等待状态）

- 线程在指定时间内等待另一个线程执行特定操作
- 通过以下方法进入TIMED_WAITING状态：
    - `Thread.sleep(long millis)`
    - `Object.wait(long timeout)`
    - `Thread.join(long millis)`
    - `LockSupport.parkNanos()`
    - `LockSupport.parkUntil()`

### 6. TERMINATED（终止状态）

- 线程执行完毕或因异常而终止
- 一旦进入此状态，线程不能再重新启动

## 状态转换图

```
NEW --> RUNNABLE --> TERMINATED
  |         |
  |         v
  |    BLOCKED/WAITING/TIMED_WAITING
  |         |
  |         v
  |    RUNNABLE
```

## 实际代码示例

```java
public class ThreadLifecycleDemo {
    private static final Object lock = new Object();
    
    public static void main(String[] args) throws InterruptedException {
        Thread thread = new Thread(() -> {
            try {
                // 模拟运行状态
                Thread.sleep(1000); // TIMED_WAITING
                
                synchronized (lock) {
                    lock.wait(); // WAITING
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        });
        
        System.out.println("1. " + thread.getState()); // NEW
        
        thread.start();
        System.out.println("2. " + thread.getState()); // RUNNABLE
        
        Thread.sleep(500);
        System.out.println("3. " + thread.getState()); // TIMED_WAITING
        
        Thread.sleep(1000);
        System.out.println("4. " + thread.getState()); // WAITING
        
        synchronized (lock) {
            lock.notify(); // 唤醒等待线程
        }
        
        thread.join();
        System.out.println("5. " + thread.getState()); // TERMINATED
    }
}
```

## 相关知识拓展

### 1. 线程状态监控

- 使用`Thread.getState()`获取线程当前状态
- 使用JVM监控工具（如jstack、jconsole）查看线程状态
- 通过ThreadMXBean进行程序化监控

### 2. 状态转换的关键点

- **不可逆性**：线程一旦TERMINATED就不能重新启动
- **并发性**：多个线程的状态转换是并发进行的
- **同步机制**：synchronized、wait/notify、join等会影响状态转换

### 3. 性能优化建议

- 避免长时间持有锁导致其他线程BLOCKED
- 合理使用超时机制避免无限WAITING
- 及时释放资源避免线程泄漏

### 4. 常见问题排查

- **死锁**：多个线程互相等待对方持有的锁
- **活锁**：线程不断改变状态但无法推进
- **饥饿**：某些线程长期无法获得CPU时间

理解线程生命周期有助于编写更高效的多线程程序，也是解决并发问题的基础。
