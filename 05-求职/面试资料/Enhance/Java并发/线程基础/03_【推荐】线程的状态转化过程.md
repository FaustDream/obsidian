# 线程的状态转化过程

Java线程的状态转化是一个重要的概念，让我详细解释一下整个过程。

## Java线程的6种状态

根据`Thread.State`枚举，Java线程有以下6种状态：

1. **NEW（新建）** - 线程被创建但未调用start()方法
2. **RUNNABLE（可运行）** - 线程正在运行或准备运行
3. **BLOCKED（阻塞）** - 线程被阻塞等待监视器锁
4. **WAITING（等待）** - 线程无限期等待另一个线程执行特定操作
5. **TIMED_WAITING（超时等待）** - 线程等待指定时间
6. **TERMINATED（终止）** - 线程执行完毕

## 状态转换过程

### 1. NEW → RUNNABLE

```java
Thread thread = new Thread(() -> {
    // 线程逻辑
});
// 此时线程处于NEW状态
thread.start(); // 调用start()方法，转换为RUNNABLE
```

### 2. RUNNABLE → BLOCKED

当线程尝试获取已被其他线程持有的synchronized锁时：

```java
synchronized (lockObject) {
    // 如果锁被其他线程持有，当前线程进入BLOCKED状态
}
```

### 3. RUNNABLE → WAITING

通过以下方式可以进入WAITING状态：

```java
// 方式1：Object.wait()
synchronized (obj) {
    obj.wait(); // 进入WAITING状态
}

// 方式2：Thread.join()
thread.join(); // 等待thread线程结束

// 方式3：LockSupport.park()
LockSupport.park(); // 进入WAITING状态
```

### 4. RUNNABLE → TIMED_WAITING

通过以下方式进入超时等待：

```java
// 方式1：Thread.sleep()
Thread.sleep(1000); // 超时等待1秒

// 方式2：Object.wait(timeout)
synchronized (obj) {
    obj.wait(1000); // 超时等待1秒
}

// 方式3：Thread.join(timeout)
thread.join(1000); // 等待thread线程结束，最多等1秒
```

### 5. 各种状态 → RUNNABLE

- **BLOCKED → RUNNABLE**: 获得锁后自动转换
- **WAITING → RUNNABLE**: 通过notify()/notifyAll()、join的线程结束、LockSupport.unpark()等
- **TIMED_WAITING → RUNNABLE**: 超时时间到或满足唤醒条件

### 6. RUNNABLE → TERMINATED

线程正常执行完毕或因异常终止。

## 状态转换图示

```

 │                  │                            │

 │                  │

```

## 实际代码示例

```java
public class ThreadStateDemo {
    private static final Object lock = new Object();
    
    public static void main(String[] args) throws InterruptedException {
        Thread thread = new Thread(() -> {
            try {
                // RUNNABLE状态
                synchronized (lock) {
                    System.out.println("获得锁，执行业务逻辑");
                    Thread.sleep(2000); // TIMED_WAITING状态
                }
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        });
        
        System.out.println("线程创建后状态: " + thread.getState()); // NEW
        
        thread.start();
        Thread.sleep(100);
        System.out.println("启动后状态: " + thread.getState()); // RUNNABLE
        
        thread.join(); // 等待线程结束
        System.out.println("线程结束后状态: " + thread.getState()); // TERMINATED
    }
}
```

## 重要注意事项

1. **线程状态不可逆**：TERMINATED状态的线程无法重新启动
2. **RUNNABLE包含两种情况**：正在CPU上运行和等待CPU调度
3. **状态转换的原子性**：状态转换是原子操作，不会出现中间状态
4. **死锁检测**：JVM可以检测BLOCKED状态的线程是否形成死锁

## 扩展知识

### 与操作系统线程状态的区别

操作系统通常有更多线程状态（如就绪、运行、挂起等），而Java将其简化为6种状态，其中RUNNABLE状态实际包含了操作系统的就绪和运行状态。

### 监控工具

可以使用jstack、jconsole、VisualVM等工具监控线程状态，这对于性能调优和问题排查非常有帮助。

理解线程状态转换有助于编写更高效的并发程序，避免死锁和资源争用问题。
