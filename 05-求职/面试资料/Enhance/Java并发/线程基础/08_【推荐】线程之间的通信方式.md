# 线程之间的通信方式

线程间通信是Java并发编程中的核心概念。以下是主要的通信方式：

## 1. 共享内存方式

### volatile关键字

- **作用**：保证变量的可见性和有序性
- **使用场景**：适用于一个线程写，多个线程读的场景

```java
private volatile boolean flag = false;
```

### synchronized同步块/方法

- **作用**：提供互斥访问和内存可见性
- **原理**：通过监视器锁实现线程同步

```java
synchronized(object) {
    // 临界区代码
}
```

## 2. 等待/通知机制

### wait()/notify()/notifyAll()

- **使用前提**：必须在synchronized块中使用
- **典型应用**：生产者-消费者模式

```java
synchronized(obj) {
    while(condition) {
        obj.wait(); // 释放锁并等待
    }
    obj.notify(); // 唤醒等待的线程
}
```

## 3. 并发工具类

### CountDownLatch

- **用途**：让一个或多个线程等待其他线程完成操作
- **应用场景**：主线程等待多个子线程完成初始化

### CyclicBarrier

- **用途**：让一组线程互相等待到达某个屏障点
- **应用场景**：多线程计算后汇总结果

### Semaphore

- **用途**：控制同时访问特定资源的线程数量
- **应用场景**：限制并发访问数

## 4. 阻塞队列

### BlockingQueue系列

- **ArrayBlockingQueue**：基于数组的有界阻塞队列
- **LinkedBlockingQueue**：基于链表的可选有界阻塞队列
- **SynchronousQueue**：不存储元素的阻塞队列

## 5. 管道通信

### PipedInputStream/PipedOutputStream

- **特点**：用于线程间的字节流传输
- **使用场景**：一个线程发送数据，另一个线程接收数据

## 6. 高级并发工具

### CompletableFuture

- **优势**：支持异步编程和链式调用
- **应用**：复杂的异步任务编排

### Exchanger

- **用途**：两个线程间交换数据
- **特点**：同步点，双方都到达才能交换

## 性能考虑

**无锁 > 偏向锁 > 轻量级锁 > 重量级锁**

- **Lock接口**：相比synchronized提供更灵活的锁操作
- **原子类**：如AtomicInteger，基于CAS操作，性能更好
- **ThreadLocal**：线程本地存储，避免同步开销

## 实际应用建议

1. **简单场景**：使用volatile或synchronized
2. **复杂协调**：选择合适的并发工具类
3. **高性能要求**：考虑无锁编程（CAS、原子类）
4. **异步处理**：使用CompletableFuture或消息队列

**核心原则**：根据具体的并发场景选择合适的通信方式，避免过度设计，同时要考虑性能和可维护性的平衡。
