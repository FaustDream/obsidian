# 死锁产生四大条件

死锁产生需要同时满足四个条件，这四个条件缺一不可：

## 1. 互斥条件（Mutual Exclusion）

资源在同一时间只能被一个进程使用。比如打印机、文件锁等，当一个进程正在使用时，其他进程必须等待。

**Java例子：**

```java
synchronized (lockA) {
    // 只有一个线程能进入这个代码块
    // 其他线程必须等待
}
```

## 2. 请求与保持条件（Hold and Wait）

进程已经获得了至少一个资源，但又请求新的资源，而该资源被其他进程占用，所以当前进程会阻塞，但不会释放自己已经获得的资源。

**Java例子：**

```java
synchronized (lockA) {
    // 已经持有lockA
    synchronized (lockB) {
        // 想要获取lockB，但可能被其他线程持有
        // 这时不会释放lockA
    }
}
```

## 3. 不可剥夺条件（No Preemption）

进程已获得的资源，在未使用完之前，不能被其他进程强行剥夺，只能由获得该资源的进程主动释放。

**Java例子：**

```java
// synchronized关键字获得的锁不能被其他线程强制释放
// 只能等待当前线程执行完毕后自动释放
```

## 4. 循环等待条件（Circular Wait）

存在一种进程资源的循环等待链，链中每个进程都在等待下一个进程所占有的资源。

**Java例子：**

```java
// 线程1
synchronized (lockA) {
    synchronized (lockB) { // 等待lockB
        // 业务逻辑
    }
}

// 线程2
synchronized (lockB) {
    synchronized (lockA) { // 等待lockA
        // 业务逻辑  
    }
}
// 形成A等B，B等A的循环等待
```

## 完整的死锁示例

```java
public class DeadlockExample {
    private final Object lockA = new Object();
    private final Object lockB = new Object();
    
    public void method1() {
        synchronized (lockA) {
            System.out.println("Thread1 acquired lockA");
            try { Thread.sleep(100); } catch (InterruptedException e) {}
            
            synchronized (lockB) {
                System.out.println("Thread1 acquired lockB");
            }
        }
    }
    
    public void method2() {
        synchronized (lockB) {
            System.out.println("Thread2 acquired lockB");
            try { Thread.sleep(100); } catch (InterruptedException e) {}
            
            synchronized (lockA) {
                System.out.println("Thread2 acquired lockA");
            }
        }
    }
}
```

## 破解死锁的方法

只需要破坏四个条件中的任意一个即可：

1. **破坏互斥条件**：使用无锁数据结构（如原子类）
2. **破坏请求与保持条件**：一次性申请所有资源
3. **破坏不可剥夺条件**：使用tryLock()设置超时
4. **破坏循环等待条件**：按固定顺序申请资源

**预防死锁的最佳实践：**

```java
// 使用tryLock避免死锁
if (lockA.tryLock()) {
    try {
        if (lockB.tryLock()) {
            try {
                // 业务逻辑
            } finally {
                lockB.unlock();
            }
        }
    } finally {
        lockA.unlock();
    }
}
```

理解这四个条件是解决并发编程中死锁问题的基础，在实际开发中应该时刻注意避免这些条件同时出现。
