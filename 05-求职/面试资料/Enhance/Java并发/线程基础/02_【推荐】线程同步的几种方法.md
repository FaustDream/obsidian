# 线程同步的几种方法

线程同步是Java并发编程中的核心概念，用于协调多个线程对共享资源的访问。以下是几种主要的线程同步方法：

## 1. synchronized关键字

### 同步方法

```java
public synchronized void method() {
    // 同步代码
}
```

### 同步代码块

```java
public void method() {
    synchronized(this) {
        // 同步代码
    }
}
```

**特点：**

- 基于对象监视器（Monitor）实现
- 具有可重入性
- 自动释放锁

## 2. ReentrantLock（可重入锁）

```java
private final ReentrantLock lock = new ReentrantLock();

public void method() {
    lock.lock();
    try {
        // 同步代码
    } finally {
        lock.unlock();
    }
}
```

**优势：**

- 可中断锁等待
- 可设置超时时间
- 支持公平锁
- 更灵活的锁控制

## 3. volatile关键字

```java
private volatile boolean flag = false;
```

**作用：**

- 保证变量的可见性
- 禁止指令重排序
- 不能保证原子性

## 4. 原子类（Atomic Classes）

```java
private AtomicInteger counter = new AtomicInteger(0);

public void increment() {
    counter.incrementAndGet();
}
```

**常用原子类：**

- AtomicInteger、AtomicLong
- AtomicBoolean
- AtomicReference

## 5. 读写锁（ReadWriteLock）

```java
private final ReadWriteLock rwLock = new ReentrantReadWriteLock();
private final Lock readLock = rwLock.readLock();
private final Lock writeLock = rwLock.writeLock();

public void read() {
    readLock.lock();
    try {
        // 读操作
    } finally {
        readLock.unlock();
    }
}

public void write() {
    writeLock.lock();
    try {
        // 写操作
    } finally {
        writeLock.unlock();
    }
}
```

## 6. Semaphore（信号量）

```java
private final Semaphore semaphore = new Semaphore(3);

public void method() throws InterruptedException {
    semaphore.acquire();
    try {
        // 限制并发数的代码
    } finally {
        semaphore.release();
    }
}
```

## 7. CountDownLatch（倒计数器）

```java
private final CountDownLatch latch = new CountDownLatch(3);

// 等待线程
public void await() throws InterruptedException {
    latch.await();
}

// 工作线程
public void work() {
    try {
        // 执行任务
    } finally {
        latch.countDown();
    }
}
```

## 8. CyclicBarrier（循环屏障）

```java
private final CyclicBarrier barrier = new CyclicBarrier(3, () -> {
    System.out.println("所有线程都到达屏障点");
});

public void method() throws InterruptedException, BrokenBarrierException {
    // 执行任务
    barrier.await(); // 等待其他线程
}
```

## 9. BlockingQueue（阻塞队列）

```java
private final BlockingQueue<String> queue = new ArrayBlockingQueue<>(10);

// 生产者
public void produce(String item) throws InterruptedException {
    queue.put(item);
}

// 消费者
public String consume() throws InterruptedException {
    return queue.take();
}
```

## 选择建议

1. **synchronized**：适合简单的同步场景，性能较好
2. **ReentrantLock**：需要更高级功能时选择
3. **volatile**：仅需要可见性保证时使用
4. **原子类**：简单的原子操作
5. **ReadWriteLock**：读多写少的场景
6. **并发工具类**：复杂的协调场景

## 性能考虑

- synchronized在JDK 1.6后进行了大量优化（偏向锁、轻量级锁、自旋锁）
- ReentrantLock在高竞争情况下性能更好
- volatile是最轻量级的同步机制
- 原子类适合高并发的简单操作

选择合适的同步机制需要根据具体场景、性能要求和代码复杂度来决定。
