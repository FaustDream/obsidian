# 创建线程的几种方式

在Java中创建线程主要有以下几种方式：

## 1. 继承Thread类

```java
class MyThread extends Thread {
    @Override
    public void run() {
        System.out.println("线程执行中: " + Thread.currentThread().getName());
    }
}

// 使用方式
MyThread thread = new MyThread();
thread.start();
```

**特点：**

- 简单直接，直接重写run()方法
- 由于Java单继承限制，无法再继承其他类
- 适合简单的线程任务

## 2. 实现Runnable接口

```java
class MyRunnable implements Runnable {
    @Override
    public void run() {
        System.out.println("线程执行中: " + Thread.currentThread().getName());
    }
}

// 使用方式
MyRunnable runnable = new MyRunnable();
Thread thread = new Thread(runnable);
thread.start();

// 或者使用Lambda表达式
Thread thread2 = new Thread(() -> {
    System.out.println("Lambda线程执行中");
});
thread2.start();
```

**特点：**

- 推荐方式，符合面向接口编程原则
- 可以继承其他类，灵活性更高
- 实现了任务与线程的分离

## 3. 实现Callable接口

```java
class MyCallable implements Callable<String> {
    @Override
    public String call() throws Exception {
        Thread.sleep(1000);
        return "任务完成，返回结果";
    }
}

// 使用方式
MyCallable callable = new MyCallable();
FutureTask<String> futureTask = new FutureTask<>(callable);
Thread thread = new Thread(futureTask);
thread.start();

// 获取返回结果
try {
    String result = futureTask.get(); // 阻塞等待结果
    System.out.println(result);
} catch (Exception e) {
    e.printStackTrace();
}
```

**特点：**

- 可以有返回值
- 可以抛出异常
- 通过FutureTask包装使用

## 4. 使用线程池

```java
// 创建线程池
ExecutorService executor = Executors.newFixedThreadPool(3);

// 提交Runnable任务
executor.submit(() -> {
    System.out.println("线程池执行任务");
});

// 提交Callable任务
Future<String> future = executor.submit(() -> {
    return "线程池返回结果";
});

// 获取结果
try {
    String result = future.get();
    System.out.println(result);
} catch (Exception e) {
    e.printStackTrace();
}

// 关闭线程池
executor.shutdown();
```

**特点：**

- 生产环境推荐方式
- 可以重复利用线程，避免频繁创建销毁
- 提供多种线程池类型

## 5. 使用CompletableFuture

```java
// 异步执行任务
CompletableFuture<String> future = CompletableFuture.supplyAsync(() -> {
    try {
        Thread.sleep(1000);
    } catch (InterruptedException e) {
        e.printStackTrace();
    }
    return "异步任务完成";
});

// 处理结果
future.thenAccept(result -> {
    System.out.println(result);
});

// 或者阻塞获取结果
try {
    String result = future.get();
    System.out.println(result);
} catch (Exception e) {
    e.printStackTrace();
}
```

**特点：**

- Java 8引入的异步编程工具
- 支持链式调用和组合操作
- 更加灵活和强大

## 各种方式的对比

| 方式                | 优点        | 缺点    | 适用场景      |
| ----------------- | --------- | ----- | --------- |
| 继承Thread          | 简单直接      | 单继承限制 | 简单任务      |
| 实现Runnable        | 灵活性高，任务分离 | 无返回值  | 一般任务      |
| 实现Callable        | 有返回值，可抛异常 | 相对复杂  | 需要返回结果的任务 |
| 线程池               | 资源管理好，性能高 | 配置复杂  | 生产环境      |
| CompletableFuture | 异步编程，功能强大 | 学习成本高 | 复杂异步场景    |

## 拓展知识点

**线程的生命周期：**

- NEW：新建状态
- RUNNABLE：就绪/运行状态
- BLOCKED：阻塞状态
- WAITING：等待状态
- TIMED_WAITING：超时等待状态
- TERMINATED：终止状态

**线程安全考虑：**

- 共享资源需要同步机制
- 使用synchronized、Lock、volatile等
- 注意死锁、活锁等问题

**性能优化：**

- 线程池参数调优
- 避免创建过多线程
- 合理使用线程本地变量ThreadLocal

在实际开发中，推荐使用线程池的方式来管理线程，既能提高性能又便于维护。
