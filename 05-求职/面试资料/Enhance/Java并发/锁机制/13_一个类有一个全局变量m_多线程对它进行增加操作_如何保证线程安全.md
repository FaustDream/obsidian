# 一个类有一个全局变量m，多线程对它进行增加操作，如何保证线程安全

这是一个经典的多线程并发问题。当多个线程同时对共享变量进行修改时，会出现线程安全问题。让我详细解释几种解决方案：

## 问题分析

```java
public class Counter {
    private int m = 0;  // 共享变量
    
    public void increment() {
        m++;  // 非原子操作，存在线程安全问题
    }
}
```

`m++` 实际上包含三个步骤：

1. 读取m的值
2. 将值加1
3. 将结果写回m

在多线程环境下，这三个步骤可能被其他线程打断，导致数据不一致。

## 解决方案

### 1. 使用 synchronized 关键字

```java
public class Counter {
    private int m = 0;
    
    // 方法级同步
    public synchronized void increment() {
        m++;
    }
    
    // 或者使用同步代码块
    public void increment2() {
        synchronized(this) {
            m++;
        }
    }
}
```

### 2. 使用 volatile + synchronized

```java
public class Counter {
    private volatile int m = 0;  // 保证可见性
    
    public synchronized void increment() {
        m++;
    }
}
```

### 3. 使用原子类 AtomicInteger

```java
import java.util.concurrent.atomic.AtomicInteger;

public class Counter {
    private AtomicInteger m = new AtomicInteger(0);
    
    public void increment() {
        m.incrementAndGet();  // 原子操作
    }
    
    public int getValue() {
        return m.get();
    }
}
```

### 4. 使用 ReentrantLock

```java
import java.util.concurrent.locks.ReentrantLock;

public class Counter {
    private int m = 0;
    private final ReentrantLock lock = new ReentrantLock();
    
    public void increment() {
        lock.lock();
        try {
            m++;
        } finally {
            lock.unlock();
        }
    }
}
```

## 性能对比与选择建议

**性能排序**（从高到低）：

1. AtomicInteger - 无锁算法，性能最好
2. ReentrantLock - 显式锁，灵活性高
3. synchronized - JVM优化后性能不错
4. volatile + synchronized - 双重保障但开销较大

**选择建议**：

- **简单的数值操作**：优先选择 `AtomicInteger`
- **复杂的业务逻辑**：使用 `synchronized` 或 `ReentrantLock`
- **需要超时、中断等高级功能**：使用 `ReentrantLock`

## 相关知识扩展

### 线程安全的本质

线程安全问题的根源在于：

- **原子性**：操作不可分割
- **可见性**：一个线程的修改对其他线程立即可见
- **有序性**：程序执行的顺序按照代码的先后顺序执行

### 常见的线程安全问题场景

1. **读-修改-写** 操作（如 i++）
2. **检查然后执行** 操作（如单例模式）
3. **复合操作** 的原子性问题

### 实际开发中的注意事项

```java
// 错误示例 - 即使get()和set()都是同步的，组合操作仍然不是线程安全的
if (counter.get() < 100) {
    counter.increment();  // 可能有其他线程已经修改了值
}

// 正确做法 - 将整个操作同步
synchronized(counter) {
    if (counter.get() < 100) {
        counter.increment();
    }
}
```
