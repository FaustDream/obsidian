# 锁的分类方式

在Java中，锁有多种分类方式，我来详细介绍一下主要的分类维度：

## 1. 按锁的获取方式分类

### 悲观锁 vs 乐观锁

- **悲观锁**：假设会发生并发冲突，每次访问数据时都加锁
    - 典型代表：synchronized、ReentrantLock
    - 适用场景：写操作频繁的场景
- **乐观锁**：假设不会发生冲突，只在更新时检查是否有其他线程修改
    - 典型代表：CAS（Compare-And-Swap）、AtomicInteger
    - 适用场景：读操作频繁的场景

## 2. 按锁的共享性分类

### 独占锁 vs 共享锁

- **独占锁（排他锁）**：同一时刻只能有一个线程持有
    - 典型代表：synchronized、ReentrantLock
- **共享锁**：同一时刻可以有多个线程持有
    - 典型代表：ReadWriteLock的读锁、CountDownLatch

## 3. 按锁的可重入性分类

### 可重入锁 vs 不可重入锁

- **可重入锁**：同一个线程可以多次获取同一把锁
    - 典型代表：synchronized、ReentrantLock
- **不可重入锁**：同一个线程不能多次获取同一把锁
    - 需要特殊实现，Java内置锁基本都是可重入的

## 4. 按锁的阻塞性分类

### 阻塞锁 vs 非阻塞锁

- **阻塞锁**：获取不到锁时线程会阻塞等待
    - 典型代表：synchronized、ReentrantLock.lock()
- **非阻塞锁**：获取不到锁时不会阻塞，而是返回失败或继续执行
    - 典型代表：ReentrantLock.tryLock()、CAS操作

## 5. 按锁的公平性分类

### 公平锁 vs 非公平锁

- **公平锁**：按照线程请求锁的顺序分配锁
    - 典型代表：ReentrantLock(true)、ReentrantReadWriteLock(true)
- **非公平锁**：不保证按顺序分配，可能出现插队
    - 典型代表：synchronized、ReentrantLock()默认模式

## 6. 按锁的自旋特性分类

### 自旋锁 vs 非自旋锁

- **自旋锁**：获取不到锁时，线程不会立即阻塞，而是循环检查
    - 适用于锁持有时间短的场景
- **非自旋锁**：获取不到锁时直接阻塞线程

## 7. 按锁的实现机制分类

### 重量级锁 vs 轻量级锁 vs 偏向锁

这是JVM对synchronized的优化：

- **偏向锁**：锁偏向于第一个获取它的线程
- **轻量级锁**：通过CAS操作避免重量级锁的开销
- **重量级锁**：传统的互斥锁，涉及操作系统调用

## 代码示例

```java
// 1. 悲观锁示例
public synchronized void pessimisticMethod() {
    // 临界区代码
}

// 2. 乐观锁示例
AtomicInteger counter = new AtomicInteger(0);
public void optimisticIncrement() {
    counter.incrementAndGet(); // 使用CAS
}

// 3. 公平锁示例
ReentrantLock fairLock = new ReentrantLock(true);
public void fairLockMethod() {
    fairLock.lock();
    try {
        // 临界区代码
    } finally {
        fairLock.unlock();
    }
}

// 4. 读写锁示例
ReadWriteLock rwLock = new ReentrantReadWriteLock();
public void readMethod() {
    rwLock.readLock().lock();
    try {
        // 读操作
    } finally {
        rwLock.readLock().unlock();
    }
}
```

## 实际应用建议

1. **高并发读场景**：使用读写锁或乐观锁
2. **写操作频繁**：使用悲观锁
3. **需要公平性**：使用公平锁
4. **短时间持锁**：考虑自旋锁
5. **简单场景**：使用synchronized
6. **需要高级功能**：使用ReentrantLock

理解这些分类有助于在实际开发中选择合适的锁机制，提高程序的性能和可靠性。
