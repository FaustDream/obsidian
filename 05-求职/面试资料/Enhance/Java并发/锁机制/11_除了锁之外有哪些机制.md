# 除了锁之外有哪些机制

在Java并发编程中，除了传统的锁机制（如synchronized、ReentrantLock），还有多种其他的并发控制机制：

## 1. 原子操作类（Atomic Classes）

- **AtomicInteger、AtomicLong、AtomicReference**等
- 基于CAS（Compare-And-Swap）算法实现
- 无锁化操作，性能更高
- 适用于简单的原子操作场景

```java
AtomicInteger counter = new AtomicInteger(0);
counter.incrementAndGet(); // 原子性自增
```

## 2. 并发集合

- **ConcurrentHashMap**：线程安全的HashMap
- **ConcurrentLinkedQueue**：无锁的线程安全队列
- **CopyOnWriteArrayList**：读多写少场景的线程安全List
- 内部使用分段锁或CAS等技术实现

## 3. 同步工具类

- **CountDownLatch**：等待多个线程完成
- **CyclicBarrier**：多线程相互等待到达屏障点
- **Semaphore**：信号量，控制访问资源的线程数量
- **Exchanger**：两个线程间数据交换

## 4. 线程本地存储

- **ThreadLocal**：为每个线程提供独立的变量副本
- 避免共享状态，从根本上消除并发问题
- 常用于数据库连接、用户会话等场景

## 5. 无锁数据结构

- **Lock-Free算法**：使用CAS操作实现
- **Wait-Free算法**：更强的无锁保证
- 避免锁的开销和死锁问题

## 6. 阻塞队列

- **BlockingQueue**系列：LinkedBlockingQueue、ArrayBlockingQueue
- 生产者-消费者模式的天然实现
- 内部处理线程间的协调

## 7. Future和CompletableFuture

- **异步编程模式**
- 将任务提交后立即返回Future对象
- 支持链式调用和组合操作

## 8. Actor模型

- **Akka框架**（虽然主要用于Scala，但Java也可使用）
- 通过消息传递避免共享状态
- 每个Actor独立处理消息

## 9. 不可变对象

- **Immutable Objects**
- 对象创建后状态不可改变
- 天然线程安全，无需额外同步

## 10. 软件事务内存（STM）

- **Clojure STM**或第三方库
- 类似数据库事务的内存操作
- 提供ACID特性

这些机制各有优缺点，选择时需要根据具体场景考虑性能、复杂度、可维护性等因素。现代Java应用通常会组合使用多种机制来解决不同的并发问题。
