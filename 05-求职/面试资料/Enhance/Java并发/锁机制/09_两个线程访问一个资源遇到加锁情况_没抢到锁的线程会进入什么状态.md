# 两个线程访问一个资源遇到加锁情况，没抢到锁的线程会进入什么状态

当两个线程访问一个资源遇到加锁情况时，没抢到锁的线程会进入**BLOCKED（阻塞）状态**。

## 详细解释

### 线程状态转换过程：

1. **初始状态**：两个线程都处于 RUNNABLE 状态，准备获取锁
2. **竞争锁资源**：线程A成功获取锁，继续执行
3. **阻塞状态**：线程B没有获取到锁，进入 BLOCKED 状态
4. **等待唤醒**：线程B会被阻塞，直到线程A释放锁
5. **重新竞争**：锁释放后，线程B重新参与锁竞争

### 代码示例：

```java
public class ThreadLockDemo {
    private static final Object lock = new Object();
    
    public static void main(String[] args) {
        Thread thread1 = new Thread(() -> {
            synchronized (lock) {
                System.out.println("Thread1 获取到锁");
                try {
                    Thread.sleep(5000); // 模拟业务处理
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                System.out.println("Thread1 释放锁");
            }
        });
        
        Thread thread2 = new Thread(() -> {
            synchronized (lock) {
                System.out.println("Thread2 获取到锁");
            }
        });
        
        thread1.start();
        thread2.start();
        
        // 查看线程状态
        new Thread(() -> {
            try {
                Thread.sleep(1000);
                System.out.println("Thread2 状态: " + thread2.getState()); // 输出: BLOCKED
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }).start();
    }
}
```

## 扩展知识点

### 1. Java线程的6种状态

- **NEW**：新建状态，线程被创建但未启动
- **RUNNABLE**：可运行状态，包括就绪和运行中
- **BLOCKED**：阻塞状态，等待获取监视器锁
- **WAITING**：等待状态，调用wait()、join()等方法
- **TIMED_WAITING**：超时等待状态，调用sleep()、wait(timeout)等
- **TERMINATED**：终止状态，线程执行完毕

### 2. BLOCKED与WAITING的区别

- **BLOCKED**：被动等待，等待获取synchronized锁
- **WAITING**：主动等待，调用wait()、join()等方法进入

### 3. 不同锁机制的阻塞行为

```java
// synchronized - 进入BLOCKED状态
synchronized(obj) { 
    // 业务逻辑 
}

// ReentrantLock - 可中断的等待
ReentrantLock lock = new ReentrantLock();
try {
    lock.lock(); // 或 lock.lockInterruptibly()
    // 业务逻辑
} finally {
    lock.unlock();
}
```

### 4. 避免长时间阻塞的策略

- 使用tryLock()尝试获取锁
- 设置超时时间
- 优化锁的粒度，减少临界区代码
- 使用读写锁分离读写操作

这种阻塞机制是Java并发编程中保证线程安全的重要机制，确保同一时刻只有一个线程能够访问共享资源。
