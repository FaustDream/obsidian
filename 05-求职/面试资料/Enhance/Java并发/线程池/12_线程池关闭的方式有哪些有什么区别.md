# 线程池关闭的方式有哪些？有什么区别？

Java中线程池关闭主要有两种方式：`shutdown()`和`shutdownNow()`，它们的区别和特点如下：

## 1. shutdown() 方法

**特点：**

- 优雅关闭，不会立即强制终止线程
- 不再接受新任务，但会继续执行已提交的任务
- 已经在队列中等待的任务也会被执行完毕
- 调用后线程池状态变为SHUTDOWN

**使用场景：**

```java
ExecutorService executor = Executors.newFixedThreadPool(5);
// 提交一些任务
executor.submit(() -> {
    // 某些耗时任务
});

executor.shutdown(); // 优雅关闭
```

## 2. shutdownNow() 方法

**特点：**

- 立即强制关闭，尝试停止所有正在执行的任务
- 不再接受新任务
- 返回等待执行的任务列表
- 通过interrupt()方法中断正在执行的线程
- 调用后线程池状态变为STOP

**使用场景：**

```java
ExecutorService executor = Executors.newFixedThreadPool(5);
// 提交任务后需要立即停止
List<Runnable> waitingTasks = executor.shutdownNow();
System.out.println("未执行的任务数量: " + waitingTasks.size());
```

## 3. 最佳实践组合

实际开发中，通常结合使用这两种方法：

```java
public void gracefulShutdown(ExecutorService executor) {
    executor.shutdown(); // 优雅关闭
    
    try {
        // 等待60秒让任务完成
        if (!executor.awaitTermination(60, TimeUnit.SECONDS)) {
            executor.shutdownNow(); // 超时后强制关闭
            
            // 再等待60秒确认是否关闭
            if (!executor.awaitTermination(60, TimeUnit.SECONDS)) {
                System.err.println("线程池未能正常关闭");
            }
        }
    } catch (InterruptedException e) {
        executor.shutdownNow();
        Thread.currentThread().interrupt();
    }
}
```

## 4. 相关知识扩展

**线程池状态变化：**

- RUNNING → SHUTDOWN (调用shutdown())
- RUNNING → STOP (调用shutdownNow())
- SHUTDOWN → STOP (调用shutdownNow())

**注意事项：**

- shutdown()不能保证任务一定完成，如果任务中有无限循环或长时间阻塞
- shutdownNow()依赖于任务对中断的响应，如果任务不响应中断，强制关闭可能失效
- 使用awaitTermination()可以等待线程池完全关闭

这种组合方式既保证了程序的优雅性，又避免了程序hang死的风险。
