# 要自定义一个线程池，哪些参数比较重要

自定义线程池时，以下几个参数是最重要的：

## 核心参数

**1. corePoolSize（核心线程数）**

- 线程池中常驻的线程数量
- 即使空闲也不会被回收
- 建议设置为：CPU密集型任务设为CPU核数，I/O密集型任务设为CPU核数的2倍

**2. maximumPoolSize（最大线程数）**

- 线程池允许的最大线程数量
- 只有当队列满了才会创建超过核心线程数的线程
- 通常设置为核心线程数的2-4倍

**3. keepAliveTime（空闲线程存活时间）**

- 超过核心线程数的空闲线程的存活时间
- 超时后会被回收，减少资源浪费
- 一般设置为30-60秒

**4. workQueue（工作队列）**

- 存储等待执行任务的队列
- 常用类型：
    - `ArrayBlockingQueue`：有界队列，防止内存溢出
    - `LinkedBlockingQueue`：无界队列，可能导致内存问题
    - `SynchronousQueue`：直接交付，适合任务量不确定的场景

**5. threadFactory（线程工厂）**

- 创建线程的工厂，可以自定义线程名称、优先级等
- 便于调试和监控

**6. rejectedExecutionHandler（拒绝策略）**

- 当队列满且线程数达到最大值时的处理策略
- 四种默认策略：
    - `AbortPolicy`：抛异常（默认）
    - `CallerRunsPolicy`：调用者线程执行
    - `DiscardPolicy`：丢弃任务
    - `DiscardOldestPolicy`：丢弃最老的任务

## 配置建议

```java
ThreadPoolExecutor executor = new ThreadPoolExecutor(
    8,                              // corePoolSize
    16,                             // maximumPoolSize  
    60L,                            // keepAliveTime
    TimeUnit.SECONDS,               // timeUnit
    new ArrayBlockingQueue<>(1000), // workQueue
    new ThreadFactoryBuilder()      // threadFactory
        .setNameFormat("my-pool-%d")
        .build(),
    new ThreadPoolExecutor.CallerRunsPolicy() // rejectedExecutionHandler
);
```

## 扩展知识

**线程池工作原理：**

1. 任务提交时，如果当前线程数 < corePoolSize，创建新线程
2. 如果当前线程数 >= corePoolSize，任务入队列
3. 如果队列满了且当前线程数 < maximumPoolSize，创建新线程
4. 如果队列满了且当前线程数 >= maximumPoolSize，执行拒绝策略

**监控指标：**

- `getActiveCount()`：活跃线程数
- `getCompletedTaskCount()`：已完成任务数
- `getTaskCount()`：总任务数
- `getQueueSize()`：队列中任务数

合理配置这些参数可以避免线程过多导致的上下文切换开销，同时防止任务积压和内存溢出。
