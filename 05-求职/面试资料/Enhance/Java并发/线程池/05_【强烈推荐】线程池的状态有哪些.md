# 线程池的状态有哪些

Java线程池主要有以下5种状态：

## 1. RUNNING（运行状态）

- **含义**：线程池正常运行，能够接受新任务并处理队列中的任务
- **特点**：这是线程池创建后的初始状态
- **状态值**：-1（二进制：11100000000000000000000000000000）

## 2. SHUTDOWN（关闭状态）

- **含义**：线程池不再接受新任务，但会继续处理队列中已有的任务
- **触发条件**：调用`shutdown()`方法
- **特点**：优雅关闭，等待所有任务完成
- **状态值**：0（二进制：00000000000000000000000000000000）

## 3. STOP（停止状态）

- **含义**：线程池不接受新任务，也不处理队列中的任务，并且会中断正在执行的任务
- **触发条件**：调用`shutdownNow()`方法
- **特点**：强制关闭，立即停止所有活动
- **状态值**：1（二进制：00100000000000000000000000000000）

## 4. TIDYING（整理状态）

- **含义**：所有任务都已终止，工作线程数为0，即将进入TERMINATED状态
- **特点**：过渡状态，会执行`terminated()`钩子方法
- **状态值**：2（二进制：01000000000000000000000000000000）

## 5. TERMINATED（终止状态）

- **含义**：线程池完全终止，`terminated()`方法执行完毕
- **特点**：最终状态，线程池生命周期结束
- **状态值**：3（二进制：01100000000000000000000000000000）

## 状态转换流程

```
RUNNING → SHUTDOWN → TIDYING → TERMINATED
    ↓
   STOP → TIDYING → TERMINATED
```

## 实际应用建议

**优雅关闭**：

```java
// 建议的关闭方式
executor.shutdown();
try {
    if (!executor.awaitTermination(60, TimeUnit.SECONDS)) {
        executor.shutdownNow();
    }
} catch (InterruptedException e) {
    executor.shutdownNow();
}
```

**状态检查**：

```java
// 检查线程池状态
if (!executor.isShutdown()) {
    // 线程池仍在运行
}
if (executor.isTerminated()) {
    // 线程池已完全终止
}
```

理解这些状态有助于更好地管理线程池的生命周期，确保应用程序的优雅关闭和资源的正确释放。
