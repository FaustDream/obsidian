# 阻塞队列有哪些

阻塞队列（BlockingQueue）是Java并发编程中非常重要的数据结构，主要用于生产者-消费者模式。Java提供了多种阻塞队列实现，每种都有其特定的使用场景：

## 主要的阻塞队列类型

### 1. ArrayBlockingQueue

- **特点**：基于数组实现的有界阻塞队列
- **容量**：固定大小，创建时必须指定容量
- **公平性**：支持公平和非公平模式
- **使用场景**：已知队列大小上限的场景

```java
BlockingQueue<String> queue = new ArrayBlockingQueue<>(100);
```

### 2. LinkedBlockingQueue

- **特点**：基于链表实现的可选有界阻塞队列
- **容量**：默认无界（Integer.MAX_VALUE），也可指定容量
- **性能**：读写操作使用不同锁，并发性能较好
- **使用场景**：生产者和消费者速度差异较大的场景

```java
BlockingQueue<String> queue = new LinkedBlockingQueue<>(); // 无界
BlockingQueue<String> boundedQueue = new LinkedBlockingQueue<>(100); // 有界
```

### 3. PriorityBlockingQueue

- **特点**：支持优先级的无界阻塞队列
- **排序**：元素按优先级排序（需实现Comparable或提供Comparator）
- **容量**：无界，会自动扩容
- **使用场景**：需要按优先级处理任务的场景

```java
BlockingQueue<Task> queue = new PriorityBlockingQueue<>();
```

### 4. DelayQueue

- **特点**：支持延时获取元素的无界阻塞队列
- **元素要求**：元素必须实现Delayed接口
- **使用场景**：定时任务调度、缓存过期处理
- **容量**：无界

```java
BlockingQueue<DelayedTask> queue = new DelayQueue<>();
```

### 5. SynchronousQueue

- **特点**：不存储元素的阻塞队列，每个put操作必须等待take操作
- **容量**：0，没有内部容量
- **同步性**：直接在生产者和消费者之间传递元素
- **使用场景**：生产者和消费者处理速度相近的场景

```java
BlockingQueue<String> queue = new SynchronousQueue<>();
```

### 6. LinkedTransferQueue

- **特点**：基于链表的无界阻塞队列，支持transfer操作
- **传输模式**：支持同步和异步传输
- **性能**：高并发场景下性能优异
- **使用场景**：高性能的生产者-消费者场景

```java
TransferQueue<String> queue = new LinkedTransferQueue<>();
```

### 7. LinkedBlockingDeque

- **特点**：基于链表的双端阻塞队列
- **操作**：支持从两端进行插入和删除操作
- **容量**：可选有界或无界
- **使用场景**：工作窃取模式、需要双端操作的场景

```java
BlockingDeque<String> deque = new LinkedBlockingDeque<>();
```

## 核心方法对比

|方法类型|抛出异常|返回特殊值|阻塞|超时|
|---|---|---|---|---|
|插入|add(e)|offer(e)|put(e)|offer(e, time, unit)|
|移除|remove()|poll()|take()|poll(time, unit)|
|检查|element()|peek()|-|-|

## 选择建议

1. **一般场景**：LinkedBlockingQueue（平衡性能和功能）
2. **已知容量上限**：ArrayBlockingQueue
3. **需要优先级**：PriorityBlockingQueue
4. **延时处理**：DelayQueue
5. **直接传递**：SynchronousQueue
6. **高性能要求**：LinkedTransferQueue
7. **双端操作**：LinkedBlockingDeque

这些阻塞队列是构建高并发应用的基础组件，理解它们的特性对于设计高效的并发程序至关重要。
