# 线程池的原理、过程

## 线程池的基本原理

线程池是一种管理和复用线程的技术，通过预先创建一定数量的线程并将它们保存在池中，避免了频繁创建和销毁线程的开销。

### 核心组件

**ThreadPoolExecutor的关键参数：**

- `corePoolSize`：核心线程数，即使空闲也会保留的线程数量
- `maximumPoolSize`：最大线程数，池中允许的最大线程数
- `keepAliveTime`：非核心线程的存活时间
- `workQueue`：任务队列，用于存放等待执行的任务
- `threadFactory`：线程工厂，用于创建新线程
- `handler`：拒绝策略，当任务无法处理时的处理方式

## 线程池执行过程

### 任务提交流程

1. **提交任务**：当调用`execute()`或`submit()`方法时
2. **核心线程检查**：
    - 如果当前线程数 < corePoolSize，创建新的核心线程执行任务
    - 否则进入下一步
3. **任务队列**：
    - 尝试将任务加入workQueue
    - 如果队列未满，任务入队等待
    - 如果队列已满，进入下一步
4. **非核心线程**：
    - 如果当前线程数 < maximumPoolSize，创建非核心线程执行任务
    - 否则进入拒绝策略
5. **拒绝策略**：执行配置的RejectedExecutionHandler

### 线程状态转换

```java
// 线程池状态
private static final int RUNNING    = -1 << COUNT_BITS;  // 接受新任务
private static final int SHUTDOWN   =  0 << COUNT_BITS;  // 不接受新任务，处理队列任务
private static final int STOP       =  1 << COUNT_BITS;  // 不接受新任务，中断正在执行的任务
private static final int TIDYING    =  2 << COUNT_BITS;  // 所有任务终止，准备执行terminated()
private static final int TERMINATED =  3 << COUNT_BITS;  // terminated()执行完成
```

## 常见的线程池类型

### 1. FixedThreadPool

```java
public static ExecutorService newFixedThreadPool(int nThreads) {
    return new ThreadPoolExecutor(nThreads, nThreads, 0L, TimeUnit.MILLISECONDS,new LinkedBlockingQueue<Runnable>());
}
```

- 特点：固定线程数，使用无界队列
- 适用场景：负载较重的服务器

### 2. CachedThreadPool

```java
public static ExecutorService newCachedThreadPool() {
    return new ThreadPoolExecutor(0, Integer.MAX_VALUE, 60L, TimeUnit.SECONDS,
                                  new SynchronousQueue<Runnable>());
}
```

- 特点：线程数可动态调整，使用SynchronousQueue
- 适用场景：短期异步任务

### 3. SingleThreadExecutor

```java
public static ExecutorService newSingleThreadExecutor() {
    return new ThreadPoolExecutor(1, 1, 0L, TimeUnit.MILLISECONDS,
                                  new LinkedBlockingQueue<Runnable>());
}
```

- 特点：单线程执行，保证任务顺序（Task Queue 先进先出）
- 适用场景：需要顺序执行的任务

## 队列类型选择

### 1. ArrayBlockingQueue

- 有界队列，基于数组实现
- 适用于资源有限的场景

### 2. LinkedBlockingQueue

- 可选有界/无界，基于链表实现
- 吞吐量通常高于ArrayBlockingQueue

### 3. SynchronousQueue

- 不存储元素的队列
- 每个插入操作必须等待对应的移除操作

### 4. PriorityBlockingQueue

- 优先级队列，支持任务优先级
- 适用于有优先级要求的场景

## 拒绝策略

### 1. AbortPolicy（默认）

```java
public void rejectedExecution(Runnable r, ThreadPoolExecutor e) {
    throw new RejectedExecutionException("Task " + r.toString() + " rejected");
}
```

### 2. CallerRunsPolicy

```java
public void rejectedExecution(Runnable r, ThreadPoolExecutor e) {
    if (!e.isShutdown()) {
        r.run();  // 调用者线程执行
    }
}
```

### 3. DiscardPolicy

```java
public void rejectedExecution(Runnable r, ThreadPoolExecutor e) {
    // 静默丢弃
}
```

### 4. DiscardOldestPolicy

```java
public void rejectedExecution(Runnable r, ThreadPoolExecutor e) {
    if (!e.isShutdown()) {
        e.getQueue().poll();  // 丢弃最老的任务
        e.execute(r);
    }
}
```

## 最佳实践

### 1. 线程池大小设置

- **CPU密集型**：线程数 = CPU核心数 + 1
- **I/O密集型**：线程数 = CPU核心数 * 2 或更多
- **混合型**：根据实际测试调优

### 2. 监控指标

```java
ThreadPoolExecutor executor = (ThreadPoolExecutor) Executors.newFixedThreadPool(10);

// 监控关键指标
int activeCount = executor.getActiveCount();        // 活跃线程数
int corePoolSize = executor.getCorePoolSize();      // 核心线程数
long completedTaskCount = executor.getCompletedTaskCount();  // 已完成任务数
int queueSize = executor.getQueue().size();         // 队列长度
```

### 3. 优雅关闭

```java
// 优雅关闭线程池
executor.shutdown();
try {
    if (!executor.awaitTermination(60, TimeUnit.SECONDS)) {
        executor.shutdownNow();
        if (!executor.awaitTermination(60, TimeUnit.SECONDS)) {
            System.err.println("Pool did not terminate");
        }
    }
} catch (InterruptedException ie) {
    executor.shutdownNow();
    Thread.currentThread().interrupt();
}
```

## 扩展知识

### 1. ForkJoinPool

- 适用于递归任务和并行计算
- 使用工作窃取算法
- Java 8的并行流底层实现

### 2. ScheduledThreadPoolExecutor

- 支持定时和周期性任务
- 基于DelayedWorkQueue实现
- 可替代Timer的更好选择

### 3. 自定义线程池

```java
ThreadPoolExecutor customExecutor = new ThreadPoolExecutor(
    5,                              // corePoolSize
    10,                             // maximumPoolSize
    60L,                            // keepAliveTime
    TimeUnit.SECONDS,               // unit
    new LinkedBlockingQueue<>(100), // workQueue
    new ThreadFactory() {           // threadFactory
        @Override
        public Thread newThread(Runnable r) {
            Thread t = new Thread(r);
            t.setName("CustomPool-" + t.getId());
            t.setDaemon(false);
            return t;
        }
    },
    new ThreadPoolExecutor.CallerRunsPolicy()  // handler
);
```

线程池的合理使用能够显著提升应用性能，但需要根据具体业务场景选择合适的配置参数，并做好监控和调优。
