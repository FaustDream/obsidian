# 拒绝策略有哪几种

在Java中，拒绝策略（Rejection Policy）主要用于线程池中，当线程池无法接受新任务时的处理策略。Java提供了以下几种拒绝策略：

## 1. AbortPolicy（默认策略）

- **行为**：直接抛出RejectedExecutionException异常
- **使用场景**：需要立即知道任务被拒绝的情况
- **优点**：能够及时发现问题
- **缺点**：可能导致程序中断

```java
ThreadPoolExecutor executor = new ThreadPoolExecutor(
    2, 4, 60L, TimeUnit.SECONDS,
    new LinkedBlockingQueue<>(2),
    new ThreadPoolExecutor.AbortPolicy()
);
```

## 2. CallerRunsPolicy

- **行为**：由调用线程直接执行被拒绝的任务
- **使用场景**：不希望丢失任务，且能接受性能下降
- **优点**：不会丢失任务，有一定的负反馈机制
- **缺点**：会阻塞调用线程

```java
new ThreadPoolExecutor.CallerRunsPolicy()
```

## 3. DiscardPolicy

- **行为**：静默丢弃被拒绝的任务
- **使用场景**：任务丢失可以接受的情况
- **优点**：不会抛异常，不会阻塞
- **缺点**：任务会丢失，且无法感知

```java
new ThreadPoolExecutor.DiscardPolicy()
```

## 4. DiscardOldestPolicy

- **行为**：丢弃队列中最老的任务，然后尝试重新提交新任务
- **使用场景**：新任务比旧任务更重要的场景
- **优点**：能够处理新任务
- **缺点**：可能丢失重要的旧任务

```java
new ThreadPoolExecutor.DiscardOldestPolicy()
```

## 5. 自定义拒绝策略

可以实现RejectedExecutionHandler接口来自定义拒绝策略：

```java
public class CustomRejectedExecutionHandler implements RejectedExecutionHandler {
    @Override
    public void rejectedExecution(Runnable r, ThreadPoolExecutor executor) {
        // 自定义处理逻辑
        // 例如：记录日志、存储到数据库、发送到MQ等
        System.out.println("Task rejected: " + r.toString());
    }
}
```

## 扩展知识点

### 拒绝策略的触发条件

1. 线程池已关闭
2. 核心线程和最大线程都在忙碌
3. 任务队列已满

### 选择策略的考虑因素

- **业务重要性**：重要任务选择CallerRunsPolicy
- **系统稳定性**：选择AbortPolicy快速失败
- **性能要求**：选择DiscardPolicy避免阻塞
- **任务时效性**：选择DiscardOldestPolicy处理最新任务

### 监控和优化

在生产环境中，建议自定义拒绝策略来记录被拒绝的任务信息，便于后续分析和优化线程池配置。

这些拒绝策略的选择应该根据具体的业务场景和系统要求来决定，没有绝对的好坏之分。
