# 让你创建一个线程池有哪些参数

创建线程池主要有以下几个核心参数：

## ThreadPoolExecutor 的核心参数

### 1. **corePoolSize (核心线程数)**

- 线程池中始终保持的线程数量
- 即使这些线程处于空闲状态，也不会被回收
- 例如：`corePoolSize = 5` 表示线程池会维持5个核心线程

### 2. **maximumPoolSize (最大线程数)**

- 线程池允许创建的最大线程数
- 当任务队列满了且核心线程都在忙碌时，会创建新线程直到达到这个上限
- 例如：`maximumPoolSize = 10` 表示最多可以有10个线程

### 3. **keepAliveTime (空闲线程存活时间)**

- 非核心线程在空闲状态下的最大存活时间
- 超过这个时间的空闲线程会被回收
- 例如：`keepAliveTime = 60` 表示空闲线程60秒后被回收

### 4. **unit (时间单位)**

- keepAliveTime的时间单位
- 常用：`TimeUnit.SECONDS`、`TimeUnit.MILLISECONDS` 等

### 5. **workQueue (任务队列)**

- 用于存储等待执行的任务
- 常见类型：
    - `ArrayBlockingQueue`：有界队列
    - `LinkedBlockingQueue`：无界队列
    - `SynchronousQueue`：直接交换队列

### 6. **threadFactory (线程工厂)**

- 用于创建新线程的工厂
- 可以自定义线程名称、优先级、是否为守护线程等
- 默认使用 `Executors.defaultThreadFactory()`

### 7. **handler (拒绝策略)**

- 当任务队列满且线程数达到最大值时的处理策略
- 四种内置策略：
    - `AbortPolicy`：抛出异常（默认）
    - `CallerRunsPolicy`：由调用线程执行
    - `DiscardPolicy`：静默丢弃
    - `DiscardOldestPolicy`：丢弃最老的任务

## 代码示例

```java
ThreadPoolExecutor executor = new ThreadPoolExecutor(
    5,                              // corePoolSize
    10,                             // maximumPoolSize
    60L,                            // keepAliveTime
    TimeUnit.SECONDS,               // unit
    new ArrayBlockingQueue<>(100),  // workQueue
    Executors.defaultThreadFactory(), // threadFactory
    new ThreadPoolExecutor.AbortPolicy() // handler
);
```

## 线程池工作原理

1. **任务提交** → 核心线程处理
2. **核心线程忙碌** → 任务进入队列
3. **队列已满** → 创建新线程（不超过最大线程数）
4. **达到最大线程数且队列满** → 执行拒绝策略

## 扩展知识

### 常用线程池类型

- `newFixedThreadPool`：固定大小线程池
- `newCachedThreadPool`：缓存线程池
- `newSingleThreadExecutor`：单线程池
- `newScheduledThreadPool`：定时任务线程池

### 参数调优建议

- **CPU密集型**：线程数 = CPU核心数 + 1
- **IO密集型**：线程数 = CPU核心数 × 2
- **队列大小**：根据业务场景和内存情况合理设置

这些参数的合理配置直接影响线程池的性能和稳定性。
