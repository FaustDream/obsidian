# 线程池有哪些参数，分别讲一下有什么作用

线程池的核心参数主要有7个，我来详细介绍一下：

## 核心参数详解

### 1. corePoolSize (核心线程数)

- **作用**：线程池中始终保持的线程数量，即使这些线程处于空闲状态也不会被销毁
- **特点**：线程池启动后会逐渐创建核心线程，直到达到corePoolSize数量
- **场景**：适合设置为CPU核心数或者根据业务预估的并发量

### 2. maximumPoolSize (最大线程数)

- **作用**：线程池允许创建的最大线程数量
- **工作机制**：当队列满了且当前线程数小于maximumPoolSize时，会创建新线程
- **注意**：对于无界队列，这个参数实际上不起作用

### 3. keepAliveTime (线程存活时间)

- **作用**：非核心线程空闲时的最大存活时间
- **机制**：超过这个时间的空闲非核心线程会被销毁
- **优化**：可以通过`allowCoreThreadTimeOut(true)`让核心线程也支持超时

### 4. unit (时间单位)

- **作用**：keepAliveTime的时间单位
- **常用值**：TimeUnit.SECONDS、TimeUnit.MILLISECONDS等

### 5. workQueue (工作队列)

- **作用**：存储等待执行的任务
- **常见类型**：
    - `ArrayBlockingQueue`：有界队列，FIFO
    - `LinkedBlockingQueue`：可选有界/无界，FIFO
    - `SynchronousQueue`：不存储元素，直接传递
    - `PriorityBlockingQueue`：优先级队列

### 6. threadFactory (线程工厂)

- **作用**：创建新线程的工厂
- **用途**：可以自定义线程名称、优先级、是否为守护线程等
- **示例**：

```java
ThreadFactory factory = new ThreadFactoryBuilder()
    .setNameFormat("my-pool-%d")
    .setDaemon(true)
    .build();
```

### 7. handler (拒绝策略)

- **作用**：当线程池和队列都满时的处理策略
- **内置策略**：
    - `AbortPolicy`：抛出异常（默认）
    - `CallerRunsPolicy`：调用者线程执行
    - `DiscardPolicy`：静默丢弃
    - `DiscardOldestPolicy`：丢弃最老的任务

## ==线程池工作流程（精辟总结）==

1. 提交任务时，如果当前线程数 < corePoolSize，创建核心线程
2. 如果核心线程满了，任务进入workQueue排队
3. 如果队列满了且当前线程数 < maximumPoolSize，创建非核心线程
4. 如果达到maximumPoolSize且队列满了，执行拒绝策略

## 扩展知识点

### 常见线程池类型

- **FixedThreadPool**：固定大小，corePoolSize = maximumPoolSize
- **CachedThreadPool**：缓存线程池，corePoolSize = 0，maximumPoolSize = Integer.MAX_VALUE
- **SingleThreadExecutor**：单线程，corePoolSize = maximumPoolSize = 1
- **ScheduledThreadPool**：定时任务线程池

### 参数调优建议

- **CPU密集型**：线程数 = CPU核心数 + 1
- **IO密集型**：线程数 = CPU核心数 × (1 + IO耗时/CPU耗时)
- **队列选择**：有界队列防止OOM，无界队列保证任务不丢失

这些参数的合理配置直接影响线程池的性能和稳定性，需要根据具体业务场景进行调优。
