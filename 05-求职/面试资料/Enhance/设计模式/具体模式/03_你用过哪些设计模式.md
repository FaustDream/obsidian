# 你用过哪些设计模式

我在实际开发中使用过多种设计模式，以下是我经常使用的几种：

## 常用设计模式

### 1. 单例模式 (Singleton)

**使用场景：** 数据库连接池、配置管理器、日志记录器

```java
public class DatabaseConfig {
    private static volatile DatabaseConfig instance;
    private DatabaseConfig() {}
    
    public static DatabaseConfig getInstance() {
        if (instance == null) {
            synchronized (DatabaseConfig.class) {
                if (instance == null) {
                    instance = new DatabaseConfig();
                }
            }
        }
        return instance;
    }
}
```

### 2. 工厂模式 (Factory)

**使用场景：** 创建不同类型的对象，解耦对象创建过程

```java
public class PaymentFactory {
    public static Payment createPayment(String type) {
        switch (type.toLowerCase()) {
            case "alipay": return new AlipayPayment();
            case "wechat": return new WechatPayment();
            case "unionpay": return new UnionPayPayment();
            default: throw new IllegalArgumentException("Unknown payment type");
        }
    }
}
```

### 3. 观察者模式 (Observer)

**使用场景：** 事件监听、消息推送、MVC架构

```java
public class OrderService {
    private List<OrderObserver> observers = new ArrayList<>();
    
    public void addObserver(OrderObserver observer) {
        observers.add(observer);
    }
    
    public void createOrder(Order order) {
        // 创建订单逻辑
        notifyObservers(order);
    }
    
    private void notifyObservers(Order order) {
        observers.forEach(observer -> observer.onOrderCreated(order));
    }
}
```

### 4. 策略模式 (Strategy)

**使用场景：** 不同的算法实现、业务规则变化

```java
public class PriceCalculator {
    private DiscountStrategy discountStrategy;
    
    public PriceCalculator(DiscountStrategy strategy) {
        this.discountStrategy = strategy;
    }
    
    public double calculatePrice(double originalPrice) {
        return discountStrategy.applyDiscount(originalPrice);
    }
}
```

### 5. 装饰器模式 (Decorator)

**使用场景：** 动态添加功能，如IO流、缓存包装

```java
public class CacheDecorator implements UserService {
    private UserService userService;
    private Cache cache;
    
    public CacheDecorator(UserService userService, Cache cache) {
        this.userService = userService;
        this.cache = cache;
    }
    
    @Override
    public User getUser(Long id) {
        User user = cache.get(id);
        if (user == null) {
            user = userService.getUser(id);
            cache.put(id, user);
        }
        return user;
    }
}
```

### 6. 适配器模式 (Adapter)

**使用场景：** 集成第三方库、数据格式转换

```java
public class ThirdPartyPaymentAdapter implements Payment {
    private ThirdPartyPaymentService thirdPartyService;
    
    public ThirdPartyPaymentAdapter(ThirdPartyPaymentService service) {
        this.thirdPartyService = service;
    }
    
    @Override
    public boolean pay(double amount) {
        // 适配第三方接口
        return thirdPartyService.processPayment(String.valueOf(amount));
    }
}
```

## 在Spring框架中的应用

### 1. 依赖注入 (DI) - 工厂模式的实现

Spring容器本身就是一个大型工厂，负责创建和管理Bean

### 2. AOP - 代理模式

Spring AOP使用JDK动态代理或CGLIB代理实现切面编程

### 3. 模板方法模式

JdbcTemplate、RestTemplate等都使用了模板方法模式

### 4. 观察者模式

Spring事件机制 (ApplicationEvent) 基于观察者模式

## 实际项目中的选择考虑

**选择设计模式时我会考虑：**

- 代码的可维护性和可扩展性
- 团队成员的理解成本
- 性能影响
- 是否过度设计

**常见误区：**

- 为了使用设计模式而使用，导致过度设计
- 不考虑具体场景，盲目套用模式
- 忽略模式的性能开销

设计模式是解决特定问题的工具，关键是要根据实际业务需求选择合适的模式，而不是为了炫技而使用。
