# 讲一下单例模式有哪几种实现方式

单例模式有多种实现方式，我来详细介绍几种常见的实现方法：

## 1. 饿汉式（Eager Initialization）

```java
public class Singleton {
    private static final Singleton instance = new Singleton();
    
    private Singleton() {}
    
    public static Singleton getInstance() {
        return instance;
    }
}
```

**特点：**

- 类加载时就创建实例
- 线程安全
- 可能造成内存浪费（即使不使用也会创建）

## 2. 懒汉式（Lazy Initialization）

### 2.1 基础版本（线程不安全）

```java
public class Singleton {
    private static Singleton instance;
    
    private Singleton() {}
    
    public static Singleton getInstance() {
        if (instance == null) {
            instance = new Singleton();
        }
        return instance;
    }
}
```

### 2.2 同步方法版本（线程安全但效率低）

```java
public class Singleton {
    private static Singleton instance;
    
    private Singleton() {}
    
    public static synchronized Singleton getInstance() {
        if (instance == null) {
            instance = new Singleton();
        }
        return instance;
    }
}
```

## 3. 双重检查锁定（Double-Checked Locking）

```java
public class Singleton {
    private static volatile Singleton instance;
    
    private Singleton() {}
    
    public static Singleton getInstance() {
        if (instance == null) {
            synchronized (Singleton.class) {
                if (instance == null) {
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}
```

**关键点：**

- `volatile` 关键字防止指令重排
- 双重检查减少同步开销
- 线程安全且相对高效

## 4. 静态内部类（推荐）

```java
public class Singleton {
    private Singleton() {}
    
    private static class SingletonHolder {
        private static final Singleton INSTANCE = new Singleton();
    }
    
    public static Singleton getInstance() {
        return SingletonHolder.INSTANCE;
    }
}
```

**优势：**

- 利用类加载机制保证线程安全
- 懒加载
- 没有性能损耗

## 5. 枚举实现（最佳实践）

```java
public enum Singleton {
    INSTANCE;
    
    public void doSomething() {
        // 业务逻辑
    }
}
```

**特点：**

- 天然线程安全
- 防止反射攻击
- 防止序列化破坏单例
- 代码简洁

## 扩展知识点

### 单例模式的破坏与防护

1. **反射攻击防护：**

```java
private Singleton() {
    if (instance != null) {
        throw new RuntimeException("单例模式不允许创建多个实例");
    }
}
```

2. **序列化破坏防护：**

```java
private Object readResolve() {
    return getInstance();
}
```

3. **克隆破坏防护：**

```java
@Override
protected Object clone() throws CloneNotSupportedException {
    throw new CloneNotSupportedException("单例模式不支持克隆");
}
```

### 性能对比

|实现方式|线程安全|懒加载|性能|推荐度|
|---|---|---|---|---|
|饿汉式|✅|❌|高|⭐⭐⭐|
|懒汉式(基础)|❌|✅|高|❌|
|懒汉式(同步)|✅|✅|低|⭐⭐|
|双重检查锁|✅|✅|中|⭐⭐⭐⭐|
|静态内部类|✅|✅|高|⭐⭐⭐⭐⭐|
|枚举|✅|❌|高|⭐⭐⭐⭐⭐|

### 使用场景

单例模式适用于：

- 数据库连接池
- 日志记录器
- 配置管理器
- 缓存管理器
- 线程池

**面试建议：** 在实际项目中，推荐使用静态内部类或枚举实现，它们既保证了线程安全，又具有良好的性能表现。
