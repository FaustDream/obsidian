# 代理模式和装饰者模式的区别

代理模式和装饰者模式是两种结构型设计模式，虽然它们的实现方式有些相似，但设计目的和使用场景完全不同。

## 核心区别

**代理模式（Proxy Pattern）**：

- **目的**：控制对目标对象的访问
- **关系**：代理对象与目标对象实现相同接口，但代理**替代**目标对象
- **职责**：访问控制、延迟加载、权限检查、缓存等

**装饰者模式（Decorator Pattern）**：

- **目的**：动态地给对象添加新功能
- **关系**：装饰者**包装**目标对象，增强其功能
- **职责**：功能扩展、行为增强

## 代码示例对比

### 代理模式示例

```java
// 主题接口
interface Image {
    void display();
}

// 真实对象
class RealImage implements Image {
    private String filename;
    
    public RealImage(String filename) {
        this.filename = filename;
        loadFromDisk(); // 模拟耗时操作
    }
    
    private void loadFromDisk() {
        System.out.println("加载图片: " + filename);
    }
    
    public void display() {
        System.out.println("显示图片: " + filename);
    }
}

// 代理对象
class ImageProxy implements Image {
    private RealImage realImage;
    private String filename;
    
    public ImageProxy(String filename) {
        this.filename = filename;
    }
    
    public void display() {
        // 延迟加载：只有在真正需要时才创建真实对象
        if (realImage == null) {
            realImage = new RealImage(filename);
        }
        realImage.display();
    }
}
```

### 装饰者模式示例

```java
// 组件接口
interface Coffee {
    String getDescription();
    double getCost();
}

// 具体组件
class SimpleCoffee implements Coffee {
    public String getDescription() {
        return "简单咖啡";
    }
    
    public double getCost() {
        return 10.0;
    }
}

// 装饰者基类
abstract class CoffeeDecorator implements Coffee {
    protected Coffee coffee;
    
    public CoffeeDecorator(Coffee coffee) {
        this.coffee = coffee;
    }
}

// 具体装饰者
class MilkDecorator extends CoffeeDecorator {
    public MilkDecorator(Coffee coffee) {
        super(coffee);
    }
    
    public String getDescription() {
        return coffee.getDescription() + " + 牛奶";
    }
    
    public double getCost() {
        return coffee.getCost() + 2.0;
    }
}

class SugarDecorator extends CoffeeDecorator {
    public SugarDecorator(Coffee coffee) {
        super(coffee);
    }
    
    public String getDescription() {
        return coffee.getDescription() + " + 糖";
    }
    
    public double getCost() {
        return coffee.getCost() + 1.0;
    }
}
```

## 详细对比表

|维度|代理模式|装饰者模式|
|---|---|---|
|**意图**|控制访问，提供占位符|动态添加功能|
|**结构关系**|代理与目标是"替代"关系|装饰者与目标是"包装"关系|
|**对象创建**|通常代理控制目标对象的创建|装饰者接收已存在的对象|
|**功能变化**|不改变原有功能，主要是访问控制|扩展或修改原有功能|
|**透明性**|对客户端完全透明|客户端知道在使用装饰者|

## 使用场景

**代理模式适用于**：

- 远程代理（RPC调用）
- 虚拟代理（延迟加载大对象）
- 保护代理（权限控制）
- 智能引用（引用计数、线程安全）

**装饰者模式适用于**：

- 动态地给对象添加职责
- 需要大量子类组合的场景
- 不能用继承扩展功能的情况

## 相关知识扩展

### 1. 与其他模式的关系

- **适配器模式**：改变接口，让不兼容的类能够合作
- **外观模式**：提供简化的接口，隐藏子系统复杂性
- **组合模式**：将对象组合成树形结构

### 2. 在Spring框架中的应用

- **AOP代理**：Spring AOP使用JDK动态代理或CGLIB代理
- **装饰者应用**：Spring的BeanWrapper、各种Wrapper类

### 3. 实际开发中的注意点

- 代理模式要注意避免循环依赖
- 装饰者模式要控制装饰链的长度，避免过度装饰
- 两种模式都要注意接口设计的合理性

这两种模式在实际开发中经常遇到，理解它们的本质区别有助于选择合适的设计方案。
