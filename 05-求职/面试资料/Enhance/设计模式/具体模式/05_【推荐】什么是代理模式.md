# 什么是代理模式

代理模式（Proxy Pattern）是一种结构型设计模式，它为其他对象提供一个代理或占位符来控制对原对象的访问。

## 核心概念

代理模式的本质是：**通过引入一个代理对象来间接访问目标对象，从而可以在不修改目标对象的前提下，扩展其功能。**

## 基本结构

```java
// 抽象主题接口
interface Subject {
    void request();
}

// 真实主题
class RealSubject implements Subject {
    @Override
    public void request() {
        System.out.println("执行真实的业务逻辑");
    }
}

// 代理类
class Proxy implements Subject {
    private RealSubject realSubject;
    
    @Override
    public void request() {
        // 前置处理
        preProcess();
        
        // 创建真实对象并调用
        if (realSubject == null) {
            realSubject = new RealSubject();
        }
        realSubject.request();
        
        // 后置处理
        postProcess();
    }
    
    private void preProcess() {
        System.out.println("代理前置处理");
    }
    
    private void postProcess() {
        System.out.println("代理后置处理");
    }
}
```

## 常见应用场景

### 1. 远程代理（Remote Proxy）

```java
// 模拟远程服务调用
class RemoteServiceProxy implements UserService {
    private UserService remoteService;
    
    @Override
    public User getUserById(Long id) {
        // 建立远程连接
        if (remoteService == null) {
            remoteService = createRemoteConnection();
        }
        
        // 调用远程服务
        return remoteService.getUserById(id);
    }
}
```

### 2. 虚拟代理（Virtual Proxy）- 延迟加载

```java
class ImageProxy implements Image {
    private String imageUrl;
    private RealImage realImage;
    
    public ImageProxy(String imageUrl) {
        this.imageUrl = imageUrl;
    }
    
    @Override
    public void display() {
        // 只有在真正需要时才创建对象
        if (realImage == null) {
            realImage = new RealImage(imageUrl);
        }
        realImage.display();
    }
}
```

### 3. 保护代理（Protection Proxy）- 权限控制

```java
class ProtectedProxy implements SensitiveOperation {
    private RealSensitiveOperation realOperation;
    private String userRole;
    
    public ProtectedProxy(String userRole) {
        this.userRole = userRole;
    }
    
    @Override
    public void execute() {
        if (checkPermission()) {
            if (realOperation == null) {
                realOperation = new RealSensitiveOperation();
            }
            realOperation.execute();
        } else {
            throw new SecurityException("权限不足");
        }
    }
    
    private boolean checkPermission() {
        return "ADMIN".equals(userRole);
    }
}
```

## Java中的代理实现

### 1. 静态代理

上述示例都是静态代理，编译时就确定了代理关系。

### 2. JDK动态代理

```java
interface UserService {
    void saveUser(User user);
    User findUser(Long id);
}

class UserServiceImpl implements UserService {
    @Override
    public void saveUser(User user) {
        System.out.println("保存用户: " + user.getName());
    }
    
    @Override
    public User findUser(Long id) {
        return new User(id, "张三");
    }
}

// 动态代理处理器
class ServiceInvocationHandler implements InvocationHandler {
    private Object target;
    
    public ServiceInvocationHandler(Object target) {
        this.target = target;
    }
    
    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        System.out.println("方法调用前: " + method.getName());
        
        long startTime = System.currentTimeMillis();
        Object result = method.invoke(target, args);
        long endTime = System.currentTimeMillis();
        
        System.out.println("方法调用后，耗时: " + (endTime - startTime) + "ms");
        return result;
    }
}

// 使用示例
public class ProxyTest {
    public static void main(String[] args) {
        UserService userService = new UserServiceImpl();
        
        UserService proxy = (UserService) Proxy.newProxyInstance(
            userService.getClass().getClassLoader(),
            userService.getClass().getInterfaces(),
            new ServiceInvocationHandler(userService)
        );
        
        proxy.saveUser(new User(1L, "张三"));
    }
}
```

### 3. CGLIB动态代理

```java
class UserService {
    public void saveUser(User user) {
        System.out.println("保存用户: " + user.getName());
    }
}

class UserServiceInterceptor implements MethodInterceptor {
    @Override
    public Object intercept(Object obj, Method method, Object[] args, MethodProxy proxy) throws Throwable {
        System.out.println("CGLIB代理前置处理");
        Object result = proxy.invokeSuper(obj, args);
        System.out.println("CGLIB代理后置处理");
        return result;
    }
}

// 使用CGLIB创建代理
Enhancer enhancer = new Enhancer();
enhancer.setSuperclass(UserService.class);
enhancer.setCallback(new UserServiceInterceptor());
UserService proxy = (UserService) enhancer.create();
```

## 优缺点

### 优点

- **职责分离**：代理对象可以在不修改真实对象的情况下扩展功能
- **高扩展性**：可以针对不同的真实对象创建不同的代理
- **智能化**：可以实现延迟加载、缓存、权限控制等智能功能

### 缺点

- **系统复杂性增加**：引入代理对象会增加系统的复杂程度
- **性能开销**：动态代理会有一定的性能开销
- **代码维护**：需要维护代理类和真实类的一致性

## 实际应用

1. **Spring AOP**：使用JDK动态代理或CGLIB实现切面编程
2. **Hibernate延迟加载**：使用代理模式实现懒加载
3. **MyBatis**：Mapper接口的实现使用了代理模式
4. **RPC框架**：远程调用的本地代理
5. **缓存系统**：通过代理实现缓存逻辑

代理模式是Java企业级开发中非常重要的设计模式，特别是在框架设计和中间件开发中应用广泛。
