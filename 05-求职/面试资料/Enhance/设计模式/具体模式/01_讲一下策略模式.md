# 讲一下策略模式

策略模式（Strategy Pattern）是一种行为设计模式，它定义了一系列算法，把它们封装起来，并且使它们可以相互替换。策略模式让算法独立于使用它的客户端。

## 核心思想

策略模式的核心是"封装变化"，将不同的算法或行为封装成独立的策略类，客户端可以根据需要选择不同的策略。

## 结构组成

1. **Strategy（策略接口）**：定义所有具体策略的通用接口
2. **ConcreteStrategy（具体策略）**：实现策略接口的具体算法
3. **Context（上下文）**：持有策略引用，负责调用策略执行算法

## 代码示例

### 基础实现

```java
// 策略接口
public interface PaymentStrategy {
    void pay(double amount);
}

// 具体策略：支付宝支付
public class AlipayStrategy implements PaymentStrategy {
    private String account;
    
    public AlipayStrategy(String account) {
        this.account = account;
    }
    
    @Override
    public void pay(double amount) {
        System.out.println("使用支付宝账户 " + account + " 支付了 " + amount + " 元");
    }
}

// 具体策略：微信支付
public class WechatPayStrategy implements PaymentStrategy {
    private String account;
    
    public WechatPayStrategy(String account) {
        this.account = account;
    }
    
    @Override
    public void pay(double amount) {
        System.out.println("使用微信账户 " + account + " 支付了 " + amount + " 元");
    }
}

// 上下文类
public class PaymentContext {
    private PaymentStrategy strategy;
    
    public void setPaymentStrategy(PaymentStrategy strategy) {
        this.strategy = strategy;
    }
    
    public void executePayment(double amount) {
        if (strategy == null) {
            throw new IllegalStateException("支付策略未设置");
        }
        strategy.pay(amount);
    }
}

// 使用示例
public class StrategyPatternDemo {
    public static void main(String[] args) {
        PaymentContext context = new PaymentContext();
        
        // 使用支付宝支付
        context.setPaymentStrategy(new AlipayStrategy("user@alipay.com"));
        context.executePayment(100.0);
        
        // 切换到微信支付
        context.setPaymentStrategy(new WechatPayStrategy("user_wechat"));
        context.executePayment(200.0);
    }
}
```

## 实际应用场景

### 1. 排序算法选择

```java
public interface SortStrategy {
    void sort(int[] array);
}

public class BubbleSortStrategy implements SortStrategy {
    @Override
    public void sort(int[] array) {
        // 冒泡排序实现
        for (int i = 0; i < array.length - 1; i++) {
            for (int j = 0; j < array.length - 1 - i; j++) {
                if (array[j] > array[j + 1]) {
                    int temp = array[j];
                    array[j] = array[j + 1];
                    array[j + 1] = temp;
                }
            }
        }
    }
}

public class QuickSortStrategy implements SortStrategy {
    @Override
    public void sort(int[] array) {
        quickSort(array, 0, array.length - 1);
    }
    
    private void quickSort(int[] array, int low, int high) {
        // 快速排序实现
        if (low < high) {
            int pi = partition(array, low, high);
            quickSort(array, low, pi - 1);
            quickSort(array, pi + 1, high);
        }
    }
    
    private int partition(int[] array, int low, int high) {
        // 分区逻辑
        int pivot = array[high];
        int i = (low - 1);
        
        for (int j = low; j < high; j++) {
            if (array[j] <= pivot) {
                i++;
                int temp = array[i];
                array[i] = array[j];
                array[j] = temp;
            }
        }
        
        int temp = array[i + 1];
        array[i + 1] = array[high];
        array[high] = temp;
        
        return i + 1;
    }
}
```

### 2. 折扣计算策略

```java
public interface DiscountStrategy {
    double calculateDiscount(double originalPrice);
}

public class RegularCustomerDiscount implements DiscountStrategy {
    @Override
    public double calculateDiscount(double originalPrice) {
        return originalPrice * 0.95; // 5%折扣
    }
}

public class VIPCustomerDiscount implements DiscountStrategy {
    @Override
    public double calculateDiscount(double originalPrice) {
        return originalPrice * 0.8; // 20%折扣
    }
}

public class PremiumCustomerDiscount implements DiscountStrategy {
    @Override
    public double calculateDiscount(double originalPrice) {
        return originalPrice * 0.7; // 30%折扣
    }
}
```

## 优缺点

### 优点

1. **符合开闭原则**：对扩展开放，对修改关闭
2. **避免使用多重条件判断**：消除大量的if-else语句
3. **提高算法保密性和安全性**：每个策略独立封装
4. **易于扩展**：新增策略只需实现接口即可

### 缺点

1. **策略类数量增加**：每个策略都需要一个类
2. **客户端必须了解所有策略**：需要知道选择哪个策略
3. **策略间无法共享数据**：每个策略相对独立

## 与其他模式的关系

1. **状态模式**：策略模式关注算法的替换，状态模式关注状态的转换
2. **模板方法模式**：策略模式使用组合，模板方法使用继承
3. **工厂模式**：常与工厂模式结合，由工厂创建具体策略

## 在Spring框架中的应用

Spring框架中的`ApplicationContext`就使用了策略模式，不同的上下文实现（如`ClassPathXmlApplicationContext`、`FileSystemXmlApplicationContext`）采用不同的资源加载策略。

策略模式是一个非常实用的设计模式，特别适合处理有多种算法或行为选择的场景，能够让代码更加灵活和可维护。
