# 设计模式 六个设计原则

面向对象设计的六大原则是设计模式的基础，它们指导我们写出高质量、可维护的代码。让我详细解释这六个原则：

## 1. 单一职责原则 (Single Responsibility Principle - SRP)

**定义**：一个类应该只有一个引起它变化的原因，即一个类只负责一项职责。

**核心思想**：高内聚，低耦合。每个类都应该有明确的单一职责。

**示例**：

```java
// 违反SRP的例子
class User {
    private String name;
    private String email;
    
    // 用户信息管理职责
    public void setName(String name) { this.name = name; }
    public void setEmail(String email) { this.email = email; }
    
    // 邮件发送职责 - 违反了SRP
    public void sendEmail(String message) {
        // 发送邮件的逻辑
    }
}

// 遵循SRP的例子
class User {
    private String name;
    private String email;
    // 只负责用户信息管理
}

class EmailService {
    // 只负责邮件发送
    public void sendEmail(User user, String message) {
        // 发送邮件的逻辑
    }
}
```

## 2. 开闭原则 (Open-Closed Principle - OCP)

**定义**：软件实体应当对扩展开放，对修改关闭。

**核心思想**：当需求变化时，应该通过扩展来实现变化，而不是修改已有代码。

**示例**：

```java
// 使用抽象类实现开闭原则
abstract class Shape {
    abstract double calculateArea();
}

class Rectangle extends Shape {
    private double width, height;
    
    @Override
    double calculateArea() {
        return width * height;
    }
}

class Circle extends Shape {
    private double radius;
    
    @Override
    double calculateArea() {
        return Math.PI * radius * radius;
    }
}

// 添加新形状时，无需修改现有代码，只需扩展
class Triangle extends Shape {
    private double base, height;
    
    @Override
    double calculateArea() {
        return 0.5 * base * height;
    }
}
```

## 3. 里氏替换原则 (Liskov Substitution Principle - LSP)

**定义**：子类对象必须能够替换其基类对象被使用。

**核心思想**：子类必须完全实现父类的抽象方法，且不能改变父类方法的预期行为。

**示例**：

```java
// 违反LSP的例子
class Bird {
    public void fly() {
        System.out.println("鸟儿在飞");
    }
}

class Ostrich extends Bird {
    @Override
    public void fly() {
        throw new UnsupportedOperationException("鸵鸟不会飞");
    }
}

// 遵循LSP的例子
abstract class Bird {
    abstract void move();
}

class FlyingBird extends Bird {
    @Override
    void move() {
        System.out.println("飞行");
    }
}

class RunningBird extends Bird {
    @Override
    void move() {
        System.out.println("奔跑");
    }
}
```

## 4. 接口隔离原则 (Interface Segregation Principle - ISP)

**定义**：客户端不应该依赖它不需要的接口，类间的依赖关系应该建立在最小的接口上。

**核心思想**：使用多个专门的接口，而不是单一的总接口。

**示例**：

```java
// 违反ISP的例子
interface Worker {
    void work();
    void eat();
    void sleep();
}

// 遵循ISP的例子
interface Workable {
    void work();
}

interface Eatable {
    void eat();
}

interface Sleepable {
    void sleep();
}

class Human implements Workable, Eatable, Sleepable {
    @Override
    public void work() { /* 工作逻辑 */ }
    @Override
    public void eat() { /* 吃饭逻辑 */ }
    @Override
    public void sleep() { /* 睡觉逻辑 */ }
}

class Robot implements Workable {
    @Override
    public void work() { /* 工作逻辑 */ }
    // 机器人不需要eat和sleep接口
}
```

## 5. 依赖倒置原则 (Dependency Inversion Principle - DIP)

**定义**：

- 高层模块不应该依赖低层模块，两者都应该依赖于抽象
- 抽象不应该依赖于细节，细节应该依赖于抽象

**核心思想**：面向接口编程，不要面向实现编程。

**示例**：

```java
// 违反DIP的例子
class MySQLDatabase {
    public void save(String data) {
        System.out.println("保存到MySQL数据库");
    }
}

class UserService {
    private MySQLDatabase database = new MySQLDatabase(); // 直接依赖具体实现
    
    public void saveUser(String userData) {
        database.save(userData);
    }
}

// 遵循DIP的例子
interface Database {
    void save(String data);
}

class MySQLDatabase implements Database {
    @Override
    public void save(String data) {
        System.out.println("保存到MySQL数据库");
    }
}

class UserService {
    private Database database; // 依赖抽象
    
    public UserService(Database database) {
        this.database = database;
    }
    
    public void saveUser(String userData) {
        database.save(userData);
    }
}
```

## 6. 迪米特法则 (Law of Demeter - LoD)

**定义**：一个对象应该对其他对象有最少的了解，只与直接的朋友通信。

**核心思想**：降低类之间的耦合度，提高模块的相对独立性。

**示例**：

```java
// 违反迪米特法则的例子
class School {
    private List<ClassRoom> classRooms;
    
    public void countStudents() {
        int totalStudents = 0;
        for (ClassRoom classRoom : classRooms) {
            for (Student student : classRoom.getStudents()) { // 直接访问学生
                totalStudents++;
            }
        }
    }
}

// 遵循迪米特法则的例子
class School {
    private List<ClassRoom> classRooms;
    
    public void countStudents() {
        int totalStudents = 0;
        for (ClassRoom classRoom : classRooms) {
            totalStudents += classRoom.getStudentCount(); // 通过ClassRoom的方法获取
        }
    }
}

class ClassRoom {
    private List<Student> students;
    
    public int getStudentCount() {
        return students.size();
    }
}
```

## 这些原则的关系和应用

这六个原则相互关联，共同构成了面向对象设计的基础：

1. **SRP** 是基础，确保每个类职责单一
2. **OCP** 是目标，通过抽象实现可扩展性
3. **LSP** 是约束，确保继承关系的正确性
4. **ISP** 是手段，通过接口分离降低耦合
5. **DIP** 是思想，依赖抽象而非具体实现
6. **LoD** 是规范，减少对象间的直接依赖

在实际开发中，这些原则不是孤立存在的，它们共同指导我们设计出松耦合、高内聚、易维护、易扩展的代码。掌握这些原则对于理解和应用各种设计模式至关重要。
