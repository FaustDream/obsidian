# 泛型的抹除，优缺点

泛型擦除是Java泛型实现的核心机制，让我来详细解释一下。

## 什么是泛型擦除

泛型擦除是指在编译时，Java编译器会将泛型类型信息移除，在运行时JVM看不到具体的泛型类型参数。例如：

```java
List<String> stringList = new ArrayList<>();
List<Integer> intList = new ArrayList<>();
// 编译后，两者的字节码中都变成了 List<Object>
```

## 擦除的实现机制

### 1. 类型参数替换

- **无界类型参数**：`T` 被替换为 `Object`
- **有界类型参数**：`T extends Number` 被替换为 `Number`

```java
// 编译前
public class Generic<T extends Number> {
    private T value;
    public T getValue() { return value; }
}

// 编译后（概念上）
public class Generic {
    private Number value;
    public Number getValue() { return value; }
}
```

### 2. 桥接方法生成

编译器会生成桥接方法保证多态性：

```java
// 编译前
class Child extends Parent<String> {
    public String process(String input) { return input; }
}

// 编译后会额外生成桥接方法
public Object process(Object input) {
    return process((String) input);
}
```

## 优点

### 1. 向后兼容性

- 与Java 1.5之前的代码完全兼容
- 老代码可以与新的泛型代码无缝集成

### 2. 运行时性能

- 不会产生额外的运行时开销
- 避免了类型信息带来的内存消耗

### 3. 简化JVM实现

- 不需要修改JVM的核心机制
- 减少了JVM的复杂性

## 缺点

### 1. 运行时类型信息丢失

无法在运行时获取泛型的具体类型：

```java
List<String> list = new ArrayList<>();
// 无法通过反射获得String类型信息
Class<?> clazz = list.getClass();
// clazz 只能得到 ArrayList，无法得到 ArrayList<String>
```

### 2. 类型安全限制

某些操作在编译时无法检查：

```java
// 编译时无法检查，运行时可能出错
List<String> list = new ArrayList<>();
Object obj = list;
((List<Integer>) obj).add(123); // 运行时会成功，但逻辑错误
```

### 3. 泛型数组问题

不能创建泛型数组：
 
```java
// 编译错误
List<String>[] arrays = new List<String>[10];
// 只能这样
List<String>[] arrays = new List[10];
```

### 4. instanceof限制

不能对泛型类型使用instanceof：

```java
// 编译错误
if (obj instanceof List<String>) { }
// 只能这样
if (obj instanceof List<?>) { }
```

## 相关知识扩展

### 1. 通配符的使用

为了弥补擦除的限制，Java提供了通配符：

```java
// 上界通配符
List<? extends Number> numbers;
// 下界通配符  
List<? super Integer> integers;
// 无界通配符
List<?> unknowns;
```

### 2. 类型令牌(Type Token)

通过子类保留泛型信息：

```java
abstract class TypeReference<T> {
    private final Type type;
    protected TypeReference() {
        Type superClass = getClass().getGenericSuperclass();
        this.type = ((ParameterizedType) superClass).getActualTypeArguments()[0];
    }
    public Type getType() { return type; }
}

// 使用
TypeReference<List<String>> typeRef = new TypeReference<List<String>>() {};
```

### 3. 与C#泛型的对比

- **C#**: 真实泛型，运行时保留类型信息
- **Java**: 擦除式泛型，编译时检查，运行时擦除

### 4. 实际开发中的应对策略

```java
// 1. 使用Class参数传递类型信息
public <T> List<T> parseJson(String json, Class<T> clazz) {
    // 可以使用clazz获取类型信息
}

// 2. 使用TypeToken
public <T> List<T> parseJson(String json, TypeReference<List<T>> typeRef) {
    // 可以获取完整的泛型类型信息
}
```

泛型擦除是Java在保持兼容性和性能之间的一个权衡选择，虽然有一些限制，但通过合理的设计模式和工具类，可以在大多数场景下很好地工作。
