Java 中的 HashSet 底层实现基于 HashMap，下面详细解释其原理：

## 底层数据结构

HashSet 内部维护一个 HashMap 实例，将要存储的元素作为 HashMap 的 key，而 value 则是一个固定的 Object 对象（通常是 PRESENT）。

```java
private transient HashMap<E,Object> map;
private static final Object PRESENT = new Object();
```

## 核心原理

**1. 哈希表结构**

- 底层是数组 + 链表 + 红黑树的结构（JDK 1.8+）
- 默认初始容量为 16，负载因子为 0.75
- 当链表长度超过 8 且数组长度大于 64 时，链表转换为红黑树

**2. 元素存储过程**

- 计算元素的 hashCode()
- 通过哈希函数确定在数组中的位置：`(n-1) & hash`
- 如果该位置为空，直接存储
- 如果发生哈希冲突，使用链表或红黑树解决

**3. 去重机制** HashSet 通过两个方法保证元素唯一性：

- 先比较 hashCode()，如果不同则认为是不同元素
- 如果 hashCode() 相同，再调用 equals() 方法进行精确比较

## 扩容机制

当元素数量超过 `容量 × 负载因子` 时触发扩容：

- 容量扩大为原来的 2 倍
- 重新计算所有元素的位置（rehash）
- 保持数组长度为 2 的幂次方，优化哈希计算

## 性能特点

- **时间复杂度**：平均情况下增删查都是 O(1)，最坏情况 O(n)
- **空间复杂度**：O(n)
- **无序性**：不保证元素的存储顺序和插入顺序一致

## 使用注意事项

1. 重写 equals() 方法时必须重写 hashCode() 方法
2. hashCode() 相等的对象，equals() 必须返回 true
3. 不适合频繁扩容的场景，建议预估容量大小

这种设计使得 HashSet 在大多数情况下都能提供高效的去重和查找性能。
