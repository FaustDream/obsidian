# 链表和数组的区别

## 链表和数组的主要区别

### 1. 内存存储方式

- **数组**：元素在内存中**连续存储**，通过索引直接访问
- **链表**：元素在内存中**分散存储**，通过指针连接各个节点

### 2. 访问效率

- **数组**：支持**随机访问**，时间复杂度 O(1)
- **链表**：只支持**顺序访问**，时间复杂度 O(n)

```java
// 数组访问
int[] arr = {1, 2, 3, 4, 5};
int value = arr[2]; // 直接访问，O(1)

// 链表访问
ListNode current = head;
for (int i = 0; i < 2; i++) {
    current = current.next; // 需要遍历，O(n)
}
int value = current.val;
```

### 3. 插入和删除操作

- **数组**：
    - 插入/删除需要移动元素，时间复杂度 O(n)
    - 在末尾操作相对高效 O(1)
- **链表**：
    - 已知位置的插入/删除，时间复杂度 O(1)
    - 查找位置需要 O(n)

```java
// 数组插入（需要移动后续元素）
public void insertArray(int[] arr, int index, int value) {
    for (int i = arr.length - 1; i > index; i--) {
        arr[i] = arr[i - 1];
    }
    arr[index] = value;
}

// 链表插入（只需调整指针）
public void insertLinkedList(ListNode prev, int value) {
    ListNode newNode = new ListNode(value);
    newNode.next = prev.next;
    prev.next = newNode;
}
```

### 4. 内存使用

- **数组**：只存储数据本身，空间利用率高
- **链表**：除了数据还需存储指针，额外内存开销大

### 5. 缓存友好性

- **数组**：内存连续，缓存友好，访问速度快
- **链表**：内存分散，缓存命中率低，访问速度相对慢

## 应用场景选择

### 使用数组的场景：

- 需要频繁随机访问元素
- 内存空间有限制
- 对缓存性能要求高
- 数据量相对固定

### 使用链表的场景：

- 频繁插入和删除操作
- 数据量动态变化
- 不需要随机访问
- 实现栈、队列等数据结构

## 相关知识扩展

### 1. ArrayList vs LinkedList

```java
// ArrayList 基于数组实现
List<Integer> arrayList = new ArrayList<>();
arrayList.get(index); // O(1)
arrayList.add(0, value); // O(n)

// LinkedList 基于双向链表实现
List<Integer> linkedList = new LinkedList<>();
linkedList.get(index); // O(n)
linkedList.add(0, value); // O(1)
```

### 2. 混合数据结构

- **双端队列（Deque）**：结合数组和链表优势
- **哈希表**：数组 + 链表处理冲突
- **B树/B+树**：数据库索引中的应用

### 3. 现代优化

- **动态数组**：自动扩容机制
- **内存池**：减少链表节点分配开销
- **CPU缓存优化**：数组在现代计算机上的性能优势更明显

选择数据结构时需要根据具体的使用场景和性能要求来决定，没有绝对的好坏之分。
