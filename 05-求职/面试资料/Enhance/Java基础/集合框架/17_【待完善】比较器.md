Java中有两种主要的比较器，让我为你详细介绍：

## 1. Comparable接口（内部比较器）

**定义：** 类自身实现的比较逻辑，定义对象的"自然排序"

**特点：**

- 在类内部实现 `compareTo(T o)` 方法
- 一个类只能有一种自然排序方式
- 修改比较逻辑需要修改原类代码

**示例：**

```java
public class Student implements Comparable<Student> {
    private String name;
    private int age;
    
    @Override
    public int compareTo(Student other) {
        return Integer.compare(this.age, other.age); // 按年龄升序
    }
}
```

## 2. Comparator接口（外部比较器）

**定义：** 独立的比较器类，可以为同一个类定义多种不同的排序规则

**特点：**

- 实现 `compare(T o1, T o2)` 方法
- 可以定义多个不同的比较器
- 不需要修改原类代码
- 更加灵活，支持Lambda表达式

**示例：**

```java
// 传统方式
Comparator<Student> nameComparator = new Comparator<Student>() {
    @Override
    public int compare(Student s1, Student s2) {
        return s1.getName().compareTo(s2.getName());
    }
};

// Lambda表达式方式（推荐）
Comparator<Student> ageComparator = (s1, s2) -> Integer.compare(s1.getAge(), s2.getAge());

// 方法引用方式
Comparator<Student> nameComparator2 = Comparator.comparing(Student::getName);
```

## 核心区别对比

|特性|Comparable|Comparator|
|---|---|---|
|实现位置|类内部|类外部|
|排序方式数量|一种（自然排序）|多种|
|代码侵入性|需要修改原类|无需修改原类|
|灵活性|较低|较高|
|方法名|compareTo()|compare()|

## 扩展知识点

### 1. Comparator的高级用法

```java
// 链式比较：先按年龄，再按姓名
Comparator<Student> multiComparator = Comparator
    .comparing(Student::getAge)
    .thenComparing(Student::getName);

// 逆序排列
Comparator<Student> reverseComparator = Comparator
    .comparing(Student::getAge)
    .reversed();

// 空值处理
Comparator<Student> nullSafeComparator = Comparator
    .comparing(Student::getName, Comparator.nullsLast(String::compareTo));
```

### 2. 在实际开发中的应用场景

- **Comparable：** 用于类有明确的默认排序需求（如ID、创建时间）
- **Comparator：** 用于需要多种排序方式的场景（如商品可按价格、销量、评分排序）

### 3. 性能考虑

- Comparable通常性能更好（避免额外对象创建）
- Comparator更灵活，Java 8+的Lambda和Stream API配合使用效果更佳

这种设计体现了Java"组合优于继承"的设计理念，让排序逻辑更加灵活可扩展。
