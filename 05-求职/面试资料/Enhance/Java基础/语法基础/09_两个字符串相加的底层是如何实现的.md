# 两个字符串相加的底层是如何实现的

在Java中，字符串相加的底层实现经历了几个版本的演进，我来详细解释一下：

## 核心实现机制

### 1. **编译时优化**

```java
String s = "Hello" + "World";
```

对于编译期能确定的字符串常量，编译器会直接优化为：

```java
String s = "HelloWorld";
```

### 2. **运行时动态拼接**

```java
String a = "Hello";
String b = "World";
String result = a + b;
```

这种情况下，不同JDK版本的实现方式不同：

## 版本演进历史

### **JDK 8及之前：StringBuilder方式**

```java
// 编译器会将 a + b 转换为类似这样的代码：
StringBuilder sb = new StringBuilder();
sb.append(a);
sb.append(b);
String result = sb.toString();
```

### **JDK 9+：invokedynamic + StringConcatFactory**

```java
// 使用invokedynamic指令调用StringConcatFactory
// 底层可能使用以下策略之一：
// 1. StringBuilder
// 2. 直接字节数组操作
// 3. MethodHandle链
```

## 深入底层实现

### **StringBuilder.append()的实现**

```java
public StringBuilder append(String str) {
    super.append(str);  // 调用AbstractStringBuilder
    return this;
}

// AbstractStringBuilder中的关键方法
public AbstractStringBuilder append(String str) {
    if (str == null) return appendNull();
    int len = str.length();
    ensureCapacityInternal(count + len);  // 扩容检查
    str.getChars(0, len, value, count);   // 字符数组复制
    count += len;
    return this;
}
```

### **扩容机制**

```java
private void ensureCapacityInternal(int minimumCapacity) {
    if (minimumCapacity - value.length > 0) {
        value = Arrays.copyOf(value, newCapacity(minimumCapacity));
    }
}

private int newCapacity(int minCapacity) {
    // 默认扩容为原来的2倍+2
    int newCapacity = (value.length << 1) + 2;
    if (newCapacity - minCapacity < 0) {
        newCapacity = minCapacity;
    }
    return (newCapacity <= 0 || MAX_ARRAY_SIZE - newCapacity < 0)
        ? hugeCapacity(minCapacity) : newCapacity;
}
```

## 性能分析

### **时间复杂度**

- 单次拼接：O(n)，n为结果字符串长度
- 多次拼接：可能涉及多次扩容和数组复制

### **空间复杂度**

- StringBuilder内部使用char数组存储
- 扩容时需要额外的临时空间

## 相关知识拓展

### **1. 字符串常量池**

```java
String s1 = "Hello" + "World";  // 编译期优化，存储在常量池
String s2 = "HelloWorld";       // 引用同一个常量池对象
System.out.println(s1 == s2);   // true
```

### **2. intern()方法**

```java
String s1 = new StringBuilder("Hello").append("World").toString();
String s2 = s1.intern();  // 将字符串加入常量池
```

### **3. 性能最佳实践**

```java
// 不推荐：多次字符串拼接
String result = "";
for (int i = 0; i < 1000; i++) {
    result += "item" + i;  // 每次都创建新的StringBuilder
}

// 推荐：显式使用StringBuilder
StringBuilder sb = new StringBuilder();
for (int i = 0; i < 1000; i++) {
    sb.append("item").append(i);
}
String result = sb.toString();
```

### **4. StringBuffer vs StringBuilder**

- StringBuffer：线程安全，方法用synchronized修饰
- StringBuilder：非线程安全，性能更好

### **5. JVM优化**

现代JVM还会进行额外优化：

- 逃逸分析
- 标量替换
- 栈上分配

这就是Java字符串相加的完整底层实现机制，从编译器优化到运行时执行，涉及多个层面的技术细节。
