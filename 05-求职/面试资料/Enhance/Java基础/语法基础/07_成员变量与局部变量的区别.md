# 成员变量与局部变量的区别

成员变量和局部变量是Java中两种重要的变量类型，它们在多个方面存在显著区别：

## 1. 定义位置不同

**成员变量（实例变量/类变量）**：

- 定义在==类内部==，==方法外部==
- 属于类的一部分

**局部变量**：

- 定义在==方法内部==、==代码块内部==或==方法参数中==
- 只在特定作用域内有效

```java
public class Example {
    private int memberVar = 10;  // 成员变量
    
    public void method() {
        int localVar = 20;       // 局部变量
    }
}
```

## 2. 作用域不同

**成员变量**：

- 作用域是整个类
- 可以在类的任何方法中访问（根据访问修饰符）

**局部变量**：

- 作用域仅限于声明它的代码块
- 超出作用域后自动销毁

## 3. 生命周期不同

**成员变量**：

- 随对象创建而创建
- 随对象销毁而销毁
- 生命周期与对象相同

**局部变量**：

- 进入作用域时创建
- 离开作用域时销毁
- 生命周期较短

## 4. 默认值不同

**成员变量**：

- 自动初始化为默认值
- 数值类型：0，布尔型：false，引用类型：null

**局部变量**：

- 必须显式初始化
- 使用前必须赋值，否则编译错误

```java
public class DefaultValueExample {
    private int memberInt;        // 自动初始化为0
    private boolean memberBool;   // 自动初始化为false
    private String memberString;  // 自动初始化为null
    
    public void method() {
        int localInt;            // 必须显式初始化
        // System.out.println(localInt); // 编译错误！
        
        localInt = 5;            // 显式初始化
        System.out.println(localInt); // 正常运行
    }
}
```

## 5. 内存存储位置不同

**成员变量**：

- 存储在堆内存中
- 随对象一起分配在堆上

**局部变量**：

- 存储在栈内存中
- 方法调用时在栈帧中分配

## 6. 访问修饰符

**成员变量**：

- 可以使用访问修饰符（public、private、protected、default）
- 控制外部访问权限

**局部变量**：

- 不能使用访问修饰符
- 只能在声明的作用域内访问

## 7. 静态修饰符

**成员变量**：

- 可以使用static修饰符，成为类变量
- 类变量属于类，所有实例共享

**局部变量**：

- 不能使用static修饰符

```java
public class StaticExample {
    private static int classVar = 100;  // 类变量（静态成员变量）
    private int instanceVar = 200;      // 实例变量
    
    public void method() {
        // static int localVar = 300;    // 编译错误！局部变量不能是static
        int localVar = 300;              // 正确
    }
}
```

## 衍生知识点

### 变量的分类体系

- **成员变量**
    - 实例变量（非static）
    - 类变量（static）
- **局部变量**
    - 方法参数
    - 方法内局部变量
    - 代码块内局部变量

### 变量隐藏（Variable Shadowing）

当局部变量与成员变量同名时，局部变量会隐藏成员变量：

```java
public class ShadowingExample {
    private int value = 10;  // 成员变量
    
    public void method() {
        int value = 20;      // 局部变量，隐藏成员变量
        System.out.println(value);      // 输出20
        System.out.println(this.value); // 输出10，通过this访问成员变量
    }
}
```

### 内存管理影响

- 成员变量过多会增加对象内存占用
- 局部变量使用完毕后立即回收，内存效率更高
- 合理使用有助于程序性能优化

这些区别在实际开发中非常重要，影响着代码的设计、性能和可维护性。
