# 深拷贝浅拷贝

深拷贝和浅拷贝是Java中对象复制的两种不同方式，它们的主要区别在于对引用类型字段的处理方式。

## 浅拷贝（Shallow Copy）

==浅拷贝只复制对象本身和其基本类型字段，但对于引用类型字段，只复制引用地址，不复制引用指向的对象。==

```java
public class Person implements Cloneable {
    private String name;
    private int age;
    private Address address; // 引用类型字段
    
    @Override
    protected Object clone() throws CloneNotSupportedException {
        return super.clone(); // 默认的浅拷贝
    }
}

class Address {
    private String city;
    private String street;
    // 构造方法和getter/setter...
}
```

浅拷贝的问题：

```java
Person person1 = new Person("张三", 25, new Address("北京", "长安街"));
Person person2 = (Person) person1.clone();

// 修改person2的地址
person2.getAddress().setCity("上海");

// person1的地址也会被修改，因为它们共享同一个Address对象
System.out.println(person1.getAddress().getCity()); // 输出：上海
```

## 深拷贝（Deep Copy）

==深拷贝不仅复制对象本身，还递归复制所有引用类型字段指向的对象，创建完全独立的副本。==

### 实现方式1：重写clone方法

```java
public class Person implements Cloneable {
    private String name;
    private int age;
    private Address address;
    
    @Override
    protected Object clone() throws CloneNotSupportedException {
        Person cloned = (Person) super.clone();
        // 深拷贝引用类型字段
        cloned.address = (Address) this.address.clone();
        return cloned;
    }
}

class Address implements Cloneable {
    private String city;
    private String street;
    
    @Override
    protected Object clone() throws CloneNotSupportedException {
        return super.clone();
    }
}
```

### 实现方式2：序列化方式

```java
import java.io.*;

public class DeepCopyUtil {
    @SuppressWarnings("unchecked")
    public static <T> T deepCopy(T obj) {
        try {
            ByteArrayOutputStream bos = new ByteArrayOutputStream();
            ObjectOutputStream oos = new ObjectOutputStream(bos);
            oos.writeObject(obj);
            
            ByteArrayInputStream bis = new ByteArrayInputStream(bos.toByteArray());
            ObjectInputStream ois = new ObjectInputStream(bis);
            return (T) ois.readObject();
        } catch (Exception e) {
            throw new RuntimeException(e);
        }
    }
}

// 使用时类需要实现Serializable接口
public class Person implements Serializable {
    private String name;
    private int age;
    private Address address;
}
```

### 实现方式3：使用第三方库

```java
// 使用Apache Commons Lang
import org.apache.commons.lang3.SerializationUtils;

Person person2 = SerializationUtils.clone(person1);

// 使用Jackson
ObjectMapper mapper = new ObjectMapper();
String json = mapper.writeValueAsString(person1);
Person person2 = mapper.readValue(json, Person.class);
```

## 性能对比

|拷贝方式|性能|内存消耗|复杂度|
|---|---|---|---|
|浅拷贝|高|低|简单|
|深拷贝|低|高|复杂|

## 应用场景

**浅拷贝适用于：**

- 对象的引用类型字段不会被修改
- 性能要求较高的场景
- 简单的对象复制

**深拷贝适用于：**

- 需要完全独立的对象副本
- 避免对象间相互影响
- 原型模式的实现

## 相关知识点扩展

### 1. 原型模式（Prototype Pattern）

深拷贝是原型模式的核心实现，用于创建对象的完整副本。

### 2. 不可变对象（Immutable Objects）

String、Integer等不可变对象在浅拷贝时不会产生问题，因为它们不能被修改。

### 3. 集合类的拷贝

```java
// ArrayList的clone()是浅拷贝
ArrayList<Person> list1 = new ArrayList<>();
ArrayList<Person> list2 = (ArrayList<Person>) list1.clone();
// list2中的Person对象仍然是原对象的引用
```

### 4. 防御性拷贝（Defensive Copying）

在类的构造方法和getter方法中使用深拷贝来保护类的不变性。

这些概念在实际开发中非常重要，特别是在处理复杂对象结构和确保数据安全性时。
