# 泛型上下限、泛型的用法

## 泛型上下限详解

### 什么是泛型上下限？

泛型上下限是Java泛型系统中的重要概念，用于限制泛型参数的类型范围：

- **上界（Upper Bound）**：使用 `<? extends T>` 表示，限制类型必须是T或T的子类
- **下界（Lower Bound）**：使用 `<? super T>` 表示，限制类型必须是T或T的父类

### 语法形式

```java
// 上界通配符
List<? extends Number> upperBound;

// 下界通配符
List<? super Integer> lowerBound;

// 有界类型参数
public class Box<T extends Number> {
    private T value;
}
```

### 上界通配符（? extends T）

**特点**：只能读取，不能写入（除了null）

```java
public class UpperBoundExample {
    public static void main(String[] args) {
        List<Integer> intList = Arrays.asList(1, 2, 3);
        List<Double> doubleList = Arrays.asList(1.1, 2.2, 3.3);
        
        // 都可以赋值给上界通配符
        List<? extends Number> numbers1 = intList;
        List<? extends Number> numbers2 = doubleList;
        
        // 只能读取，返回类型是Number
        Number num = numbers1.get(0);
        
        // 编译错误：无法确定具体类型，不能写入
        // numbers1.add(10); // 编译错误
        // numbers1.add(1.5); // 编译错误
    }
    
    // 实际应用：计算数字集合的总和
    public static double sum(List<? extends Number> numbers) {
        double total = 0;
        for (Number num : numbers) {
            total += num.doubleValue();
        }
        return total;
    }
}
```

### 下界通配符（? super T）

**特点**：只能写入，读取时只能当作Object处理

```java
public class LowerBoundExample {
    public static void main(String[] args) {
        List<Number> numberList = new ArrayList<>();
        List<Object> objectList = new ArrayList<>();
        
        // 都可以赋值给下界通配符
        List<? super Integer> numbers1 = numberList;
        List<? super Integer> numbers2 = objectList;
        
        // 可以写入Integer及其子类
        numbers1.add(10);
        numbers1.add(new Integer(20));
        
        // 读取时只能当作Object
        Object obj = numbers1.get(0);
        // Integer num = numbers1.get(0); // 编译错误
    }
    
    // 实际应用：向集合添加元素
    public static void addNumbers(List<? super Integer> list) {
        list.add(1);
        list.add(2);
        list.add(3);
    }
}
```

### PECS原则

**Producer Extends, Consumer Super**

- **Producer（生产者）**：如果你需要从集合中读取数据，使用 `? extends T`
- **Consumer（消费者）**：如果你需要向集合中写入数据，使用 `? super T`

```java
public class PECSExample {
    // 从source读取数据（Producer），使用extends
    // 向dest写入数据（Consumer），使用super
    public static <T> void copy(List<? extends T> source, 
                                List<? super T> dest) {
        for (T item : source) {
            dest.add(item);
        }
    }
    
    public static void main(String[] args) {
        List<Integer> source = Arrays.asList(1, 2, 3);
        List<Number> dest = new ArrayList<>();
        
        copy(source, dest); // 正确使用PECS原则
        System.out.println(dest); // [1, 2, 3]
    }
}
```

### 有界类型参数

```java
// 单个边界
public class NumberBox<T extends Number> {
    private T value;
    
    public void setValue(T value) {
        this.value = value;
    }
    
    public double getDoubleValue() {
        return value.doubleValue(); // 可以调用Number的方法
    }
}

// 多个边界
public class ComparableBox<T extends Number & Comparable<T>> {
    private T value;
    
    public void setValue(T value) {
        this.value = value;
    }
    
    public boolean isGreaterThan(T other) {
        return value.compareTo(other) > 0; // 可以调用Comparable的方法
    }
}
```

### 实际应用场景

#### 1. 集合操作工具类

```java
public class CollectionUtils {
    // 查找最大值
    public static <T extends Comparable<T>> T max(List<T> list) {
        if (list.isEmpty()) return null;
        
        T max = list.get(0);
        for (T item : list) {
            if (item.compareTo(max) > 0) {
                max = item;
            }
        }
        return max;
    }
    
    // 添加所有元素
    public static <T> void addAll(List<? super T> dest, 
                                  List<? extends T> source) {
        for (T item : source) {
            dest.add(item);
        }
    }
}
```

#### 2. 排序算法

```java
public class SortExample {
    public static <T extends Comparable<? super T>> void sort(List<T> list) {
        Collections.sort(list);
    }
    
    public static <T> void sort(List<T> list, 
                               Comparator<? super T> comparator) {
        Collections.sort(list, comparator);
    }
}
```

### 常见面试问题

#### Q1：为什么上界通配符不能写入？

**答案**：因为编译器无法确定具体的类型。`List<? extends Number>` 可能是 `List<Integer>` 或 `List<Double>`，如果允许写入，可能会导致类型安全问题。

#### Q2：什么时候使用上界，什么时候使用下界？

**答案**：遵循PECS原则

- 需要读取数据时使用上界 `? extends T`
- 需要写入数据时使用下界 `? super T`
- 既需要读取又需要写入时，不使用通配符

#### Q3：能否同时使用上界和下界？

**答案**：不能。`<? extends T super S>` 这种语法是不允许的。

### 总结

泛型上下限是Java类型系统的重要特性，正确使用可以：

1. 提高代码的灵活性和复用性
2. 在编译时保证类型安全
3. 让API设计更加优雅

掌握PECS原则是使用泛型上下限的关键，它能帮助你在不同场景下选择合适的通配符类型。
