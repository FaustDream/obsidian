# 实例变量的默认值

## 实例变量的默认值

在Java中，当创建对象时，实例变量会被自动初始化为其对应类型的默认值。这是Java虚拟机在对象创建过程中自动完成的。

### 各种数据类型的默认值

**基本数据类型的默认值：**

- `byte`: 0
- `short`: 0
- `int`: 0
- `long`: 0L
- `float`: 0.0f
- `double`: 0.0d
- `char`: '\u0000' (空字符)
- `boolean`: false

**引用数据类型的默认值：**

- 所有对象引用（String、数组、自定义类等）: `null`

### 代码示例

```java
public class DefaultValueExample {
    // 基本数据类型实例变量
    private int intValue;
    private double doubleValue;
    private boolean booleanValue;
    private char charValue;
    
    // 引用数据类型实例变量
    private String stringValue;
    private int[] arrayValue;
    
    public void printDefaultValues() {
        System.out.println("int默认值: " + intValue);           // 0
        System.out.println("double默认值: " + doubleValue);     // 0.0
        System.out.println("boolean默认值: " + booleanValue);   // false
        System.out.println("char默认值: " + (int)charValue);    // 0
        System.out.println("String默认值: " + stringValue);     // null
        System.out.println("数组默认值: " + arrayValue);        // null
    }
}
```

### 重要知识点扩展

**1. 局部变量 vs 实例变量**

- 实例变量有默认值，局部变量没有默认值
- 局部变量必须先初始化才能使用，否则编译错误

```java
public void method() {
    int localVar; // 局部变量
    // System.out.println(localVar); // 编译错误！必须先初始化
}
```

**2. 静态变量的默认值**

- 静态变量（类变量）也有默认值，规则与实例变量相同
- 在类加载时初始化，而不是对象创建时

**3. 数组元素的默认值**

```java
int[] numbers = new int[5]; // 所有元素默认为0
String[] names = new String[3]; // 所有元素默认为null
```

**4. 包装类的默认值**

```java
private Integer integerValue; // 默认为null，不是0
private Boolean booleanValue; // 默认为null，不是false
```

**5. 初始化顺序** 当创建对象时，初始化顺序为：

1. 默认值初始化
2. 显式初始化（声明时赋值）
3. 构造代码块初始化
4. 构造方法初始化

### 面试常见问题

**Q: 为什么局部变量没有默认值？** A: 这是Java的设计哲学，强制程序员明确初始化局部变量，避免使用未初始化的值导致的bug。

**Q: 实例变量的默认值是什么时候设置的？** A: 在对象创建过程中，JVM分配内存后立即设置默认值，这发生在任何构造方法执行之前。

这种默认值机制确保了对象的实例变量始终处于一个已知的、安全的状态。
