# 怎么暂停线程

在Java中，线程的暂停有几种不同的方式，我来详细解释一下：

## 1. 使用 wait() 和 notify() 方法

这是最常用的线程暂停机制：

```java
public class ThreadPauseExample {
    private final Object lock = new Object();
    private volatile boolean paused = false;
    
    public void pauseThread() {
        synchronized(lock) {
            paused = true;
        }
    }
    
    public void resumeThread() {
        synchronized(lock) {
            paused = false;
            lock.notify(); // 唤醒等待的线程
        }
    }
    
    public void checkPause() throws InterruptedException {
        synchronized(lock) {
            while(paused) {
                lock.wait(); // 线程进入等待状态
            }
        }
    }
}
```

## 2. 使用 Thread.sleep() 方法

让线程休眠一段时间：

```java
public class SleepExample {
    public void pauseExecution() {
        try {
            Thread.sleep(5000); // 暂停5秒
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }
}
```

## 3. 使用 CountDownLatch

适合需要等待某个条件的场景：

```java
public class CountDownLatchExample {
    private CountDownLatch pauseLatch = new CountDownLatch(1);
    
    public void pauseThread() throws InterruptedException {
        pauseLatch.await(); // 线程等待
    }
    
    public void resumeThread() {
        pauseLatch.countDown(); // 释放等待的线程
    }
}
```

## 4. 使用 Semaphore

通过信号量控制线程的执行：

```java
public class SemaphoreExample {
    private Semaphore semaphore = new Semaphore(0);
    
    public void pauseThread() throws InterruptedException {
        semaphore.acquire(); // 获取许可，没有则等待
    }
    
    public void resumeThread() {
        semaphore.release(); // 释放许可
    }
}
```

## 重要注意事项

**为什么不推荐使用 suspend() 和 resume()**：

- 这两个方法已被废弃（@Deprecated）
- 容易造成死锁问题
- 不安全，可能导致数据不一致

**最佳实践**：

1. 使用 `volatile` 关键字确保可见性
2. 结合 `wait()/notify()` 机制
3. 正确处理 `InterruptedException`
4. 避免在持有锁的情况下调用 `sleep()`

## 扩展知识

**线程状态转换**：

- RUNNABLE → WAITING (调用wait())
- WAITING → RUNNABLE (调用notify())
- RUNNABLE → TIMED_WAITING (调用sleep())

**与线程中断的关系**： 暂停的线程仍然可以响应中断信号，这是一个重要的设计考虑。

这些方法各有适用场景，选择时需要考虑具体的业务需求和性能要求。
