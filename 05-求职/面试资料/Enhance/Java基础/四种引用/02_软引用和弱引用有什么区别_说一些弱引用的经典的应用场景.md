# 软引用和弱引用有什么区别，说一些弱引用的经典的应用场景

## 软引用和弱引用的区别

**软引用（SoftReference）** 和 **弱引用（WeakReference）** 的主要区别在于**垃圾回收的触发条件**不同：

### 软引用（SoftReference）

- **回收条件**：只有在内存不足时才会被垃圾回收器回收
- **生存能力**：相对较强，能够在内存充足时长期存活
- **典型用途**：缓存系统，在内存紧张时自动清理缓存

### 弱引用（WeakReference）

- **回收条件**：只要发生垃圾回收就会被回收，不管内存是否充足
- **生存能力**：较弱，生命周期短暂
- **典型用途**：防止内存泄漏，临时性关联

## 弱引用的经典应用场景

### 1. WeakHashMap - 防止内存泄漏

```java
WeakHashMap<Object, String> weakMap = new WeakHashMap<>();
Object key = new Object();
weakMap.put(key, "value");

key = null; // 断开强引用
System.gc(); // 触发垃圾回收
// key会被回收，对应的entry也会从WeakHashMap中移除
```

**应用场景**：缓存系统中，当key对象不再被其他地方引用时，自动清理缓存条目。

### 2. ThreadLocal的内存泄漏预防

```java
// ThreadLocal内部使用WeakReference
static class ThreadLocalMap {
    static class Entry extends WeakReference<ThreadLocal<?>> {
        Object value;
        Entry(ThreadLocal<?> k, Object v) {
            super(k); // ThreadLocal作为弱引用的key
            value = v;
        }
    }
}
```

**原理**：ThreadLocal对象作为弱引用的key，当ThreadLocal对象被回收时，对应的Entry的key会变为null，帮助识别需要清理的条目。

### 3. 观察者模式中的监听器管理

```java
public class EventManager {
    private List<WeakReference<EventListener>> listeners = new ArrayList<>();
    
    public void addListener(EventListener listener) {
        listeners.add(new WeakReference<>(listener));
    }
    
    public void fireEvent(Event event) {
        Iterator<WeakReference<EventListener>> it = listeners.iterator();
        while (it.hasNext()) {
            WeakReference<EventListener> ref = it.next();
            EventListener listener = ref.get();
            if (listener == null) {
                it.remove(); // 清理已被回收的监听器
            } else {
                listener.onEvent(event);
            }
        }
    }
}
```

**好处**：监听器对象被回收时，不会因为EventManager持有强引用而导致内存泄漏。

### 4. 父子对象关系管理

```java
public class Parent {
    private List<WeakReference<Child>> children = new ArrayList<>();
    
    public void addChild(Child child) {
        children.add(new WeakReference<>(child));
        child.setParent(this); // 子对象持有父对象的强引用
    }
}

public class Child {
    private Parent parent; // 强引用
    
    public void setParent(Parent parent) {
        this.parent = parent;
    }
}
```

**场景**：避免父子对象之间的循环引用导致的内存泄漏。

### 5. 图像缓存系统

```java
public class ImageCache {
    private Map<String, WeakReference<Image>> cache = new ConcurrentHashMap<>();
    
    public Image getImage(String path) {
        WeakReference<Image> ref = cache.get(path);
        if (ref != null) {
            Image image = ref.get();
            if (image != null) {
                return image; // 缓存命中
            }
            cache.remove(path); // 清理已被回收的缓存
        }
        
        // 加载新图像
        Image image = loadImage(path);
        cache.put(path, new WeakReference<>(image));
        return image;
    }
}
```

**优势**：当图像对象不再被使用时，会被自动回收，避免缓存占用过多内存。

## 使用弱引用的注意事项

1. **及时检查null**：使用`get()`方法获取对象时，务必检查返回值是否为null
2. **配合强引用使用**：在需要使用对象的期间，保持一个强引用
3. **线程安全**：在多线程环境中，对象可能在任何时候被回收

弱引用的核心价值在于**自动化的内存管理**，特别适用于缓存、监听器模式和防止内存泄漏的场景。
