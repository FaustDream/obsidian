# HTTP与HTTPS的区别？

HTTP和HTTPS的主要区别体现在以下几个方面：

## 安全性差异

**HTTP（超文本传输协议）**是明文传输，数据在网络中以未加密形式传输，容易被中间人截取和篡改。

**HTTPS（HTTP安全版）**在HTTP基础上加入了SSL/TLS加密层，所有数据都经过加密传输，即使被截取也无法直接读取内容。

## 端口号不同

- HTTP默认使用80端口
- HTTPS默认使用443端口

## 证书认证

HTTPS需要向CA（证书颁发机构）申请SSL证书，用于验证服务器身份的真实性，而HTTP无需证书。

## 性能影响

HTTPS由于需要进行加密解密操作，会消耗更多的CPU资源和时间：

- 需要额外的握手过程建立安全连接
- 加密解密会增加一定的延迟
- 但现代硬件和优化技术已经将这种影响降到很低

## 浏览器显示

- HTTP网站在浏览器地址栏显示"不安全"警告
- HTTPS网站显示锁形图标，表示连接安全

## 相关知识扩展

**SSL/TLS握手过程**：

1. 客户端向服务器发送支持的加密套件列表
2. 服务器选择加密套件并发送证书
3. 客户端验证证书有效性
4. 双方协商生成会话密钥
5. 开始加密通信

**混合加密机制**： HTTPS实际使用非对称加密（RSA/ECDHE）交换密钥，然后使用对称加密（AES）进行数据传输，这样既保证了安全性又提高了效率。

**HTTP/2和HTTP/3**： 现代的HTTP/2和HTTP/3协议都要求使用HTTPS，进一步推动了网络安全的发展。

在实际开发中，建议所有生产环境的网站都使用HTTPS，这不仅保护用户数据安全，也有利于SEO优化。
