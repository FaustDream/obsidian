# 讲讲分布式锁的执行过程

分布式锁的执行过程可以分为**获取锁**、**执行业务逻辑**、**释放锁**三个核心阶段。我来详细解释每个阶段的实现细节。

## 核心执行流程

### 1. 获取锁阶段

```java
public boolean tryLock(String lockKey, String requestId, long expireTime) {
    // 使用SET命令的NX和EX参数原子性地设置锁
    String result = jedis.set(lockKey, requestId, "NX", "EX", expireTime);
    return "OK".equals(result);
}
```

**关键点：**

- 使用唯一的requestId标识锁的持有者
- 设置过期时间防止死锁
- 原子性操作确保线程安全

### 2. 业务逻辑执行阶段

```java
public void executeWithLock(String lockKey, Runnable business) {
    String requestId = UUID.randomUUID().toString();
    try {
        if (tryLock(lockKey, requestId, 30)) {
            // 执行业务逻辑
            business.run();
        } else {
            throw new RuntimeException("获取锁失败");
        }
    } finally {
        // 确保释放锁
        releaseLock(lockKey, requestId);
    }
}
```

### 3. 释放锁阶段

```java
public boolean releaseLock(String lockKey, String requestId) {
    String script = "if redis.call('get', KEYS[1]) == ARGV[1] then " +
                   "return redis.call('del', KEYS[1]) else return 0 end";
    
    Object result = jedis.eval(script, Collections.singletonList(lockKey), 
                              Collections.singletonList(requestId));
    return "1".equals(result.toString());
}
```

## 不同实现方案的执行过程

### Redis分布式锁

1. **SET操作**：`SET lock_key request_id NX EX 30`
2. **业务处理**：执行临界区代码
3. **Lua脚本释放**：原子性验证并删除锁

### Zookeeper分布式锁

1. **创建临时顺序节点**：`/locks/lock_000000001`
2. **判断是否最小节点**：如果是则获得锁
3. **监听前一个节点**：等待通知
4. **删除节点**：释放锁

### 数据库分布式锁

1. **INSERT唯一记录**：插入锁记录
2. **更新时间戳**：续期机制
3. **DELETE记录**：释放锁

## 关键技术细节

### 锁续期机制

```java
// 看门狗机制
private void renewLock(String lockKey, String requestId) {
    ScheduledExecutorService scheduler = Executors.newScheduledThreadPool(1);
    scheduler.scheduleAtFixedRate(() -> {
        if (isLockHeld(lockKey, requestId)) {
            jedis.expire(lockKey, 30); // 续期30秒
        }
    }, 10, 10, TimeUnit.SECONDS);
}
```

### 重试策略

```java
public boolean tryLockWithRetry(String lockKey, String requestId, 
                               long expireTime, int maxRetries) {
    for (int i = 0; i < maxRetries; i++) {
        if (tryLock(lockKey, requestId, expireTime)) {
            return true;
        }
        try {
            Thread.sleep(100 + (int)(Math.random() * 100)); // 随机退避
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            return false;
        }
    }
    return false;
}
```

## 异常处理流程

### 锁超时处理

- **主动续期**：定时任务延长锁有效期
- **被动检测**：业务执行前检查锁是否还存在
- **优雅降级**：锁失效时的补偿机制

### 网络分区处理

- **心跳检测**：定期验证锁状态
- **故障转移**：主节点故障时的切换逻辑
- **一致性保证**：防止脑裂问题

## 性能优化要点

### 减少网络开销

- **批量操作**：合并多个Redis命令
- **连接池复用**：避免频繁创建连接
- **本地缓存**：缓存锁状态信息

### 提高并发度

- **分段锁**：将大锁拆分为多个小锁
- **读写分离**：读操作不加锁或使用读锁
- **异步处理**：非阻塞的锁获取方式

分布式锁的执行过程看似简单，但要处理好各种边界情况和异常场景，需要深入理解分布式系统的特性。在实际应用中，建议优先考虑成熟的框架如Redisson，它已经处理了大部分复杂场景。
