String：Redis 中最基本的数据结构，每个键对应一个字符串值。支持对字符串的直接操作。
> 使用场景：常用来缓存数据库查询结果（当下次再有相同查询时，先从 Redis 中获取数据，若存在则直接返回，大大提高响应速度。）计数器功能。
> 
> 可以通过 SETNX （SET if Not eXists）命令实现简单的分布式锁。

Hash：用于存储键值对集合，类似于 Java 中的HashMap。适合存储对象，每个字段都可以单独进行读写操作。
> 使用场景：存储用户（商品）信息：适合存储包含多个字段的对象，购物车功能

List：按照插入顺序排序的字符串链表。支持从两端进行插入和删除操作，可通过索引获取元素。
> 使用场景：消息队列：（可作为简单的消息队列使用，生产者向列表右侧添加消息，消费者从左侧读取消息，实现异步任务处理。），最新消息展示

Set：无序、唯一的字符串集合。支持添加、删除元素，以及求交集、并集、差集等操作。
> 使用场景：社交功能：点赞和关注功能，抽奖活动

(有序集合Sorted Set)Zset：与集合类似，但每个元素都关联一个分数（score），根据分数进行排序。支持按分数范围或成员来获取元素。
> 使用场景：排行榜,推荐系统
