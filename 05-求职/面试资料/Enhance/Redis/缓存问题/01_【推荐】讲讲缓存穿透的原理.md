# 讲讲缓存穿透的原理

缓存穿透是指**查询一个不存在的数据**，由于缓存中没有命中，系统会直接去数据库查询，而数据库中也没有这条数据，因此不会写入缓存。这样每次查询这个不存在的数据都会穿透缓存直接打到数据库，造成数据库压力过大。

## 缓存穿透的原理

正常的缓存流程：

1. 用户请求数据
2. 先查缓存，命中则直接返回
3. 缓存未命中，查询数据库
4. 数据库有数据，写入缓存并返回

缓存穿透的异常流程：

1. 用户请求**不存在的数据**（如 userId = -1）
2. 缓存中没有该数据
3. 查询数据库，数据库中也没有
4. 由于数据不存在，不会写入缓存
5. 下次同样的请求重复上述流程

## 典型场景

```java
// 用户查询不存在的商品
public Product getProduct(Long productId) {
    // 1. 查缓存
    Product product = redisTemplate.get("product:" + productId);
    if (product != null) {
        return product;
    }
    
    // 2. 查数据库
    product = productMapper.selectById(productId);
    if (product != null) {
        // 3. 写入缓存
        redisTemplate.set("product:" + productId, product);
    }
    // 如果product为null，不会写入缓存，下次请求会再次查询数据库
    return product;
}
```

## 解决方案

### 1. 缓存空对象

```java
public Product getProduct(Long productId) {
    Product product = redisTemplate.get("product:" + productId);
    if (product != null) {
        return "null".equals(product) ? null : product;
    }
    
    product = productMapper.selectById(productId);
    if (product != null) {
        redisTemplate.set("product:" + productId, product, 30, TimeUnit.MINUTES);
    } else {
        // 缓存空对象，设置较短的过期时间
        redisTemplate.set("product:" + productId, "null", 5, TimeUnit.MINUTES);
    }
    return product;
}
```

### 2. 布隆过滤器

```java
@Component
public class BloomFilterService {
    private BloomFilter<Long> bloomFilter;
    
    @PostConstruct
    public void init() {
        // 初始化布隆过滤器
        bloomFilter = BloomFilter.create(Funnels.longFunnel(), 1000000, 0.01);
        // 将所有存在的productId加入布隆过滤器
        List<Long> productIds = productMapper.selectAllIds();
        productIds.forEach(bloomFilter::put);
    }
    
    public Product getProduct(Long productId) {
        // 先用布隆过滤器判断
        if (!bloomFilter.mightContain(productId)) {
            return null; // 一定不存在
        }
        
        // 正常的缓存逻辑
        return getFromCacheOrDB(productId);
    }
}
```

### 3. 参数校验和限流

```java
@RestController
public class ProductController {
    
    @GetMapping("/product/{id}")
    public Product getProduct(@PathVariable Long id) {
        // 参数校验
        if (id == null || id <= 0) {
            throw new IllegalArgumentException("Invalid product id");
        }
        
        return productService.getProduct(id);
    }
}
```

## 扩展知识点

### 缓存雪崩

- **现象**：大量缓存同时失效，请求直接打到数据库
- **解决**：设置随机过期时间、使用多级缓存、熔断降级

### 缓存击穿

- **现象**：热点数据过期瞬间，大量请求同时查询数据库
- **解决**：互斥锁、逻辑过期、预加载

### 实际应用考虑

**监控告警**：

```java
@Component
public class CacheMonitor {
    private final Counter penetrationCounter = Counter.build()
        .name("cache_penetration_total")
        .help("Cache penetration count")
        .register();
        
    public void recordPenetration() {
        penetrationCounter.inc();
    }
}
```

**性能对比**：

- 布隆过滤器：内存占用小，查询快，但有误判率
- 缓存空对象：实现简单，但占用缓存空间，需要设置合理过期时间
- 参数校验：在应用层就拦截，效率最高

缓存穿透虽然是个基础问题，但在高并发场景下影响很大，需要根据业务特点选择合适的解决方案。
