# 有用过MQ对吧，那你说说怎么解决消息重传的？你是怎么做消息幂等性处理的？

好的，我来详细解释一下消息重传和幂等性处理的解决方案。

## 消息重传的解决方案

### 1. 生产者重传机制

```java
@Component
public class MessageProducer {
    
    @Autowired
    private RabbitTemplate rabbitTemplate;
    
    public void sendMessage(String message) {
        // 配置重试机制
        rabbitTemplate.setRetryTemplate(createRetryTemplate());
        
        // 发送消息
        rabbitTemplate.convertAndSend("exchange", "routing.key", message);
    }
    
    private RetryTemplate createRetryTemplate() {
        RetryTemplate retryTemplate = new RetryTemplate();
        
        // 设置重试策略
        FixedBackOffPolicy backOffPolicy = new FixedBackOffPolicy();
        backOffPolicy.setBackOffPeriod(1000); // 1秒间隔
        retryTemplate.setBackOffPolicy(backOffPolicy);
        
        // 设置重试次数
        SimpleRetryPolicy retryPolicy = new SimpleRetryPolicy();
        retryPolicy.setMaxAttempts(3);
        retryTemplate.setRetryPolicy(retryPolicy);
        
        return retryTemplate;
    }
}
```

### 2. 消费者重传处理

```java
@RabbitListener(queues = "message.queue")
public void handleMessage(
    @Payload String message,
    @Header Map<String, Object> headers,
    Channel channel,
    @Header(AmqpHeaders.DELIVERY_TAG) long deliveryTag) {
    
    try {
        // 处理业务逻辑
        processMessage(message);
        
        // 手动确认消息
        channel.basicAck(deliveryTag, false);
        
    } catch (Exception e) {
        // 获取重试次数
        Long retryCount = (Long) headers.get("x-retry-count");
        if (retryCount == null) retryCount = 0L;
        
        if (retryCount < 3) {
            // 重新投递到延迟队列
            sendToRetryQueue(message, retryCount + 1);
            channel.basicAck(deliveryTag, false);
        } else {
            // 超过重试次数，发送到死信队列
            channel.basicNack(deliveryTag, false, false);
        }
    }
}
```

## 消息幂等性处理方案

### 1. 基于消息ID的幂等性

```java
@Service
public class IdempotentMessageProcessor {
    
    @Autowired
    private RedisTemplate<String, String> redisTemplate;
    
    private static final String IDEMPOTENT_KEY_PREFIX = "msg:idempotent:";
    private static final long EXPIRE_TIME = 24 * 60 * 60; // 24小时
    
    public boolean processMessage(String messageId, String messageContent) {
        String key = IDEMPOTENT_KEY_PREFIX + messageId;
        
        // 使用Redis的SET NX命令实现分布式锁
        Boolean result = redisTemplate.opsForValue()
            .setIfAbsent(key, "processed", Duration.ofSeconds(EXPIRE_TIME));
        
        if (Boolean.TRUE.equals(result)) {
            try {
                // 执行业务逻辑
                doBusinessLogic(messageContent);
                return true;
            } catch (Exception e) {
                // 如果业务处理失败，删除幂等性标记
                redisTemplate.delete(key);
                throw e;
            }
        } else {
            // 消息已经处理过，直接返回
            log.info("Message {} already processed", messageId);
            return false;
        }
    }
}
```

### 2. 基于业务唯一性的幂等性

```java
@Service
@Transactional
public class OrderMessageProcessor {
    
    @Autowired
    private OrderService orderService;
    
    public void processOrderMessage(OrderMessage orderMessage) {
        String orderId = orderMessage.getOrderId();
        
        // 使用数据库唯一约束保证幂等性
        try {
            Order existingOrder = orderService.findByOrderId(orderId);
            if (existingOrder != null) {
                log.info("Order {} already exists, skip processing", orderId);
                return;
            }
            
            // 创建订单
            Order order = new Order();
            order.setOrderId(orderId);
            order.setStatus("CREATED");
            // ... 其他字段设置
            
            orderService.save(order);
            
        } catch (DataIntegrityViolationException e) {
            // 唯一约束冲突，说明消息已被处理
            log.info("Order {} already processed by another thread", orderId);
        }
    }
}
```

### 3. 基于状态机的幂等性处理

```java
@Service
public class PaymentMessageProcessor {
    
    @Autowired
    private PaymentService paymentService;
    
    public void processPaymentMessage(PaymentMessage message) {
        String paymentId = message.getPaymentId();
        
        // 获取当前支付状态
        Payment payment = paymentService.findById(paymentId);
        
        if (payment == null) {
            // 创建支付记录
            payment = new Payment();
            payment.setId(paymentId);
            payment.setStatus(PaymentStatus.PENDING);
            paymentService.save(payment);
        }
        
        // 根据当前状态判断是否需要处理
        switch (payment.getStatus()) {
            case PENDING:
                // 处理支付
                processPayment(payment, message);
                break;
            case SUCCESS:
            case FAILED:
                // 已经处理过，直接返回
                log.info("Payment {} already processed, status: {}", 
                    paymentId, payment.getStatus());
                break;
        }
    }
}
```

## 综合解决方案

### 完整的消息处理器

```java
@Component
public class IdempotentMessageHandler {
    
    @Autowired
    private RedisTemplate<String, String> redisTemplate;
    
    @Autowired
    private MessageService messageService;
    
    @RabbitListener(queues = "business.queue")
    public void handleMessage(
        @Payload BusinessMessage message,
        @Header Map<String, Object> headers,
        Channel channel,
        @Header(AmqpHeaders.DELIVERY_TAG) long deliveryTag) {
        
        String messageId = message.getMessageId();
        
        try {
            // 幂等性检查
            if (isMessageProcessed(messageId)) {
                log.info("Message {} already processed", messageId);
                channel.basicAck(deliveryTag, false);
                return;
            }
            
            // 处理业务逻辑
            messageService.processMessage(message);
            
            // 标记消息已处理
            markMessageAsProcessed(messageId);
            
            // 确认消息
            channel.basicAck(deliveryTag, false);
            
        } catch (Exception e) {
            log.error("Failed to process message {}", messageId, e);
            
            // 获取重试次数
            Long retryCount = getRetryCount(headers);
            
            if (retryCount < 3) {
                // 重新投递
                sendToRetryQueue(message, retryCount + 1);
                channel.basicAck(deliveryTag, false);
            } else {
                // 发送到死信队列
                channel.basicNack(deliveryTag, false, false);
            }
        }
    }
    
    private boolean isMessageProcessed(String messageId) {
        String key = "msg:processed:" + messageId;
        return redisTemplate.hasKey(key);
    }
    
    private void markMessageAsProcessed(String messageId) {
        String key = "msg:processed:" + messageId;
        redisTemplate.opsForValue().set(key, "1", Duration.ofHours(24));
    }
}
```

## 关键要点总结

1. **消息重传**：
    
    - 生产者端配置重试机制
    - 消费者端实现重试逻辑
    - 设置合理的重试次数和间隔
    - 使用死信队列处理最终失败的消息
2. **幂等性处理**：
    
    - 消息ID + Redis分布式锁
    - 业务唯一键 + 数据库约束
    - 状态机模式
    - 时间窗口控制
3. **最佳实践**：
    
    - 消息必须包含唯一ID
    - 设置合理的过期时间
    - 考虑并发场景
    - 监控和告警机制

这样的设计既保证了消息的可靠传递，又避免了重复处理的问题。
