MySQL 的默认存储引擎是 InnoDB，底层数据结构是 B+ 树；

- hash 数据结构只适合精准查询，不适合范围查询。
- 二叉树可能会退化为链表。
- 平衡二叉树在插入，修改，删除的时候会有自平衡的过程，自平衡的范围比较大。
- 红黑树是局部自平衡，但是一个节点只能存储一行的数据，会导致扫描磁盘的次数更多，制约MySQL的性能。
- B树的叶子节点和非叶子节点都会存放数据，不适合范围查询。

B+ 树的数据只存放在叶子节点，非叶子节点只存储索引字段数据，可以存储更多的信息，磁盘读写代价更低，并且 B+ 树对叶子节点加了双向链表增强，查询效率更加稳定，便于扫库和范围查询。

hash 本质是一个 map，使用 key 查 value，精准查询时可以使用。
