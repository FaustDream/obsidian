# 什么时候需要建联合索引

联合索引（复合索引）是在多个字段上创建的索引，以下几种情况需要建立联合索引：

## 1. **多字段查询条件**

当查询经常涉及多个字段的WHERE条件时：

```sql
SELECT * FROM user WHERE name = '张三' AND age = 25 AND city = '北京';
```

建立联合索引：`INDEX idx_name_age_city (name, age, city)`

## 2. **最左前缀原则应用**

联合索引遵循最左前缀原则，可以支持多种查询组合：

```sql
-- 联合索引 (name, age, city) 可以支持以下查询：
SELECT * FROM user WHERE name = '张三';
SELECT * FROM user WHERE name = '张三' AND age = 25;
SELECT * FROM user WHERE name = '张三' AND age = 25 AND city = '北京';
```

## 3. **排序优化**

当查询需要按多个字段排序时：

```sql
SELECT * FROM user ORDER BY name, age;
```

建立联合索引：`INDEX idx_name_age (name, age)`

## 4. **覆盖索引需求**

当查询的所有字段都包含在索引中，避免回表查询：

```sql
SELECT name, age FROM user WHERE name = '张三';
```

联合索引 `(name, age)` 可以直接返回结果，无需回表。

## 5. **范围查询优化**

处理范围查询时的性能优化：

```sql
SELECT * FROM order WHERE user_id = 123 AND create_time BETWEEN '2024-01-01' AND '2024-12-31';
```

建立联合索引：`INDEX idx_userid_time (user_id, create_time)`

## 6. **分组统计查询**

GROUP BY多个字段时：

```sql
SELECT department, position, COUNT(*) FROM employee GROUP BY department, position;
```

建立联合索引：`INDEX idx_dept_position (department, position)`

## **联合索引设计原则**

1. **字段顺序很重要**：将选择性高的字段放在前面
2. **考虑查询频率**：高频查询字段优先
3. **避免冗余索引**：不要创建过多类似的联合索引
4. **控制索引长度**：过长的联合索引会影响写入性能

## **实际应用示例**

```sql
-- 电商订单表
CREATE TABLE orders (
    id INT PRIMARY KEY,
    user_id INT,
    status VARCHAR(20),
    create_time DATETIME,
    amount DECIMAL(10,2)
);

-- 常见查询场景的联合索引
INDEX idx_user_status (user_id, status),           -- 查询用户的特定状态订单
INDEX idx_status_time (status, create_time),       -- 按状态和时间范围查询
INDEX idx_user_time (user_id, create_time)         -- 查询用户的时间范围订单
```

合理使用联合索引能显著提升查询性能，但也要注意维护成本和存储空间的平衡。
