# 如何实现分布式锁？

分布式锁是在分布式系统中确保同一时间只有一个节点能够访问共享资源的机制。以下是几种常见的实现方式：

## 1. 基于Redis的分布式锁

### 基本实现

```java
@Component
public class RedisDistributedLock {
    
    @Autowired
    private StringRedisTemplate redisTemplate;
    
    public boolean tryLock(String key, String value, long expireTime) {
        Boolean result = redisTemplate.opsForValue()
            .setIfAbsent(key, value, expireTime, TimeUnit.MILLISECONDS);
        return Boolean.TRUE.equals(result);
    }
    
    public void unlock(String key, String value) {
        String script = "if redis.call('get', KEYS[1]) == ARGV[1] then " +
                       "return redis.call('del', KEYS[1]) else return 0 end";
        redisTemplate.execute(new DefaultRedisScript<>(script, Long.class), 
                            Arrays.asList(key), value);
    }
}
```

### 使用示例

```java
@Service
public class OrderService {
    
    @Autowired
    private RedisDistributedLock distributedLock;
    
    public void processOrder(String orderId) {
        String lockKey = "order_lock:" + orderId;
        String lockValue = UUID.randomUUID().toString();
        
        try {
            if (distributedLock.tryLock(lockKey, lockValue, 30000)) {
                // 业务逻辑处理
                handleOrder(orderId);
            } else {
                throw new RuntimeException("获取锁失败");
            }
        } finally {
            distributedLock.unlock(lockKey, lockValue);
        }
    }
}
```

## 2. 基于ZooKeeper的分布式锁

```java
@Component
public class ZookeeperDistributedLock {
    
    private CuratorFramework client;
    
    public ZookeeperDistributedLock() {
        client = CuratorFrameworkFactory.builder()
            .connectString("localhost:2181")
            .sessionTimeoutMs(5000)
            .connectionTimeoutMs(3000)
            .retryPolicy(new ExponentialBackoffRetry(1000, 3))
            .build();
        client.start();
    }
    
    public void lock(String lockPath, Runnable business) throws Exception {
        InterProcessMutex mutex = new InterProcessMutex(client, lockPath);
        try {
            if (mutex.acquire(30, TimeUnit.SECONDS)) {
                business.run();
            } else {
                throw new RuntimeException("获取锁超时");
            }
        } finally {
            mutex.release();
        }
    }
}
```

## 3. 基于数据库的分布式锁

### 表结构设计

```sql
CREATE TABLE distributed_lock (
    id INT PRIMARY KEY AUTO_INCREMENT,
    lock_name VARCHAR(64) NOT NULL UNIQUE,
    lock_value VARCHAR(255) NOT NULL,
    expire_time TIMESTAMP,
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_lock_name (lock_name)
);
```

### Java实现

```java
@Repository
public class DatabaseDistributedLock {
    
    @Autowired
    private JdbcTemplate jdbcTemplate;
    
    public boolean tryLock(String lockName, String lockValue, long expireTime) {
        try {
            String sql = "INSERT INTO distributed_lock (lock_name, lock_value, expire_time) " +
                        "VALUES (?, ?, ?)";
            Timestamp expireTimestamp = new Timestamp(System.currentTimeMillis() + expireTime);
            
            int result = jdbcTemplate.update(sql, lockName, lockValue, expireTimestamp);
            return result > 0;
        } catch (DuplicateKeyException e) {
            return false;
        }
    }
    
    public void unlock(String lockName, String lockValue) {
        String sql = "DELETE FROM distributed_lock WHERE lock_name = ? AND lock_value = ?";
        jdbcTemplate.update(sql, lockName, lockValue);
    }
    
    // 清理过期锁
    @Scheduled(fixedRate = 60000)
    public void cleanExpiredLocks() {
        String sql = "DELETE FROM distributed_lock WHERE expire_time < ?";
        jdbcTemplate.update(sql, new Timestamp(System.currentTimeMillis()));
    }
}
```

## 4. 使用Redisson实现（推荐）

### 配置

```java
@Configuration
public class RedissonConfig {
    
    @Bean
    public RedissonClient redissonClient() {
        Config config = new Config();
        config.useSingleServer().setAddress("redis://localhost:6379");
        return Redisson.create(config);
    }
}
```

### 使用示例

```java
@Service
public class BusinessService {
    
    @Autowired
    private RedissonClient redissonClient;
    
    public void processWithLock(String resourceId) {
        RLock lock = redissonClient.getLock("business_lock:" + resourceId);
        
        try {
            // 尝试获取锁，最多等待10秒，锁30秒后自动释放
            if (lock.tryLock(10, 30, TimeUnit.SECONDS)) {
                // 业务逻辑
                doBusinessLogic();
            } else {
                throw new RuntimeException("获取锁失败");
            }
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            throw new RuntimeException("获取锁被中断");
        } finally {
            if (lock.isHeldByCurrentThread()) {
                lock.unlock();
            }
        }
    }
}
```

## 各种实现方式的对比

### Redis分布式锁

**优点：**

- 性能高，响应快
- 支持锁过期，避免死锁
- 实现相对简单

**缺点：**

- 可能存在锁误删问题
- 主从切换时可能丢失锁
- 需要考虑时钟偏移问题

### ZooKeeper分布式锁

**优点：**

- 强一致性保证
- 支持阻塞等待
- 自动删除临时节点，避免死锁

**缺点：**

- 性能相对较低
- 运维复杂度高
- 网络分区时可能不可用

### 数据库分布式锁

**优点：**

- 实现简单
- 事务支持好
- 不需要额外组件

**缺点：**

- 性能最差
- 可能成为瓶颈
- 需要定期清理过期锁

## 分布式锁的关键考虑点

### 1. 锁的可重入性

```java
// Redisson支持可重入锁
RLock lock = redissonClient.getLock("reentrant_lock");
lock.lock();
try {
    // 同一线程可以再次获取锁
    method1(); // 内部也使用同一个锁
} finally {
    lock.unlock();
}
```

### 2. 锁的公平性

```java
// 公平锁确保按请求顺序获取锁
RLock fairLock = redissonClient.getFairLock("fair_lock");
```

### 3. 读写锁

```java
// 读写分离，提高并发性能
RReadWriteLock readWriteLock = redissonClient.getReadWriteLock("rw_lock");
RLock readLock = readWriteLock.readLock();
RLock writeLock = readWriteLock.writeLock();
```

### 4. 锁的监控和告警

```java
@Component
public class LockMonitor {
    
    private final MeterRegistry meterRegistry;
    
    public LockMonitor(MeterRegistry meterRegistry) {
        this.meterRegistry = meterRegistry;
    }
    
    public void recordLockAcquisition(String lockName, boolean success, long duration) {
        Timer.Sample sample = Timer.start(meterRegistry);
        sample.stop(Timer.builder("lock.acquisition")
            .tag("lock.name", lockName)
            .tag("success", String.valueOf(success))
            .register(meterRegistry));
    }
}
```

## 最佳实践

1. **选择合适的实现方式**：根据业务需求选择，一般推荐Redis + Redisson
2. **设置合理的过期时间**：避免死锁，但不能太短导致业务未完成锁就释放
3. **使用唯一标识**：确保只有加锁的客户端才能释放锁
4. **异常处理**：在finally块中释放锁，处理各种异常情况
5. **监控和告警**：监控锁的获取成功率、持锁时间等指标
6. **避免长时间持锁**：尽量缩短临界区代码执行时间

分布式锁是分布式系统中重要的同步机制，正确实现和使用对系统的稳定性和性能都有重要影响。
