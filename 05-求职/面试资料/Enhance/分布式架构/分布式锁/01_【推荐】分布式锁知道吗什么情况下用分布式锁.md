# 分布式锁知道吗？什么情况下用分布式锁？

分布式锁是在分布式系统中用来控制多个节点对共享资源访问的一种机制，确保在同一时刻只有一个节点能够访问特定的资源或执行特定的操作。

## 什么是分布式锁？

分布式锁本质上是一种同步原语，它在分布式环境中提供与单机环境中`synchronized`关键字或`ReentrantLock`类似的功能。不同的是，分布式锁需要在多个独立的JVM进程之间协调。

## 什么情况下使用分布式锁？

### 1. **防止重复执行**

```java
// 典型场景：定时任务防重复执行
@Scheduled(fixedRate = 60000)
public void scheduledTask() {
    String lockKey = "scheduled_task_lock";
    if (distributedLock.tryLock(lockKey, 50, TimeUnit.SECONDS)) {
        try {
            // 执行定时任务逻辑
            processScheduledTask();
        } finally {
            distributedLock.unlock(lockKey);
        }
    }
}
```

### 2. **库存扣减**

```java
public boolean decreaseStock(String productId, int quantity) {
    String lockKey = "stock_" + productId;
    if (distributedLock.tryLock(lockKey, 5, TimeUnit.SECONDS)) {
        try {
            int currentStock = stockService.getStock(productId);
            if (currentStock >= quantity) {
                stockService.updateStock(productId, currentStock - quantity);
                return true;
            }
            return false;
        } finally {
            distributedLock.unlock(lockKey);
        }
    }
    return false;
}
```

### 3. **分布式ID生成**

当需要生成全局唯一ID时，确保不同节点生成的ID不冲突。

### 4. **缓存更新**

防止缓存击穿时多个节点同时更新缓存。

## 常见的分布式锁实现方案

### 1. **基于Redis的分布式锁**

```java
public class RedisDistributedLock {
    private RedisTemplate<String, String> redisTemplate;
    private String lockScript = 
        "if redis.call('get', KEYS[1]) == ARGV[1] then " +
        "return redis.call('del', KEYS[1]) " +
        "else return 0 end";
    
    public boolean tryLock(String key, String value, long expireTime) {
        Boolean result = redisTemplate.opsForValue()
            .setIfAbsent(key, value, expireTime, TimeUnit.SECONDS);
        return Boolean.TRUE.equals(result);
    }
    
    public void unlock(String key, String value) {
        redisTemplate.execute(new DefaultRedisScript<>(lockScript, Long.class), 
            Arrays.asList(key), value);
    }
}
```

## 分布式锁需要考虑的问题

### 1. **死锁问题**

- 设置合理的过期时间
- 实现锁的自动续期机制

### 2. **性能问题**

- 选择合适的实现方案（Redis > ZooKeeper > Database）
- 避免长时间持有锁

### 3. **可靠性问题**

- 处理网络分区情况
- 确保锁的原子性操作

### 4. **可重入性**

```java
public class ReentrantDistributedLock {
    private ThreadLocal<Integer> reentrantCount = new ThreadLocal<>();
    
    public boolean lock(String key) {
        Integer count = reentrantCount.get();
        if (count != null && count > 0) {
            reentrantCount.set(count + 1);
            return true;
        }
        
        if (tryLock(key)) {
            reentrantCount.set(1);
            return true;
        }
        return false;
    }
}
```

## 扩展知识点

### 1. **CAP定理对分布式锁的影响**

分布式锁的设计需要在一致性(C)、可用性(A)、分区容错性(P)之间做权衡。

### 2. **红锁(RedLock)算法**

Redis官方提出的分布式锁算法，通过多个Redis实例提高可靠性。

### 3. **与本地锁的区别**

- 性能：分布式锁性能较低
- 复杂性：需要处理网络异常、节点故障等
- 范围：跨JVM进程

### 4. **替代方案**

- 消息队列：通过队列的顺序性保证
- 乐观锁：通过版本号机制
- 分布式事务：如Seata、TCC等

分布式锁是分布式系统中的重要组件，选择合适的实现方案和正确的使用方式对系统的稳定性和性能至关重要。
