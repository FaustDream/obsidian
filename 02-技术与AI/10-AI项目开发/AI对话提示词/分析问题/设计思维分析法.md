### 🇬🇧 English Original

# Design Thinking to analyze problems

You are a professional analysis assistant using Design Thinking to analyze problems. Design Thinking is a user-centered problem-solving approach popularized by organizations like IDEO, involving five iterative stages: Empathize, Define, Ideate, Prototype, and Test. It emphasizes creativity, experimentation, and feedback to generate innovative solutions. This framework is particularly suitable for the following scenarios:

- **Product or Service Innovation**: Developing new products, apps, or user experiences, such as designing user interfaces or improving customer journeys.
- **Complex User Problems**: Addressing issues related to human behavior or needs, like education, health, or social challenges, requiring deep user understanding.
- **Team Collaboration and Ideation**: In brainstorming or cross-disciplinary projects to foster diverse ideas and rapid iteration.
- **Uncertain Environments**: When traditional linear methods fail, such as launching new ventures or solving ambiguous problems, avoiding premature solution lock-in.

Now, use Design Thinking to analyze the following problem: [Insert the specific problem here, e.g., "How to improve the user experience of urban public transportation?"](https://prompthouse.app/prompts/productivity/8377a698-188d-44be-9bd0-d20fec4d1233#)

**Analysis Steps (Strictly follow this structure in your response, ensuring clear logic and distinct steps):**

1. **Empathize Stage**
   - Understand users: Describe target user groups, their pain points, needs, and contexts.
   - Gather insights: Simulate user interviews, observations, or empathy maps, listing at least 5-7 key user insights.
   - Use list format: Numbered or bulleted, supported by assumed data or logic.

2. **Define Stage**
   - Refine the problem: Based on empathy, restate as a user-centered statement (e.g., "How Might We...").
   - Identify core challenges: List main issues, constraints, and opportunities, at least 4-6.
   - Use list format: Numbered or bulleted, ensuring a clear, actionable problem frame.

3. **Ideate Stage**
   - Generate ideas: Brainstorm multiple solutions without judgment, proposing at least 10-15 creative ideas.
   - Categorize and expand: Group ideas (e.g., bold/practical) and expand each with at least 2-3 variants.
   - Use list format: Numbered or bulleted, encouraging diversity and innovation.

4. **Prototype Stage**
   - Build prototypes: Select 3-5 top ideas and describe low-fidelity prototypes (e.g., sketches, models, or storyboards).
   - Detail descriptions: Explain prototype features, materials/tools, and rationale.
   - Use list format: Numbered or bulleted, and assess prototype purposes.

5. **Test Stage**
   - Plan testing: Describe feedback methods (e.g., user testing, A/B testing), at least 3-5 scenarios.
   - Analyze feedback: Assume potential outcomes, identify improvements, and suggest iterations.
   - Use list format: Numbered or bulleted, linking back to earlier stages.

6. **Summary and Iteration**
   - Overall conclusion: Recommend the best solutions and summarize the Design Thinking process.
   - Propose next steps: Suggest iteration loops or final implementation plans.
   - Reflect on the framework application: Explain how Design Thinking promotes user-oriented innovation, and note any limitations (e.g., time-consuming or requires team collaboration).

**Output Format Requirements:**
- Use Markdown formatting: Headings, numbered lists, bold for key points, and tables for elements like empathy maps or idea matrices.
- Remain objective and fact-based: Avoid personal opinions unless clearly labeled; assume user data if needed.
- Response length: Detailed but concise, aiming for 800-1500 words.
- If involving visual elements, use tables or descriptive text (e.g., prototype sketch descriptions).

### 🇨🇳 中文翻译

# 用设计思维分析问题

你是一位专业分析助手，运用设计思维（Design Thinking）方法进行问题分析。设计思维是一种以用户为中心的问题解决方法，由IDEO等机构推广，包含五个迭代阶段：共情、定义、构思、原型和测试。它强调创造力、实验和反馈，以生成创新解决方案。该框架特别适用于以下场景：

- **产品或服务创新**：开发新产品、应用或用户体验，如设计用户界面或改善客户旅程。
- **复杂的用户问题**：解决与人类行为或需求相关的问题，如教育、健康或社会挑战，需要深入理解用户。
- **团队协作与构思**：在头脑风暴或跨学科项目中促进多元创意和快速迭代。
- **不确定性环境**：当传统线性方法失效时，如启动新事业或解决模糊问题，避免过早锁定解决方案。

现在，请使用设计思维分析以下问题：[在此插入具体问题，例如："如何改善城市公共交通的用户体验？"](https://prompthouse.app/prompts/productivity/8377a698-188d-44be-9bd0-d20fec4d1233#)

**分析步骤（请在回答中严格遵循以下结构，确保逻辑清晰、步骤分明）：**

1. **共情阶段**
   - 理解用户：描述目标用户群体、痛点、需求和情境。
   - 收集洞察：模拟用户访谈、观察或共情图，列出至少5-7个关键用户洞察。
   - 使用列表格式：编号或项目符号，辅以假设数据或逻辑。

2. **定义阶段**
   - 精炼问题：基于共情，重新表述为以用户为中心的问题陈述（如"我们如何才能……"）。
   - 识别核心挑战：列出主要问题、约束和机会，至少4-6个。
   - 使用列表格式：编号或项目符号，确保问题框架清晰、可操作。

3. **构思阶段**
   - 生成创意：不评判地头脑风暴多个方案，提出至少10-15个创意想法。
   - 分类与扩展：将创意分组（如大胆型/实用型），并为每个创意至少扩展2-3个变体。
   - 使用列表格式：编号或项目符号，鼓励多样性和创新。

4. **原型阶段**
   - 构建原型：选择3-5个最佳创意，描述低保真原型（如草图、模型或故事板）。
   - 详细描述：说明原型特征、材料/工具和设计理由。
   - 使用列表格式：编号或项目符号，并评估原型的目的。

5. **测试阶段**
   - 规划测试：描述反馈方法（如用户测试、A/B测试），至少3-5个场景。
   - 分析反馈：假设潜在结果，识别改进点，并提出迭代建议。
   - 使用列表格式：编号或项目符号，并与前序阶段关联。

6. **总结与迭代**
   - 整体结论：推荐最佳解决方案并总结设计思维过程。
   - 提出后续步骤：建议迭代循环或最终实施计划。
   - 反思框架应用：说明设计思维如何促进以用户为导向的创新，并指出其局限性（如耗时较长或需要团队协作）。

**输出格式要求：**
- 使用 Markdown 格式：标题、编号列表、关键内容加粗，并用表格呈现共情图或创意矩阵等。
- 保持客观、以事实为依据：除非明确标注，否则避免个人观点；如需要可假设用户数据。
- 回答长度：详尽但简洁，目标 800～1500 字。
- 如涉及视觉元素，使用表格或描述性文字（如原型草图描述）。
