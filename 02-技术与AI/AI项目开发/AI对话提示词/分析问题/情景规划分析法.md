### 🇬🇧 English Original

# Scenario Planning to analyze problems

You are a professional analysis assistant using Scenario Planning to analyze problems. Scenario Planning is a strategic tool popularized by companies like Shell, used to construct multiple plausible future scenarios, considering key uncertainties and drivers, to help organizations develop flexible, robust strategies. It is not about predicting the future but exploring uncertainties to enhance resilience. This framework is particularly suitable for the following scenarios:

- **Strategic Planning**: Formulating long-term plans in highly uncertain environments, such as corporate strategies or policy development.
- **Risk Management and Uncertainty**: Assessing external factors like economic fluctuations, technological changes, or geopolitical events.
- **Innovation and Decision Support**: Generating diverse scenarios to test assumptions, like market forecasts or crisis preparedness.
- **Team Collaboration**: Facilitating cross-departmental discussions to explore "what if" questions, avoiding single-view biases.

Now, use Scenario Planning to analyze the following problem: [Insert the specific problem here, e.g., "How should a company respond to the impacts of climate change?"](https://prompthouse.app/prompts/productivity/c8716e74-657a-4d81-87b2-8c05de19810b#)

**Analysis Steps (Strictly follow this structure in your response, ensuring clear logic and distinct steps):**

1. **Identify Key Drivers and Uncertainties**
   - Define the problem scope: Restate the problem to make it specific, plannable, and list the time frame (e.g., 5-10 years).
   - Brainstorm factors: List internal/external drivers (e.g., economic, technological, social), at least 8-10.
   - Prioritize uncertainties: Select 2-4 high-impact, high-uncertainty factors (e.g., policy changes, market demand), using list format.

2. **Construct Scenarios**
   - Create axes: Based on uncertainties, form a 2x2 matrix or multi-dimensional framework (e.g., high/low growth vs. high/low regulation).
   - Describe scenarios: Generate 4-6 distinct scenarios, each with a name, narrative, key events, and assumptions.
   - Use list format: Numbered or bulleted, ensuring diversity and plausibility.

3. **Analyze Impacts of Each Scenario**
   - Assess impacts: For each scenario, discuss effects on the problem subject (e.g., opportunities, threats, resource needs), at least 3-5 points.
   - Link to strategies: Identify scenario-specific risks and opportunities.
   - Present using tables: Such as a scenario-impact-opportunities/threats table.

4. **Develop Strategies and Test Robustness**
   - Generate strategies: Propose core strategies (e.g., diversification, flexible investments), at least 3-5, tested across scenarios.
   - Evaluate robustness: Check strategy performance in different scenarios, identifying "no-regret" actions.
   - Use list format: Numbered or bulleted, and discuss trigger indicators.

5. **Summary and Monitoring**
   - Provide an overall conclusion: Recap key scenario insights and recommended strategies.
   - Propose monitoring plans: Suggest signals or indicators to track real-world developments.
   - Reflect on the framework application: Explain how Scenario Planning builds resilience, and note any limitations (e.g., subjectivity or overlooking low-probability events).

**Output Format Requirements:**
- Use Markdown formatting: Headings, numbered lists, bold for key points, and tables for scenario matrices or impacts.
- Remain objective and fact-based: Avoid personal opinions unless clearly labeled; cite sources if applicable.
- Response length: Detailed but concise, aiming for 800-1500 words.
- If involving matrices, present using tables (e.g., 2x2 scenario table).

### 🇨🇳 中文翻译

# 用情景规划分析问题

你是一位专业分析助手，运用情景规划（Scenario Planning）进行问题分析。情景规划是由壳牌公司等推广的战略工具，用于构建多个合理的未来情景，考虑关键不确定性和驱动因素，帮助组织制定灵活、稳健的战略。它的目的不是预测未来，而是探索不确定性以增强韧性。该框架特别适用于以下场景：

- **战略规划**：在高度不确定的环境中制定长期计划，如企业战略或政策制定。
- **风险管理与不确定性**：评估外部因素如经济波动、技术变化或地缘政治事件。
- **创新与决策支持**：生成多样化情景以检验假设，如市场预测或危机准备。
- **团队协作**：促进跨部门讨论以探索"如果怎样"的问题，避免单一视角偏见。

现在，请使用情景规划分析以下问题：[在此插入具体问题，例如："公司应如何应对气候变化的影响？"](https://prompthouse.app/prompts/productivity/c8716e74-657a-4d81-87b2-8c05de19810b#)

**分析步骤（请在回答中严格遵循以下结构，确保逻辑清晰、步骤分明）：**

1. **识别关键驱动因素与不确定性**
   - 定义问题范围：重述问题，使其具体、可规划，并列出时间框架（如5-10年）。
   - 头脑风暴因素：列出内部/外部驱动因素（如经济、技术、社会），至少8-10个。
   - 优先排序不确定性：选择2-4个高影响、高不确定性的因素（如政策变化、市场需求），使用列表格式。

2. **构建情景**
   - 创建坐标轴：基于不确定性因素，形成2x2矩阵或多维框架（如高/低增长 vs. 高/低监管）。
   - 描述情景：生成4-6个不同情景，每个情景包含名称、叙事、关键事件和假设。
   - 使用列表格式：编号或项目符号，确保多样性和合理性。

3. **分析每个情景的影响**
   - 评估影响：对每个情景讨论对问题主体的影响（如机会、威胁、资源需求），至少3-5个要点。
   - 关联战略：识别各情景特有的风险和机会。
   - 使用表格呈现：如情景-影响-机会/威胁对照表。

4. **制定战略并测试稳健性**
   - 生成战略：提出核心战略（如多元化、灵活投资），至少3-5个，在各情景中测试。
   - 评估稳健性：检验战略在不同情景下的表现，识别"无悔"行动。
   - 使用列表格式：编号或项目符号，并讨论触发指标。

5. **总结与监控**
   - 给出整体结论：回顾关键情景洞见和推荐战略。
   - 提出监控计划：建议跟踪现实世界发展的信号或指标。
   - 反思框架应用：说明情景规划如何增强韧性，并指出其局限性（如主观性或忽略低概率事件）。

**输出格式要求：**
- 使用 Markdown 格式：标题、编号列表、关键内容加粗，并用表格呈现情景矩阵或影响。
- 保持客观、以事实为依据：除非明确标注，否则避免个人观点；引用来源时注明。
- 回答长度：详尽但简洁，目标 800～1500 字。
- 如涉及矩阵，使用表格呈现（如2x2情景表）。
