# 站点仓库同步通用说明

## 通用结构

- 网站目录本身就是 Git 仓库
- Nginx 直接指向网站目录
- 定时任务统一跑 `/root/repo-sync/sync-repos.sh`
- 仓库清单统一写在 `/root/repo-sync/repos.conf`

## 仓库清单格式

`repos.conf` 每行一个仓库，格式：

```txt
name|path|branch|remote
```

字段含义：

- `name`：同步脚本里使用的仓库标识
- `path`：服务器上的本地 Git 仓库目录
- `branch`：同步分支
- `remote`：远程仓库地址

## 接入步骤

1. 在服务器上准备网站目录，并确保目录本身是 Git 仓库。
2. 为仓库准备可读的 GitHub 认证。
3. 在 `/root/repo-sync/repos.conf` 追加一行仓库配置。
4. 为站点写好 Nginx 配置并 reload。
5. 先手动执行一次同步，确认能拉到最新提交。

## 同步脚本行为

执行：

```bash
bash /root/repo-sync/sync-repos.sh
```

或只同步单仓库：

```bash
bash /root/repo-sync/sync-repos.sh <name>
```

脚本行为：

- 读取 `/root/repo-sync/repos.conf`
- 对目标仓库执行 `fetch -> checkout -> merge --ff-only`
- 工作区不干净时会跳过
- 不会主动 `push`

## 定时任务

当前服务器已有统一 cron：

```cron
17 3 * * * /bin/bash /root/repo-sync/sync-repos.sh >> /root/repo-sync/logs/cron.log 2>&1
```

含义：

- 每天 `03:17` 自动同步全部已登记仓库
- 新项目只要加入 `repos.conf`，就自动纳入定时同步

## 常用检查命令

```bash
cat /root/repo-sync/repos.conf
bash /root/repo-sync/sync-repos.sh <name>
bash /root/repo-sync/show-latest-log.sh
nginx -t
ss -ltn | grep <port>
```
