# 从零连接服务器并发布网页

## 这份文档适合谁

这份文档是写给第一次接触云服务器的人看的，尽量用最少的概念，把事情一步一步做完。

本文默认你是下面这种情况：

- 你在自己电脑上有一个网页项目，或者至少有一个 `index.html`
- 你买了一台云服务器，并且已经拿到了公网 IP、登录账号、密码
- 服务器系统默认按 `Ubuntu 22.04` 讲解
- 你的本地电脑默认按 `Windows` 讲解

如果你的系统不是 Ubuntu，也不用慌，整体思路是一样的，只是命令可能有一点不同。

## 你最后会得到什么

跟着本文做完后，你会得到下面两种访问方式中的一种：

1. 推荐方式：直接在浏览器打开 `http://你的公网IP`
2. 进阶方式：在浏览器打开 `http://你的公网IP:端口号`

对小白来说，最推荐的是第 1 种，也就是用 `Nginx + 80 端口` 发布网页，因为最稳定，也最好理解。

---

## 一、先准备好这些信息

开始之前，请先把下面几项记到一个地方，后面会反复用到：

- 公网 IP：比如 `123.123.123.123`
- 服务器登录账号：常见是 `root` 或 `ubuntu`
- 服务器密码：如果是密钥登录，就准备好密钥文件
- 你的网页项目文件：比如 `dist` 文件夹，或者 `index.html`
- 云厂商控制台入口：阿里云、腾讯云、华为云都可以

如果你连“公网 IP 在哪里看”都不知道，可以这样找：

1. 登录云服务器厂商后台
2. 进入“云服务器”或“实例”页面
3. 找到你买的服务器
4. 查看“公网 IP”

---

## 二、理解一件最重要的事

网页能不能从外网访问，通常要同时满足下面 4 个条件：

1. 你的服务器真的启动了网页服务
2. 网页服务监听的不是本机 `127.0.0.1`，而是外部可访问地址
3. 云服务器安全组放行了对应端口
4. 服务器系统防火墙也放行了对应端口

只要其中有 1 项没做好，浏览器就可能打不开。

---

## 三、连接服务器

### 方法一：用 Windows 自带 PowerShell 连接

1. 在 Windows 中按 `Win + R`
2. 输入 `powershell`
3. 回车
4. 输入下面命令：

```bash
ssh root@你的公网IP
```

例如：

```bash
ssh root@123.123.123.123
```

如果你的账号不是 `root`，就把 `root` 改成你的用户名，比如：

```bash
ssh ubuntu@123.123.123.123
```

第一次连接时，系统可能会问你是否继续，输入：

```bash
yes
```

然后输入密码。输入密码时屏幕一般不会显示字符，这是正常现象，输完直接回车就行。

### 连接成功后，你会看到什么

如果连接成功，屏幕上通常会出现类似下面的提示：

```bash
root@你的服务器名:~#
```

看到这个，说明你已经成功进入服务器了。

---

## 四、先把服务器基础环境装好

连接上服务器后，按顺序执行下面命令。

### 1. 更新软件列表

```bash
sudo apt update
```

### 2. 安装 Nginx

```bash
sudo apt install -y nginx
```

### 3. 启动 Nginx

```bash
sudo systemctl start nginx
```

### 4. 设置开机自启

```bash
sudo systemctl enable nginx
```

### 5. 查看 Nginx 是否正常运行

```bash
sudo systemctl status nginx
```

如果看到 `active (running)`，说明 Nginx 已经正常启动。

你也可以直接在浏览器访问：

```text
http://你的公网IP
```

如果此时就能看到 Nginx 默认欢迎页，说明服务器、Nginx、网络基本已经通了。

---

## 五、上传你的项目文件

对小白来说，最简单的上传方式是用图形化工具，比如：

- WinSCP
- Xftp
- FinalShell

如果你还没有安装，推荐先装 `WinSCP`，操作相对直观。

### 用 WinSCP 上传的大致步骤

1. 打开 WinSCP
2. 新建站点
3. 协议选择 `SFTP`
4. 主机名填写你的公网 IP
5. 用户名填写 `root` 或 `ubuntu`
6. 密码填写你的服务器密码
7. 点击连接

连接成功后：

1. 左边是你本地电脑文件
2. 右边是服务器文件
3. 把你的项目文件拖到服务器里

### 推荐上传到哪个目录

建议你把网页文件放到：

```bash
/var/www/my-web
```

先在服务器里创建这个目录：

```bash
sudo mkdir -p /var/www/my-web
```

然后给当前用户一个可写权限（下面命令里的 `$USER` 不要改）：

```bash
sudo chown -R $USER:$USER /var/www/my-web
```

接着，把你的网页文件上传到这个目录。

### 你应该上传什么

按你的项目类型来：

- 如果你是纯静态网页：上传 `index.html`、`css`、`js`、图片等文件
- 如果你是 Vue、React、Vite 这类前端项目：先在本地执行打包，再把 `dist` 里的内容上传到服务器

如果你是前端打包项目，通常先在本地项目目录执行：

```bash
npm install
npm run build
```

打包完成后，把生成出来的 `dist` 目录中的文件上传到 `/var/www/my-web`

注意：不是把整个源码项目都传上去给浏览器访问，而是优先传“打包后的网页文件”。

## 六、把 Nginx 指向你的项目目录

这一步的意思很简单：

让浏览器访问服务器时，Nginx 知道应该去 `/var/www/my-web` 这个目录里找网页文件。

### 1. 编辑 Nginx 配置

执行：

```bash
sudo nano /etc/nginx/sites-available/my-web
```

把下面内容完整贴进去：

```nginx
server {
    listen 80;
    server_name _;

    root /var/www/my-web;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

说明：

- `listen 80` 表示监听 80 端口
- `root /var/www/my-web` 表示网页文件目录
- `try_files` 这行对前端路由项目很有用，Vue、React 项目一般建议保留

### 2. 启用这个配置

```bash
sudo ln -s /etc/nginx/sites-available/my-web /etc/nginx/sites-enabled/
```

### 3. 删除默认站点配置（避免冲突）

```bash
sudo rm -f /etc/nginx/sites-enabled/default
```

### 4. 检查配置有没有写错

```bash
sudo nginx -t
```

如果看到 `test is successful`，说明配置没问题。

### 5. 重启 Nginx

```bash
sudo systemctl restart nginx
```

到这里，网页服务就已经部署完成了。

## 七、开放公网访问端口

这一步非常关键，很多人明明项目部署好了，但还是打不开，原因就是端口没放行。

公网访问一般要检查两层：

1. 云厂商安全组
2. 服务器系统防火墙

### 1. 放行云厂商安全组

去你的云服务器后台，找到：

- 安全组
- 防火墙
- 入方向规则

然后新增规则：

- `22` 端口：用于 SSH 登录
- `80` 端口：用于网页访问

如果你后面想直接用 `IP:3000` 或 `IP:8080` 访问，还要再额外放行这些端口。

常见填写方式如下：

- 协议：`TCP`
- 端口范围：`80`
- 来源：`0.0.0.0/0`

如果是 SSH：

- 协议：`TCP`
- 端口范围：`22`
- 来源：`0.0.0.0/0`

### 2. 放行服务器自己的防火墙

先查看防火墙状态：

```bash
sudo ufw status
```

如果服务器启用了 `ufw`，就执行：

```bash
sudo ufw allow 22
sudo ufw allow 80
```

如果你要开放其他端口，比如 `3000`，还要执行：

```bash
sudo ufw allow 3000
```

## 八、现在怎么访问网页

### 推荐访问方式

浏览器输入：

```text
http://你的公网IP
```

例如：

```text
http://123.123.123.123
```

这是最简单、最推荐的访问方式。

### 如果你一定要用“公网 IP + 端口”

比如你想访问：

```text
http://123.123.123.123:3000
```

那你必须同时满足下面 3 条：

1. 你的程序真的运行在 `3000` 端口
2. 程序监听的是 `0.0.0.0`，不是 `127.0.0.1`
3. 安全组和防火墙都已经放行 `3000`

很多后端程序默认只监听本机，也就是：

```text
127.0.0.1:3000
```

这样外网是访问不到的。

你要改成监听：

```text
0.0.0.0:3000
```

这样外网才能通过公网 IP 访问。

## 九、如果你的项目不是静态网页，而是 Node 项目

如果你的项目需要通过 `node app.js` 或 `npm run dev` 才能启动，那就不是单纯的静态网页了。

这时推荐做法是：

1. 让 Node 项目运行在服务器的 `3000` 端口
2. 让 Nginx 接收外部的 `80` 端口请求
3. 再由 Nginx 转发到 `3000`

这比直接裸露 `3000` 更规范，也更稳定。

### 例子：Node 项目监听 3000 端口

你的程序需要类似这样启动：

```bash
npm install
npm run start
```

并确保监听地址是：

```text
0.0.0.0:3000
```

### Nginx 反向代理配置示例

如果你是 Node 项目，可以把刚才的 Nginx 配置改成：

```nginx
server {
    listen 80;
    server_name _;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

这表示：

- 浏览器访问 `80` 端口
- Nginx 再把请求转给本机 `3000` 端口上的 Node 项目

## 十、最常见的报错和解决办法

### 1. 浏览器打不开，提示连接超时

优先检查：

1. 公网 IP 是否填对
2. 安全组是否放行 `80`
3. 服务器防火墙是否放行 `80`
4. Nginx 是否启动

检查命令：

```bash
sudo systemctl status nginx
sudo ss -tlnp
```

### 2. 能连接服务器，但网页是空白

优先检查：

1. `/var/www/my-web` 里是不是真的有网页文件
2. 有没有 `index.html`
3. Nginx 配置中的 `root` 路径是否写对

### 3. 访问 `IP:3000` 打不开

优先检查：

1. 程序是否真的启动了
2. 程序监听的是不是 `0.0.0.0:3000`
3. 安全组和防火墙有没有放行 `3000`

检查命令：

```bash
sudo ss -tlnp | grep 3000
```

### 4. 修改了 Nginx 配置后网页打不开

先检查配置：

```bash
sudo nginx -t
```

再重启：

```bash
sudo systemctl restart nginx
```

## 十一、给小白的最短操作路线

如果你只想先快速成功一次，可以只记这几步：

1. 用 `ssh 用户名@公网IP` 连上服务器
2. 执行 `sudo apt update`
3. 执行 `sudo apt install -y nginx`
4. 创建目录 `/var/www/my-web`
5. 把你的 `index.html` 或打包后的 `dist` 文件上传进去
6. 配置 Nginx 指向 `/var/www/my-web`
7. 在安全组放行 `80`
8. 浏览器访问 `http://公网IP`

只要这 8 步走通，你的网页大概率就已经能打开了。

---

## 十二、建议你这样理解“公网 IP + 端口”

可以把它想成：

- 公网 IP：你这台服务器在互联网中的门牌号
- 端口：这台服务器里具体是哪一个服务在接客

例如：

- `123.123.123.123` 是门牌号
- `80` 是网页大厅入口
- `3000` 可能是你自己的程序入口

浏览器访问：

```text
http://123.123.123.123:3000
```

意思就是：

“去 123.123.123.123 这台机器的 3000 号入口找服务。”

---

## 十三、本文使用的默认假设

为了让步骤更容易成功，本文默认做了这些假设：

- 本地电脑是 Windows
- 云服务器系统是 Ubuntu
- 你先部署的是静态网页或前端打包结果
- 推荐优先使用 `Nginx + 80 端口`
- 如果后面是后端项目，再考虑 `3000`、`8080` 之类的端口

如果你后面想继续扩展成：

- 域名访问
- HTTPS 证书
- Node 项目常驻运行
- 数据库部署

可以在这份文档的基础上继续往下加。

## 十四、一个完整的最小成功示例

假设你的服务器公网 IP 是：

```text
123.123.123.123
```

假设你本地有一个 `index.html`，那最小成功流程就是：

1. `ssh root@123.123.123.123`
2. `sudo apt update`
3. `sudo apt install -y nginx`
4. `sudo mkdir -p /var/www/my-web`
5. 上传 `index.html` 到 `/var/www/my-web`
6. 写入 Nginx 配置，让 `root` 指向 `/var/www/my-web`
7. `sudo nginx -t`
8. `sudo systemctl restart nginx`
9. 在云厂商安全组放行 `80`
10. 浏览器打开 `http://123.123.123.123`

如果你能看到网页，就说明已经成功了。
