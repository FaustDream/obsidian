# 执行概算各个字段计算逻辑和值来源

1、**核准概算**：实例概算管理科目表中 概算金额

2、**执行概算**：已发生项目金额（小计）【6】+（拟发生项目金额）（小计）【9】

3、**执行概算余额**：核准概算【1】+执行概算【2】

4、**（已发生项目金额）已签订合同**：已签订合同金额 合同项目金额（元）+已集约合同金额 合同项目金额（元）
    **已签订合同金额 ：合同项目金额（元）：** 表：合同清单、合同清单变更记录子表。  计算逻辑：SUM(projTZJR（合同清单：合价）) + SUM(d.projTZJR1 （合同清单变更记录子表：合价）) - SUM(a.planBasicPrepaid(合同清单：计划基本预备费	)) - SUM(a.planSpreadsPrepaid（合同清单：计划价差预备费	）) 。 根据合同分组。条件：，关联概算，概算不为空（版本：2024新版概算。概算科目不能为“不计科目”）
    **已集约合同金额 ：合同项目金额（元）：** =集约合同项目金额*(分摊到该项目的集约合同项目累计结算金额/集约合同项目累计结算金额）。表：合同清单，资金分摊明细，典型概算表
    c.projTZJR * (SUM(a.amount) / b.amount) 。*集约合同项目金额 （合同清单（合价））** 分摊到该项目的集约合同项目累计结算金额（资金分摊明细（分摊金额））/  集约合同项目累计结算金额（ 资金分摊明细（分摊金额） “典型概算”下该合同的全部资金分配之和）**    条件：概算（版本：2024新版概算。概算科目不能为“不计科目”）。获取到典型概算表数据，然后根据典型概算数据对概算数据进行分组。

5、**（已发生项目金额）已发生费用**：表单：非合同费用支付清单。字段：本次支付金额 （合计）  计算逻辑：predictCost(预备费) IS NULL or = '空' 且 joinBudgetaryEstimates（概算管理科目） IS NOT NULL	（COALESCE(SUM(CASE WHEN (predictCost IS NULL OR predictCost = '空') AND joinBudgetaryEstimates IS NOT NULL THEN paymentMoney ELSE 0 END), 0) AS extra_cost） 条件：概算（版本：2024新版概算。概算科目不能为“不计科目”）

6、**（已发生项目金额）小计**：（已发生项目金额）已签订合同【4】+（已发生项目金额）已发生费用【5】

7、**（拟发生项目金额）拟签订合同**：拟签订合同表 中 概算管理 子表 。字段： 概算科目（费用）estimatedItems ** 。条件： budgetReserve（预备费） IS NULL

8、**（拟发生项目金额）拟发生费用**： 拟发生费用表。 字段 ：金额（元）amount 。条件： `contingency(预备费) IS NULL or = '空'` 且 `joinBudgetaryEstimates(概算科目) IS NOT NULL
`
9、**（拟发生项目金额）小计**：（拟发生项目金额）拟签订合同【7】+（拟发生项目金额）拟发生费用【8】

10、**（累计完成金额）已完成合同**：已签订合同金额  合同项目累计结算金额（元）+ 已集约合同金额  合同项目累计结算金额（元）
    **已签订合同金额** ： **合同项目累计结算金额（元）**：计算逻辑：settlementAmount（报量清单：结算金额） - SUM(actualBasicPrepaid（合同清单：实际基本预备费）) - SUM(ActualSpreadsPrepaid（合同清单：实际价差预备费）) 根据概算管理科目进行分组。 条件：合同清单（有效、子项数据、关联概算不为空）、概算（版本：2024新版概算。概算科目不能为“不计科目”）
    **已集约合同金额** ：  **合同项目累计结算金额（元）**：合同项目（预计）结算金额（元）：资金分摊明细表。字段：分摊金额。计算逻辑：分摊金额合计  条件：概算（版本：2024新版概算。概算科目不能为“不计科目”）

11、**（累计完成金额）已完成费用**：值等于 **（已发生项目金额）已发生费用**【5】

12、**（累计完成金额）小计**：（累计完成金额）已完成合同【10】+（累计完成金额）已完成费用【11】

13、**累计已完成比例**：（累计完成金额）小计【12】  **/**    执行概算【2】

14、**已签订合同（预计）结算金额**：已签订合同金额  合同项目（预计）结算金额（元）+ 已集约合同金额  合同项目（预计）结算金额（元）
    **已签订合同金额** ： **合同项目（预计）结算金额（元）**： 计算逻辑：IF contractState（合同状态） = '已结算' THEN settlementAmount（报量清单：结算金额） ELSE SUM(a.projTZJR) + SUM(d.projTZJR1)（projTZJR：合同清单：合价）+（projTZJR1：合同清单变更记录子表：合价）  根据概算管理科目进行分组。 条件：合同清单（有效、子项数据、关联概算不为空）、概算（版本：2024新版概算。概算科目不能为“不计科目”）
    **已集约合同金额**：  **合同项目（预计）结算金额（元）**：资金分摊明细表。字段：分摊金额。计算逻辑：分摊金额合计  条件：概算（版本：2024新版概算。概算科目不能为“不计科目”）

15、**执行概算（复核）**：已签订合同（预计）结算金额【14】+（已发生项目金额）已发生费用【5】+（拟发生项目金额）小计【9】

注意：**“已签订合同”和“已集约合同”的相关概算金额数据，都是**  **先根据合同进行分组查询对应的数据**，  **然后将查出来概算数据相关的内容保存到对应合同的概算子表中。**
以上涉及到：**“已签订合同”和“已集约合同” 数据的都是这个逻辑，通过 已签订合同清单和已集约合同清单作为中间表。概算相关金额数据都是从这些中间表中的概算子表获取需要的数据。
