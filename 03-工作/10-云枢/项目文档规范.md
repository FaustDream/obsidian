云枢每个文件夹都是一个子项目，所以云枢的每个子项目都要有api.md

```
project/
  AGENTS.md                 # 给所有智能体读：项目规则、入口、禁止事项、验证命令
  README.md                 # 给人读：项目用途、启动方式
  docs/
    architecture.md         # 架构、模块边界、数据流
    decisions.md            # 关键决策记录：为什么这样做
    conventions.md          # 命名、代码风格、目录规范
    api.md                  # 接口、字段、业务对象说明
    changelog-ai.md         # AI/人工改动摘要，只写影响上下文的变化
  services/
    payment/
      AGENTS.md             # 子模块专用规则
```
多人维护时，统一规则应进 Git：`AGENTS.md + docs/ + PR 模板 + changelog-ai.md`。每次改架构、字段、公共方法、业务规则，都要求提交时同步更新对应文档；上下文分界点 = Git 提交，之前靠历史记录，之后靠仓库文档，避免每个人本地记忆不一致。不同智能体也按同一套 `AGENTS.md` 工作；AGENTS.md 本身就是面向编码智能体的 README，已有独立规范项目。
