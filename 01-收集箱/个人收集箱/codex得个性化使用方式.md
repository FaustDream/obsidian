降低 token 的核心做法：`AGENTS.md` 只放“高频、稳定、必须遵守”的内容，控制在 200-500 行以内；大文档放 `docs/`，在 `AGENTS.md` 里写“需要改接口先读 docs/api.md”，让智能体按任务读取。Codex 官方说明：Codex 会按全局、项目、子目录层级读取 `AGENTS.md`，默认项目说明合并上限是 `32 KiB`；Skill 也采用渐进读取，先只暴露名称、描述、路径，命中后才读完整 `SKILL.md`，这正是节省 token 的机制。

多人维护时，统一规则应进 Git：`AGENTS.md + docs/ + PR 模板 + changelog-ai.md`。每次改架构、字段、公共方法、业务规则，都要求提交时同步更新对应文档；上下文分界点 = Git 提交，之前靠历史记录，之后靠仓库文档，避免每个人本地记忆不一致。不同智能体也按同一套 `AGENTS.md` 工作；AGENTS.md 本身就是面向编码智能体的 README，已有独立规范项目。
