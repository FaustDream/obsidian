# 第二阶段：90日认知重建计划

> 周期：第31-120天
> 前提：第一阶段目标80%达标（睡眠稳定、步行习惯形成、自慰/网文可控）
> 目标：把被多巴胺脱敏和14年碎片阅读毁掉的注意力结构重建起来

---

## 一、本阶段的总目标

| 目标维度 | 第30天的状态(预估) | 第120天的目标值 | 判断标准 |
|---------|------------------|----------------|---------|
| 专注时长 | 10-15分钟 | 连续专注30-45分钟 | 使用番茄钟记录 |
| AI依赖 | 写代码必须用AI | 核心功能不使用AI写完 | 独立完成1个功能模块 |
| 独立编程 | 翻旧代码+复制粘贴 | 看懂并手写100行常用代码 | 每日代码回忆测试 |
| 知识输入 | 碎片阅读为主 | 读完1本技术书籍 | 笔记+可复述 |
| 学习习惯 | 偶尔努力 | 固定学习时间(日均30-45min) | 连续21天记录 |
| 自控力 | 仍需要大量意志力 | 一部分变成自动习惯 | 主观感受 |
| 工作摸鱼 | 仍存在 | 工作时专注率≥60% | 工作产出可量化 |

## 二、生理基础与认知训练的并行关系

本阶段 Body 和 Mind 需要**并行推进**。到第120天时，两者应该形成正向循环，而非互相消耗。

### 第31-90天：Body为基础训练，Mind为认知重建

| 周数 | Body 层 | Mind 层 | 核心逻辑 |
|------|---------|---------|---------|
| 第5-6周 | 保持30天建立的基础习惯 | 引入番茄钟：15分钟专注起步 | "低功率启动"——不给前额叶压力 |
| 第7-8周 | 步行时长增至30-40分钟 | 番茄钟增至20分钟+技术阅读 | 注意力肌肉开始重建 |
| 第9-10周 | 每周增加1次低强度运动(散步/拉伸微升级) | 番茄钟25分钟+代码手写练习 | 注意力和技能同步训练 |
| 第11-12周 | 甲亢复查+指标优化 | 独立完成1个小功能(不用AI) | 第一个可验证产出 |

### 第91-120天：Body开始支撑Mind

| 周数 | Body 层 | Mind 层 | 核心逻辑 |
|------|---------|---------|---------|
| 第13-14周 | 步行习惯自动化(不需要意志力) | 番茄钟提升至30分钟+深度阅读 | Body开始为Mind供能 |
| 第15-17周 | 尝试周末轻度徒步/骑行 | 技术文档深度理解+代码重构 | 注意力结构基本重建 |

## 三、周计划模版（第5-12周适用）

### 每日框架（与30天计划的主干保持一致，增加学习块）

```
┌─────────┬──────────────┬──────────────────────────────────┬──────────┐
│  时间   │  活动        │  说明                            │  容错    │
├─────────┼──────────────┼──────────────────────────────────┼──────────┤
│ 07:30   │ 起床+喝水    │ 与30天计划一致                    │ 正常     │
│ 07:35   │ 拉伸5分钟    │ 坚持,已经30天了                   │ 偶尔跳过 │
│ 09:00-18:00│工作       │ ★新增每工作45分钟→番茄钟学习15分钟│ 核心     │
│          │              │ 的"微学习法"(后面祥述)            │          │
│ 18:00-18:45│通勤+晚餐   │ —                                │ —        │
│ ★19:00-19:30│步行       │ 与30天一致,但可以走更快一点      │ 不可跳过 │
│ ★19:30-20:00│固定学习   │ 30分钟,只学一个技能方向         │ ★不可跳过 │
│ 20:00-20:30│自由时间    │ 允许轻度娱乐(≤30min)            │ —        │
│ 20:30-21:00│洗澡+泡脚   │ 放松身体                        │ 正常     │
│ ★21:00   │ 关机        │ 手机放客厅——已成习惯              │ 不可跳过 │
│ 21:00-22:00│自由(有书看) │ 读纸质书/发呆/写日志            │ —        │
│ ★22:00   │ 熄灯        │ 已成习惯                         │ 不可跳过 │
└─────────┴──────────────┴──────────────────────────────────┴──────────┘
```

**关于"微学习法"：**

这是一种专门为你设计的低起点的学习方法。因为你的注意力跨度只有15分钟，所以：

```
工作9:00-18:00 → 在其中插入4-6个"15分钟微学习块"
而不是 → 在晚上额外划出2小时"学习时间"

为什么这样设计：
1. 15分钟——在你的注意力带宽内
2. 放在工作间隙——利用"工作模式"的残留专注
3. 不额外占用休息时间——减少"学习是额外负担"的心理阻力
```

### 微学习块示例：

```markdown
微学习块1 (10:30-10:45): 看一篇React文档,不要求记住,看完即可
微学习块2 (14:00-14:15): 对着昨天的代码,自己重新打一遍(不用AI)
微学习块3 (16:00-16:15): 看一个低代码平台的功能实现,理解思路
```

## 四、逐周分解与缓冲机制

### 第5-6周（第31-44天）：认知训练入门

| 周 | 新增行动 | 缓冲方案 |
|----|---------|---------|
| 第5周 | 每天1个番茄钟(15min),内容随意(技术文档/视频/文章) | 如果太困,延长步行时间,跳过学习 |
| 第6周 | 每天1-2个番茄钟,固定学一个方向(选好了就别换) | 如果焦虑,降低到"只看1个代码片段" |

**为什么这周不能跳过学习？** 因为你30天建立的身体基础(稳定的能量供应)必须在第5周开始接入认知训练,否则这个"能量管道"只用来"不崩溃"而不用来"生长",等于浪费。

**但如果实在太累怎么办？** 允许调整：用"技术播客/视频"替代"阅读",降低认知负载,但维持"每天接触"的频率。

### 第7-8周（第45-58天）：注意力提升

| 周 | 新增行动 | 缓冲方案 |
|----|---------|---------|
| 第7周 | 番茄钟延至20min,周中写一篇"今天学的东西"笔记 | 如果写不出,用语音记录代替 |
| 第8周 | 尝试不用AI写一个小函数(如:日期格式化,数组处理) | 如果写不出,先看文档再默写 |

**注意：** 第8周你可能发现"没有AI我连基本函数都写不出来"。这不是可耻的——这是真实的起点。承认它,然后从那里开始重建。

### 第9-10周（第59-72天）：巩固期

| 周 | 新增行动 | 缓冲方案 |
|----|---------|---------|
| 第9周 | 每天1次"代码回忆"——不查资料写旧代码 | 如果忘记,回看后合上再写 |
| 第10周 | 开始整理"个人代码库"(常用函数/模式/笔记) | 格式不重要,有就行 |

### 第11-12周（第73-86天）：第一个产出

| 周 | 新增行动 | 缓冲方案 |
|----|---------|---------|
| 第11周 | 用2周时间,不依赖AI实现一个完整小功能 | 允许查文档,不允许AI生成代码 |
| 第12周 | 完成并记录这个功能的实现过程 | 不完美也接受 |

**为什么必须有一个"不依赖AI的完整产出"？**

你的大脑神经通路已经被"AI→我不用记→我不用想→我不需要知道→我其实不会"这个回路钝化了。**唯一的修复方式是：在没有拐杖的情况下,走通一次。**

这就像骨折后康复治疗——你必须拆除石膏,慢慢走路,即使摇摇晃晃。一直拄拐杖,肌肉永远长不回来。

### 第13-17周（第87-120天）：加速期

**前提：** 前12周的基础已经打牢——睡眠稳定、步行自动化、番茄钟可用。

| 周 | Body层新增 | Mind层新增 |
|----|-----------|-----------|
| 第13周 | 尝试快走(不是散步速度) | 番茄钟→30min专注 |
| 第14周 | 甲亢复诊+确认指标变化 | 读完第一本技术书的前3章 |
| 第15周 | 周末轻徒步(1-2h不累) | 开始用技术文档做笔记 |
| 第16周 | 尝试1次周末晨跑(3km/快走升级) | 代码库积累至50+条 |
| 第17周 | 评估Body状态 | 评估Mind状态—确认是否可进入第三阶段 |

## 五、学习方向的固定选择

**规则：** 在本阶段90天内只能选一个学习方向。不准换。

### 选项A：前端基础重建（推荐）

**理由：** 你的低代码开发与前端相关,前端基础可以复用当前工作。React/Vue生态招聘需求大。

**学习内容（分周）：**
```
第5-6周: HTML/CSS 基础(你只会低代码用到的,这是补)
第7-8周: JavaScript 基础(变量/函数/作用域/闭包)
第9-10周: DOM操作+事件循环(前端基础的核心)
第11-12周: 框架入门(选React或Vue中一个)
第13-17周: 用框架写一个小demo(不计入工作,纯粹学习)
```

### 选项B：低代码平台底层

**理由：** 你已有4年经验,如果吃透一个低代码平台的底层逻辑,你可以在该领域建立壁垒。

**学习内容：**
```
第5-6周: 搞清楚你用的平台的架构(不是怎么用,是怎么实现的)
第7-8周: 阅读官方文档的"架构"部分
第9-10周: 尝试用原生JS复现平台的某个功能
第11-12周: 做一个小项目的完整实现
第13-17周: 整理成你的技术笔记
```

### 选项C：数据库基础

**理由：** 低代码平台常涉及数据管理,懂数据库是加分项。

**学习内容：**
```
第5-6周: SQL基本语法(CRUD)
第7-8周: 表设计+查询优化
第9-10周: 索引/事务基础
第11-12周: 用SQLite做一个小实践
第13-17周: 将学到的应用于工作场景
```

**选择方法：** 读完描述,凭直觉选一个。不要超过30分钟纠结。选好之后**删除其他选项**,不要让它们分散你的注意力。

## 六、本阶段的"为什么"说明

### 为什么第5-8周只能学15分钟？

> 因为你的前额叶皮层(负责学习和自控的大脑区域)已经10年没有被正常使用过了。
> 
> 10年网文训练出的注意力模式是"快速扫描+即时奖励"。
> 切换到"缓慢理解+延迟满足"的学习模式,需要神经通路的重建。
> 
> 这个过程叫"突触修剪"(Synaptic Pruning)——不用的连接被剪掉,新用的连接被加强。
> 这个过程至少需要6-8周的可感变化,完全巩固需要90天。
> 
> **你不是学不会,你是需要先重新学会"怎么学"。**

### 为什么不能同时学多个方向？

> 因为你的注意力带宽目前是"一条窄路"。
> 
> 一个方向 = 只训练一条路 = 路变宽
> 多个方向 = 同时训练多条路 = 每条路都窄
> 
> 在注意力恢复期,"深"远比"广"重要。

### 为什么要严格限制AI使用？

> AI对你的大脑产生了类似"计算器对心算能力"的影响——你越依赖它,你的能力就越退化。
> 
> 技术开发的核心能力是"理解→记忆→应用→创造"的链条。AI帮你跳过了前两步,直接给了第三步。这让你看起来在"产出",但实际上你的理解和记忆能力在萎缩。
> 
> 90天内禁止AI生成代码,只允许AI解释概念。这是为了重新激活你的"理解→记忆"神经通路。

## 七、缓冲与容错机制

### 每日缓冲：
- 每个番茄钟之间有5-10分钟休息(可以刷手机——这是你的"豁免时间")
- 如果当天特别累,学习时间可缩短至10分钟(但不可跳过)
- 周末的周六设为"完全自由日"——不设任何学习任务,是为了防止疲劳积累

### 每周缓冲：
- 每周至少1天"完全不上钟"(不设番茄钟,不做学习任务)
- 如果一周内连续3天失败→降低下周的番茄钟时长(从25min降至15min)
- 如果连续2周失败→回到第一阶段,再加30天Body基础

### 每月缓冲：
- 每30天做一个复盘,但不是"分析问题",而是"调整节奏"
- 如果感觉极度疲惫→休息2天(不做任何事,包括步行)

## 八、紧急应对方案

| 情况 | 处理 |
|------|------|
| 15分钟学习都坚持不了 | 降到5分钟。5分钟总可以吧？ |
| 学习时极度烦躁想关掉 | 做3次深呼吸,然后继续2分钟→如果还想关,允许关 |
| 觉得"学了没用" | 写下"我现在学的内容,和我的工作有什么关系" |
| 回到旧习惯(连续3天不学) | 不要自责。重新从5分钟开始。 |
| 身体出现新问题(加剧的不适) | 暂停学习,去看医生。身体优先。 |
| 工作突然变忙 | 维持学习的"存在感"就行(每天5分钟看一段代码) |
| 失去动力 | 记住: 前90天不靠动力,靠纪律。动力在第90天后才会出现。 |

## 九、120天后的验收标准

**进入第三阶段（职业向上）的前提条件：**

| 标准 | 衡量方式 |
|------|---------|
| ☐ 连续专注30分钟不碰手机 | 番茄钟记录 |
| ☐ 独立完成1个功能模块(无AI) | 代码产出 |
| ☐ 记住并使用20+常用API | 代码回忆测试 |
| ☐ 日均屏幕使用时间(娱乐)≤2h | 手机记录 |
| ☐ 自慰稳定在≤1次/周 | 主观记录 |
| ☐ 睡眠稳定(23:00前入睡≥80%天数) | 睡眠记录 |
| ☐ 步行自动化(不需要提醒) | 习惯养成 |
| ☐ 体重减轻2-3kg或体感更轻盈 | 主观记录 |

**达标？进入第三阶段。**
**未达标？** 检查原因：是计划太严（降低标准）还是执行太松（加强环境设计）。调整后继续本阶段。**不要急于进入下一阶段——Vocation问题比认知重建更耗费能量。**
