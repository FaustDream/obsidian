# 第三阶段：180日职业向上计划

> 周期：第121-300天
> 前提：前两阶段达标——睡眠稳定、身体有能量、注意力可连续30分钟以上
> 目标：将重建的认知能力转化为职业竞争力，实现薪资突破

---

## 一、本阶段的总目标

| 目标维度 | 前两阶段基础 | 第300天目标 | 判断标准 |
|---------|-------------|------------|---------|
| 编程能力 | 可独立完成功能模块 | 可独立负责一个完整项目 | 项目交付记录 |
| 核心技能 | 20+常用API | 100+核心API/模式掌握 | 代码测验 |
| AI使用 | 完全禁用AI生成代码 | 恢复AI使用,但控制节奏 | 优先级:自己>AI |
| 工作状态 | 部分专注 | 工作日产出稳定 | 周报/进度 |
| 薪资水平 | 7k | 8-10k | 跳槽或加薪 |
| 可验证作品 | 无 | 1-2个个人项目 | GitHub/部署链接 |
| 资产状况 | 9.5w存款 | 存款稳定/略有增长 | 月度盘点 |
| 身体基础 | 步行习惯·甲亢稳定 | 保持+轻度运动习惯 | 体重≤80kg |

## 二、为什么第三阶段才能处理职业问题

前两个阶段（共120天）都是"修复你自己"——把你的操作系统从"崩溃模式"恢复到"基本可用模式"。只有在基本可用模式下，才能开始讨论"性能优化"（职业向上）。

**类比：**
```
前120天 → 把一台蓝屏死机的电脑修到能开机
第121-300天 → 开始装软件、联网、跑应用
第301-365天+ → 升级硬件、多任务处理
```

如果你在前两阶段没有打好基础就跳到第三阶段，你会：
1. 学习压力触发身体的逃避回路（熬夜看小说）
2. 技能提升受挫后产生"我不行了"的归因（下跪心理激活）
3. 最终回到原点

**所以，第三阶段的前提条件不是"想进步"，而是"有能力进步"。**

## 五、逐月分解

### 第5个月（第121-150天）：职业评估期

**核心动作：**
1. 盘点当前技能——写一份诚实的"我能做什么/我不能做什么"
2. 市场调研:
   - 在招聘网站搜"低代码"+"前端"+"广州",看薪资区间
   - 记录10个目标岗位的JD关键词
3. 方向确认——选定一个技能提升方向（不准中途换）
4. 开始整理工作成果——记录工作中完成的功能、解决的问题

**每晚的60分钟学习安排：**
```
前15分钟: 看招聘JD+记录市场需求
中间30分钟: 对比自身差距,选定一个差距补充
后15分钟: 将学到的写进"技能笔记"
```

**每周缓冲：**
- 周六不设学习任务,但必须花30分钟更新"技能地图"
- 周日晚上做一次"本周学了什么"的复习(15分钟)

### 第6-7个月（第151-210天）：技能深耕期

**核心动作：**
1. 选定技能方向后,制定一个60天的深入学习计划
2. 每天的学习产出是一段可运行的代码（不是笔记）
3. 每2周完成一个"微型项目"（如：一个登录页面、一个API接口）
4. 开始在工作中有意识地用新学的技能（慢慢替代AI依赖）

**60天深耕计划模版：**
```
第1-2周: 精读你选的方向的核心文档
第3-4周: 每天手写20行代码(不用AI)+理解每一行
第5-6周: 写一个小项目(如:个人博客后端/前端组件库)
第7-8周: 把项目部署到GitHub Pages/免费服务器
```

**每周缓冲：**
- 每2周可以休1天学习假——完全不学,彻底放松
- 如果某天写不出代码→回看之前写的,理解即可

### 第8-9个月（第211-270天）：项目产出期

**核心动作：**
1. 完成1-2个可公开的个人项目
2. 项目标准:
   - 独立完成（允许查文档,不允许AI生成核心逻辑）
   - 完整功能（从前端到后端,或者一个完整的组件）
   - 写README（说明架构、技术选型、你的贡献）
3. 把项目部署到可访问的环境
4. 更新简历——把项目写入"个人项目"部分

**可产出类型（选一个）：**
```
选项A: 一个低代码平台的增强插件(利用你的4年经验)
选项B: 一个React/Vue组件库(展示前端能力)
选项C: 一个数据面板(展示数据库+中间件能力)
```

**为什么要有"可公开项目"？**
```
面试官看简历时，最有说服力的不是你写"熟悉React",而是你有一个部署在网上的React项目。
可验证产出是打破"AI依赖"指控的唯一证据。
```

### 第10个月（第271-300天）：价值变现准备

**核心动作：**
1. 更新简历和招聘平台信息
2. 投递3-5家目标公司的岗位（作为测试）
3. 记录面试中暴露的弱点
4. 针对弱点做1-2周的补强
5. 决定是跳槽还是找当前公司谈加薪

**为什么是"测试投递"不是"正式跳槽"？**
```
目的是收集市场反馈，不是立刻换工作。
通过实际面试，你才知道：
  - 你的能力在市场上的真实价格
  - 你还有哪些盲区
  - 当前的晋升/跳槽时机是否成熟
面试不是"去被考核",是"去收集信息"。
```

## 六、职业方向选择（正式版）

基于你的背景（4年低代码）和当前重建后的基础能力，推荐以下路径排名：

### 推荐1：低代码平台专家（最稳）

**理由：** 你可以利用4年积累的经验优势。在这个垂直领域深耕,比从零学一个全新的领域更快见效。

**路径：**
```
深入掌握你所用平台的全部功能
↓
学习平台的扩展/插件开发
↓
成为一个"低代码+定制开发"的混合型人才  
↓
市场定位: 低代码不是低端,而是效率——你需要证明"我既能用低代码快速交付,也能在需要时写原生代码"
```

### 推荐2：前端开发（市场最广）

**理由：** 前端开发是技术岗位中最多、最稳定、门槛相对入门的选择。你已经有低代码开发经验,对前端框架有概念。

**路径：**
```
补齐原生三件套(HTML/CSS/JS) → 第5-8周
深入学习React或Vue(选一个) → 第9-20周
用框架做1-2个项目 → 第20-30周
准备面试题+刷算法入门 → 第30-40周
```

### 推荐3：运维/DevOps（体感最不累）

**理由：** 运维不需要持续的高强度编码,对注意力要求低于开发。结合AI运维工具会有较好前景。

**路径：**
```
Linux基础 → 网络基础 → 容器化(Docker) → CI/CD → 云服务
(第5个月开始,每月一个主题)
```

**选择决策规则：** 在第121-135天内,每周花1晚时间了解一个方向,第136天必须选定。选定后删除其他选项。

## 七、财务恢复策略

职业向上的最终目标是"收入和财务状况的可持续改善"。本阶段同步进行财务优化：

### 当前财务状态回顾

| 项目 | 第120天预估 | 问题 |
|------|-----------|------|
| 存款9.5w | 可能会略高或持平 | 可以维持 |
| 黄金7w(占比74%) | 过高 | 不建议操作(精力有限) |
| 车贷5w | 持续还贷中 | 正常月供 |
| 月存3.5k | 如涨薪可增至4-5k | 合理 |

### 本阶段财务策略

1. **不要动黄金。** 当前精力集中在技能提升上,没时间研究黄金走势。亏损的3k就当是学费。
2. **继续维持高储蓄率。** 就算涨到10k月薪,月存5k左右即可。不要因为涨薪就扩大消费。
3. **建议加的保障：** 如果公司有补充医疗保险→投保。因为你的甲亢需要长期治疗。
4. **不建议做的事：** 借贷投资、购买理财课程、花钱买"涨薪课程"。

## 八、缓冲与容错

### 阶段级缓冲

| 缓冲级别 | 触发条件 | 应对 |
|---------|---------|------|
| 轻度 | 一周内任务完成率<60% | 暂停新任务,复习已学内容 |
| 中度 | 连续2周无法专注 | 回到第二阶段维持1-2周 |
| 重度 | 身体出现不适(甲亢加重/肩颈疼痛加重) | 暂停所有学习,只保留基础习惯 |

### 职业转变的缓冲原则

- **不辞职。** 在任何情况下,不裸辞。工作是你唯一的收入来源和社保来源。
- **不借贷** (用于学习或过渡期)。
- **不急于跳槽。** 多拿几个offer后,才能做信息对称的决策。
- **不接受低于当前收入的offer。** 这不是"从头开始",你值得留在这个水平。
